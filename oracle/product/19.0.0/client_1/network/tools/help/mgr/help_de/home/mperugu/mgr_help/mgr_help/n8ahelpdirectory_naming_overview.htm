Überblick über die Verzeichnisbenennung

Verzeichnisbenennung ist eine der wichtigsten Benennungsmethoden, mit der ein Connect-Bezeichner als Verbindungsdeskriptor aufgelöst wird, mit dem der Client die Verbindung zu einer Datenbank herstellt. Bei der Verzeichnisbenennung werden Connect-Bezeichner und deren Verbindungsdeskriptoren in einem LDAP-konformen Directory Server gespeichert. Die Connect-Bezeichner werden unter einem Unterbaum Oracle Context gespeichert, der Einträge zur Benutzung mit der Oracle-Software enthält.

Ein Endbenutzer gibt einen Connect String, der den Connect Identifier enthält, in folgendem Format ein:

CONNERCT username/password@connect_identifier

Im folgenden Beispiel ist sales der Connect Identifier, der in dem Verzeichnis gefunden wird:

CONNECT scott/tiger@sales

Der Verbindungsdeskriptor, der an den Client zurückgegeben wird, umfasst:

  • Protokolladresse des Listeners

  • Service-Name der Datenbank

Der Client verwendet diese Protokolladress-Informationen, um sich bei einem Listener anzumelden. Der Listener, der mit denselben Adressinformationen konfiguriert ist, empfängt die Anforderung. Wenn der Listener auf demselben Datenbankdienst auf Anforderungen hört, wie in dem Verbindungsdeskriptor angegeben, wird eine Verbindung hergestellt.

Damit die Verzeichnisbenennung vorgenommen werden kann, konfigurieren Sie einen Directory-Server für Oracle Net Services Features und Clients für den Zugriff auf das Verzeichnis.

Weitere Informationen finden Sie im Kapitel Configuration Management Concepts im Oracle10i Net Administrator's Guide.
