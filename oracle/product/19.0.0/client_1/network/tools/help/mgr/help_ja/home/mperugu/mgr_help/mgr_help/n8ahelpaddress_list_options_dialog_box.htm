「アドレス・リスト・オプション」ダイアログ・ボックス

「アドレス・リスト・オプション」ダイアログ・ボックスでは、接続記述子内のプロトコル・アドレスを接続に使用する順序と使用方法を指定できます。

成功するまで順番に各アドレスを入力

このオプションを選択すると、Oracle9iまたはOracle8iのクライアントで接続時フェイルオーバーが有効になります。最初のアドレスが接続できない場合、クライアントではフェイルオーバーにより次のリスナー・プロトコル・アドレスを利用できます。このオプションおよび「Net8 8.0クライアントと互換性のあるオプションを使用」オプションを選択して、リリース8.0以前のクライアントでソース・ルーティングをオフにします。

成功するまでランダムに各アドレスを入力

このオプションを選択すると、クライアント・ロード・バランシングおよび接続時フェイルオーバーがOracle9iまたはOracle8iのリリースのクライアントで使用可能になります。

任意に選択したアドレスの1つを入力

このオプションを選択すると、クライアント・ロード・バランシングがOracle9iまたはOracle8iのリリースのクライアントで使用可能になります。

接続先に到達するまで各アドレスを順番に使用

このオプションを選択すると、ソース・ルーティングがOracle8iまたはOracle8で使用可能になります。

接続先に到達するまでに2回以上のアドレス・ストップが必要な場合に使用します。通常、この機能を使用すると、Oracle Connection Manager機能が有効になります。

このパラメータは、Oracle9iクライアントには必要ありません。

最初のアドレスのみ使用

このオプションを選択すると、ソース・ルーティング、接続時フェイルオーバーおよびクライアント・ロード・バランシング機能が使用禁止になります。

Net8 8.0クライアントと互換性のあるオプションを使用

リリース8.0のクライアントを設定するときに選択します。この場合、選択できるのは次のオプションのみです。

-成功するまで順番に各アドレスを入力

接続先に到達するまで各アドレスを順番に使用

Oracle9iまたはOracle8iのリリースのクライアントを構成する場合は、このオプションの選択を解除します。
