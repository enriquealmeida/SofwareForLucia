Oracle Advanced Security: Integridad

En el separador Integridad puede configurar la integridad de los datos para los métodos de autenticación. La integridad de los datos es la comparación de los datos enviados por la conexión, tal como los ven el cliente y el servidor.

Nota: Utilice el separador SSL si desea configurar Series Cipher para SSL (Secure Sockets Layer).

Integridad

Active la casilla de control Servidor o Cliente para indicar si la computadora que está configurando actualmente es un Cliente o un Servidor. La opción Servidor no se aplica a los servidores a los que se conecta la computadora.

Nivel de Total de Control

Seleccione un nivel de total de control. Los cuatro niveles posibles son:

Elemento Descripción
aceptado El servicio estará activo si el otro lado de la conexión especifica "necesario" o "solicitado" y en el otro lado está disponible un algoritmo compatible. De lo contrario, estará inactivo.
rechazado El servicio no debe estar activo y la conexión fracasará si el otro lado especifica "necesario".
solicitado El servicio estará activo si el otro lado de la conexión especifica "aceptado", "necesario" o "solicitado" y en el otro lado está disponible un algoritmo compatible.
necesario El servicio debe estar activo y la conexión fracasará si el otro lado especifica "rechazado" o si no existe un algoritmo compatible en el otro lado.

Lista Métodos Disponibles

Muestra los algoritmos de integridad que se utilizan para crear totales de control. Hay dos métodos:

Nombre del Algoritmo Valores Válidos
Algoritmo de Comprobación Aleatoria Protegido (SHA-1) SHA1
Message Digest 5 (MD5) MD5

Lista Métodos Seleccionados

Muestra los algoritmos seleccionados. El orden de éstos en la lista determina su orden de uso.
