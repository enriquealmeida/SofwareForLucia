データの整合性の構成

データの整合性のチェックは、メッセージの内容に基づいて数量が計算されるプロセスです。メッセージの送信側がチェックサムを計算します。チェックサムの値は、メッセージ・パケットの一部として組み込まれます。メッセージの受信側は、受信時にメッセージのチェックサムを再計算します。この2つのチェックサム値が一致すると、メッセージが転送中に変更されていないものとみなされます。

データの整合性を構成する手順は、次のとおりです。

  1. ナビゲータ・ペインで、「Oracle Netの構成」→「ローカル」→「プロファイル」を展開します。

  2. 右ペインのリストから「Oracle Advanced Security」を選択します。

  3. 「整合性」タブを選択します。

  4. 「クライアント」または「サーバー」のいずれかのチェック・ボックスを選択して、クライアントまたはサーバーのデータの整合性を使用可能にします。

  5. クライアントとサーバーによるデータの整合性のネゴシエーション方法を指定します。クライアントとサーバーの両方について、「チェックサム・レベル」を選択する必要があります。レベルの説明は次のとおりです。

    要素 説明
    適用 一方の接続側で「必要」または「要求」が指定されており、もう一方の接続側で矛盾のないアルゴリズムが使用可能になっている場合、サービスはアクティブになります。それ以外の場合、サービスはアクティブにはなりません。
    拒否 サービスはアクティブにはなりません。一方の接続側で「必要」が指定されている場合、接続は失敗します。
    要求 一方の接続側で「適用」、「必要」または「要求」が指定されており、矛盾のないアルゴリズムが使用可能になっている場合、サービスはアクティブになります。それ以外の場合、サービスはアクティブにはなりません。
    必要 サービスはアクティブになります。一方の接続側で「拒否」が指定されている場合、または両立するアルゴリズムがない場合、接続は失敗します。

  6. 「ファイル」→「ネットワーク構成の保存」をクリックします。
