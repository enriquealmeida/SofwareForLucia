Fichier listener.ora

Fichier de configuration du processus d'écoute indiquant les éléments suivants :

  • Nom du processus d'écoute

  • Adresses de protocole sur lesquelles le processus d'écoute accepte les demandes de connexion

  • Services (service de base de données ou service autre qu'un service de base de données) écoutés par le processus d'écoute

Le fichier listener.ora doit résider dans l'un des emplacements suivants :

  1. Le répertoire indiqué par la variable d'environnement TNS_ADMIN. Si la variable d'environnement TNS_ADMIN n'est pas définie en tant que variable dans Windows NT, il se peut qu'elle soit dans la base de registres.

  2. Le répertoire de configuration global du noeud. Pour Sun Solaris, ce répertoire est /var/opt/oracle. Windows NT ne dispose pas d'un répertoire central.

  3. Le répertoire $ORACLE_HOME/network/admin dans UNIX ou le répertoire ORACLE_HOME\network\admin dans les systèmes d'exploitation Windows.

Remarque : Oracle9i et Oracle8i ne requièrent aucune identification du service de base de données grâce à la fonction d'inscription de service. Toutefois, si vous prévoyez d'utiliser Oracle Enterprise Manager pour gérer la base de données, vous devez en configurer le service statiquement.
