Configuración de Opciones de Varias Direcciones

Si se puede acceder a un servicio de base de datos mediante varias direcciones de protocolo del listener, es importante especificar el orden de uso de las direcciones. Por ejemplo, puede seleccionar las direcciones aleatoriamente o probarlas secuencialmente.

Si se han configurado varias direcciones de protocolo para un nombre de servicio de red o un servicio de base de datos, se pueden configurar las siguientes opciones:

  • Redirección Automática en Caso de Fallo del Tiempo de Conexión para seleccionar la primera dirección de protocolo y utilizar la siguiente dirección si falla la primera.

  • Equilibrio de Carga del Cliente para seleccionar una dirección de protocolo aleatoriamente.

  • Direccionamiento de Origen para utilizar secuencialmente todas las direcciones de protocolo.

Para configurar opciones de varias direcciones:

  1. En el panel del navegador, amplíe Configuración de Red de Oracle > Directorio > Local o Nomenclatura de Servicios.

  2. Seleccione un nombre de servicio de red o de base de datos. El panel de la derecha muestra el servicio de destino actual y una lista de direcciones.

  3. En el campo Configuración de Direcciones, haga clic en Avanzado. Aparece el recuadro de diálogo Opciones de la Lista de Direcciones.

  4. Desactive Usar Opciones Compatibles con Net8 8.0 Clients si realiza la configuración para un cliente de Oracle9i u Oracle8i.

  5. Seleccione las opciones adecuadas para la lista de direcciones de protocolo, a continuación haga clic en Aceptar.

  6. Haga clic en Aplicar en el panel de la derecha.

  7. Si desea realizar estos cambios en la carpeta Local, seleccione Archivo > Guardar Configuración de Red.
