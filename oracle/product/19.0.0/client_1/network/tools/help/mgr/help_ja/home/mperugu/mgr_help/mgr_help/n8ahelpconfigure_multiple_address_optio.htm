複数のアドレス・オプションの構成

データベース・サービスに複数のリスナー・プロトコル・アドレスからアクセスできるようにする場合は、プロトコル・アドレスの使用順序を指定することが重要です。たとえば、プロトコル・アドレスをランダムに選択する方法と、順番に試行する方法があります。

1つのネット・サービス名またはデータベース・サービスに対して複数のプロトコル・アドレスが構成されている場合は、次のオプションを構成できます。

  • 接続時フェイルオーバー。最初のプロトコル・アドレスを選択し、そのプロトコル・アドレスにアクセスできない場合は、次のプロトコル・アドレスにフェイルオーバーします。

  • クライアント・ロード・バランシング。プロトコル・アドレスをランダムに選択します。

  • ソース・ルーティング。すべてのプロトコル・アドレスを順番に使用します。

複数のアドレス・オプションを構成する手順は、次のとおりです。

  1. ナビゲータ・ペインで、「Oracle Netの構成」→「ディレクトリ」→「ローカル」または「サービス・ネーミング」を展開します。

  2. ネット・サービス名またはデータベース・サービスを選択します。右ペインに現行の接続先サービスおよびアドレス・リストが表示されます。

  3. 「アドレスの構成」ボックスで「詳細」を選択します。「アドレス・リスト・オプション」ダイアログ・ボックスが表示されます。

  4. Oracle9iまたはOracle8iのクライアント用に構成する場合は、「Net8 8.0クライアントと互換性のあるオプションを使用」の選択を解除します。

  5. プロトコル・アドレスのリストに適したオプションを選択し、「OK」を選択します。

  6. 右ペインで「適用」を選択します。

  7. この変更を「ローカル」フォルダに対して行う場合は、「ファイル」→「ネットワーク構成の保存」を選択します。
