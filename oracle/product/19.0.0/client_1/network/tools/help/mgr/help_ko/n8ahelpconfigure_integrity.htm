데이터 무결성 구성

데이터 무결성 검사는 메시지의 내용에 준하여 데이터 양을 계산하는 프로세스로서 메시지 발신자가 체크섬을 계산합니다. 체크섬 값은 메시지 패킷의 일부로 포함됩니다. 메시지 수신자는 메시지 수신 즉시 체크섬을 다시 계산합니다. 메시지가 전송되는 동안 변경되지 않은 경우 두 개의 체크섬 값은 일치해야 합니다.

데이터 무결성을 구성하려면 다음과 같이 하십시오.

  1. 네비게이터 창에서 Oracle Net 구성 > 로컬 > 프로파일을 확장합니다.

  2. 오른쪽 창에 있는 목록에서 Oracle Advanced Security를 선택합니다.

  3. 무결성 탭을 선택합니다.

  4. 클라이언트 또는 [서버] 체크 박스를 선택하여 클라이언트나 서버의 데이터 무결성을 설정합니다.

  5. 클라이언트와 서버의 데이터 무결성 조정 방법을 지정합니다. 클라이언트와 서버 모두에 대해 체크섬 레벨을 선택해야 합니다. 다음은 각 레벨에 대한 설명입니다.

    요소 설명
    승인됨 상대편 접속이 "필요함" 또는 "요청됨"을 지정하고 사용 가능한 호환 알고리즘이 있으면 서비스를 사용할 수 있습니다. 그렇지 않으면 서비스를 사용할 수 없습니다.
    거부됨 서비스를 사용할 수 없으며 상대편 접속이 "필요함"을 지정해도 접속에 실패합니다.
    요청됨 상대편 접속이 "승인됨", "필요함", "요청됨" 중 하나를 지정하고 사용 가능한 호환 알고리즘이 있으면 서비스를 사용할 수 있습니다. 그렇지 않으면 서비스를 사용할 수 없습니다.
    필요함 서비스를 사용할 수 있으며, 상대편 접속이 "거부됨"을 지정하거나 호환 알고리즘이 없으면 접속에 실패하게 됩니다.

  6. 파일 > 네트워크 구성 저장을 누릅니다.
