Visão Geral da Configuração do Oracle Net Services

O Oracle Net Manager permite a configuração de diversos elementos de rede, incluindo métodos de nomeação, perfis de cliente e servidor e listeners.

Métodos de Nomeação

Defina nomes simples para identificar a localização de um serviço, com um banco de dados. Esses nomes simples, chamados de identificadores de conexão, são mapeados para descritores de conexão, que contêm a localização da rede e a identificação do serviço.

Com o Oracle Net Manager, você pode configurar várias maneiras pelas quais os identificadores de conexão são resolvidos em descritores de conexão através de métodos de nomeação.

Perfis de Cliente e Servidor

Defina as preferências para ativar e configurar os recursos do Oracle Net em clientes e servidores por meio de perfis. Com os perfis, você pode definir preferências que priorizam os métodos de nomeação, ativar os recursos do Oracle Advanced Security, ativar o log e rastreamento ou rotear conexões.

Configuração de Listener no Servidor

Configure o listener para receber conexões clientes. O listener deverá ser configurado com:

  • Um ou mais endereços de protocolo para listening.

  • Informações de serviço sobre o serviço de destino

Os endereços de protocolo são configurados no arquivo de configuração de listener, o listener.ora. As informações sobre o serviço poderão ou não ser configuradas no arquivo listener.ora:

  • Um banco de dados Oracle8i ou posterior registra automaticamente determinadas informações no listener, como nomes de serviço, nomes de instância, handlers de serviço e informações sobre carga. Esse recurso, denominado registro de serviço, não exige configuração no arquivo listener.ora.

  • Outros serviços, inclusive de um banco de dados Oracle7 ou Oracle8, requerem a configuração do serviço no arquivo listener.ora.
