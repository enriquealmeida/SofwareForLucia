ディレクトリ・サーバーへのtnsnames.oraファイル・エントリのエクスポート

tnsnames.oraファイル内にすでに構成されているネット・サービス名は、ディレクトリ・サーバーにエクスポートできます。エクスポートは、一度に1つのドメインに対して実行されます。

tnsnames.oraファイルからエントリをエクスポートすると、ディレクトリ内で現在選択しているOracleコンテキスト内にネット・サービス名が作成されます。Oracleコンテキストを変更する場合は、「ツール」→「ディレクトリ・サーバー」→「Oracleコンテキストの変更」を選択します。

tnsnames.oraファイルに格納されているネット・サービス名をディレクトリ・サーバーにエクスポートする手順は、次のとおりです。

  1. 「ツール」→「ディレクトリ」→「サービス名のエクスポート」を選択します。ディレクトリ移行ウィザードが起動します。

  2. 「次へ」を選択します。

    複数ドメインが検出されると、「ドメインの選択」ページが表示されます。手順3に進みます。

    ネット・サービス名がドメインで修飾されていない場合、「ネット・サービス名の選択」ページが表示されます。手順4に進みます。

  3. エクスポートするネット・サービス名のネットワーク・ドメインを選択し、「次へ」を選択します。ネット・サービス名の選択ページが表示されます。

  4. エクスポートするネット・サービス名をリストから選択し、「次へ」を選択します。接続先コンテキストの選択ページが表示されます。

    「接続先コンテキストの選択」ページで、次の操作を実行します。

    a.「ディレクトリ・ネーミング・コンテキスト」リストから、Oracleコンテキストを含むネーミング・コンテキスト(naming context)を選択します。ディレクトリ・ネーミング・コンテキストは、Oracleエントリを含む1つ以上のOracleコンテキストを格納する、ディレクトリ・サブツリーの一部です。

    b.「Oracleコンテキスト」リストから、選択したネット・サービス名をエクスポートするOracleコンテキストを選択します。

    c.「次へ」を選択します。エクスポート操作のステータスとともにディレクトリ・サーバーの更新ページが表示されます。

  5. 「完了」を選択して「ディレクトリ・サーバー移行ウィザード」を終了します。
