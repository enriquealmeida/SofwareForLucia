Oracle Advanced Security : Chiffrement

L'onglet Cryptage vous permet de configurer les paramètres de cryptage des méthodes d'authentification sélectionnées.

Remarque : utilisez l'onglet SSL en vue de configurer les mécanismes de cryptage pour la couche SSL (Secure Sockets Layer).

Encryption

Dans la liste, sélectionnez l'ordinateur (client ou serveur) que vous configurez.

Type de cryptage

Sélectionnez une valeur pour indiquer le niveau de cryptage du client ou du serveur lors de la négociation du cryptage et de l'intégrité. Il existe quatre types de cryptage :

Elément Description
accepté Le service n'est actif que si l'autre machine participant à la connexion indique "requis" ou "demandé", et qu'elle dispose d'un algorithme compatible.
rejeté Le service ne doit pas être actif : la connexion échoue si l'autre machine indique "requis".
demandé Le service n'est actif que si l'autre machine participant à la connexion indique "accepté", "requis" ou "demandé", et qu'elle dispose d'un algorithme compatible.
requis Le service doit être actif et la connexion échoue si l'autre machine indique "rejeté" ou ne dispose pas d'un algorithme compatible.

EncryptionSeed

Dans le champ Valeurs de départ de cryptage, entrez une chaîne aléatoire de 256 caractères au maximum. Cette valeur de départ permet de générer des clés cryptographiques. Cela s'avère nécessaire lorsque l'intégrité ou le cryptage est activé.

Remarque : Si vous choisissez d'utiliser des caractères spéciaux tels que la virgule [,] ou la parenthèse fermante [)] dans Valeurs de départ de chiffrement, Oracle vous conseille de mettre la valeur du paramètre entre apostrophes.

Liste Méthodes disponibles/Méthodes sélectionnées

Affiche les méthodes d'authentification pouvant être sélectionnées et les méthodes de cryptage sélectionnées. L'ordre d'apparition des méthodes dans la liste détermine l'ordre des préférences pour la négociation.

Utilisez les boutons "flèche vers la gauche" (<) et "flèche vers la droite" (>) pour faire passer l'algorithme de cryptage sélectionné d'une liste à l'autre. Oracle Advanced Security prend en charge les algorithmes de cryptage suivants :

Elément Description
AES256 AES clé 256-bits
RC4_256 RC4 clé 256-bits
AES192 AES clé 192-bits
3DES168 DES triple avec option trois clés (168 bits)
AES128 AES clé 128-bits
RC4_128 RC4 clé 128-bits
3DES112 DES triple avec option deux clés (112 bits)
DES DES clé 56-bits
RC4_40 RC4 clé 40-bits
DES40 DES clé 40-bits
