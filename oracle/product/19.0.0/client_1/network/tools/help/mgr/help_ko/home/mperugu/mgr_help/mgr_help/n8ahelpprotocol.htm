프로토콜

데이터가 네트워크를 통해 전송되는 방법을 정의하는 규칙입니다. Oracle Net Manager를 통해 Oracle에서 지원하는 프로토콜은 다음과 같습니다.

IPC

IPC(프로세스 간 통신) 프로토콜은 동일한 컴퓨터에 리스너로 존재하는 클라이언트 응용 프로그램에 대해 사용됩니다. IPC는 TCP/IP보다 빠른 로컬 접속을 제공할 수 있습니다.

Named Pipes

Named Pipes는 TCP/IP를 사용하지 않는 PC에서 사용됩니다. Named Pipes는 Windows NT에서만 사용할 수 있습니다.

TCP/IP

TCP/IP(전송 제어 프로토콜/인터넷 프로토콜)는 네트워크에서 클라이언트/데이터베이스 서버 대화를 위해 사용되는 표준 이더넷 프로토콜입니다. TCP/IP를 사용하여 클라이언트의 오라클 응용 프로그램은 TCP/IP를 통해 원격 Oracle 데이터베이스와 통신할 수 있습니다(Oracle 데이터베이스가 TCP/IP를 사용하여 네트워크 통신을 지원하는 호스트 시스템에서 실행되는 경우).

TCP/IP는 표준 네트워크 프로토콜로 UNIX와 PC 기반 네트워크에서 많이 사용되며 인터넷 접속에도 사용합니다. 인터넷 접속에도 사용합니다.

SSL을 사용하는 TCP/IP

SSL(Secure Sockets Layer) 프로토콜을 사용하는 TCP/IP를 사용하여 클라이언트 상의 오라클 응용 프로그램은 TCP/IP 및 SSL을 통해 원격 오라클 데이터베이스와 통신할 수 있습니다. (오라클 데이터베이스가 TCP/IP 및 SSL을 사용하여 네트워크 통신을 지원하는 호스트 시스템에서 실행되는 경우)SSL을 사용하는 TCP/IP는 SSL 인증 방법 전용으로 사용됩니다.

주: 해당 운영 체제가 추가 프로토콜을 지원할 수 있습니다.
