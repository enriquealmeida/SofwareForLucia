Creación de un Alias de Servicio de Red

Notas:

  • Sólo los usuarios que son miembros de los grupos OracleNetAdmins u OracleContextAdmins pueden crear entradas de alias de servicio de red en un directorio.

  • Los alias de servicio de red no están soportados si utiliza Microsoft Active Directory.

  • Puede exportar los objetos de alias existentes desde Oracle Names Server. Consulte el capítulo sobre la exportación de datos de nomenclatura a un servidor de directorios, en Oracle10i Net Services Administrator's Guide (Guía del Administrador de Servicios de Red de Oracle10i) para obtener más información.

Puede crear los alias de servicio de red para los objetos de nomenclatura de directorios (nombres de servicios de red o servicios de bases de datos) en un servidor de directorios. Un alias de servicio de red almacena el nombre del objeto al que hace referencia.

Cuando un cliente solicita una consulta de directorio de un alias de servicio de red, el directorio determina que la entrada es un alias de servicio de red y realiza la consulta como si se tratara de la entrada a la que se hace referencia. Por ejemplo, se puede crear un alias de servicio de red salesalias para un servicio de red de nombre sales. Si salesalias se utiliza para conectar a un servicio de base de datos, como en CONNECT scott/tiger@salesalias, se resolverá y utilizará la información del descriptor de conexión para sales.

Hay dos usos principales de los alias de servicio de red:

  • Para que los clientes hagan referencia a un nombre de servicio de red o servicio de base de datos con otro nombre.

  • Utilizar un nombre de servicio de red en un Oracle Context para un nombre de servicio de red o servicio de base de datos en otro Oracle Context. De esta forma, el nombre de servicio de red o servicio de base de datos sólo se debe definir una vez en el servidor de directorios, pero se puede utilizar fuera de otros Oracle Contexts.

Para crear un alias de servicio de red:

  1. En el panel del navegador, amplíe Configuración de Red de Oracle > Directorio > Nomenclatura de Servicios > Alias.

  2. Haga clic en el signo más (+) en la barra de herramientas o seleccione Editar > Crear de la barra de menús. Aparece el recuadro de diálogo Crear Alias de Servicio de Red.

  3. En el campo Alias de Servicio de Red, introduzca un nombre para el alias de servicio de red.

  4. En el recuadro Nombre de Servicio de Red o Servicio de Base de Datos, introduzca la siguiente información sobre el nombre de servicio de red o servicio de base de datos:

    • Oracle Context: Especifique el Oracle Context del servicio de base de datos o del nombre del servicio de red seleccionando uno de la lista o introduciendo uno en el campo.

    • Nombre: Especifique el nombre distinguido (DN) del servicio de bases de datos o del nombre del servicio de red seleccionando uno de la lista o introduciendo uno en el campo.

  5. Seleccione Crear.
