Link de Banco de Dados

Um ponteiro que define um caminho de comunicação em uma direção de um servidor de banco de dados Oracle para outro servidor de banco de dados. O ponteiro do link é, na verdade, definido como uma entrada em uma tabela de dicionário de dados. Para acessar o link, você precisa estar conectado ao banco de dados local que contém a entrada de dicionário de dados.

Uma conexão de link de banco de dados é em uma direção pois um cliente conectado ao banco de dados local A pode usar um link armazenado no banco de dados A para acessar informações no banco de dados remoto B, mas usuários conectados ao banco de dados B não podem usar o mesmo link para acessar dados no banco de dados A. Se usuários locais do banco de dados B desejarem acessar dados no banco de dados A, eles precisarão definir um link que esteja armazenado no dicionário de dados do banco de dados B.

Estes tipos de link de banco de dados são suportados:

  • Um link de banco de dados particular em um esquema específico de um banco de dados. Apenas o proprietário de um link de banco de dados particular pode usá-lo.

  • Um link público para um banco de dados. Todos os usuários do banco de dados podem usar esse link.

Para obter mais informações sobre como criar links do banco de dados privado e público, consulte o Oracle10i Database Administrator's Guide.
