Oracle Advanced Security: SSL

El separador SSL permite modificar la configuración de SSL (Secure Sockets Layer). SSL es un protocolo estándar para asegurar las comunicaciones por red. SSL proporciona autenticación, cifrado e integridad de datos. Utilice la función SSL para asegurar las comunicaciones entre cualquier cliente y servidor. En concreto, puede utilizar SSL para autenticar cualquier cliente o servidor ante uno o más servidores Oracle o para autenticar un servidor Oracle ante un cliente.

Configuración de SSL

En la lista, seleccione para especificar los valores para el cliente o el servidor.

Los valores que debe configurar para el servidor son similares a los definidos para el cliente. Hay un parámetro adicional: Una casilla de control denominada Requerir Autenticación de Cliente.

Comprobación de Revocación (Sólo Servidor)

Especifique una comprobación de revocación para un certificado. Seleccione uno de los siguientes valores:

  • Ninguno: Selecciónelo para desactivar la comprobación de revocación de certificados.

  • Necesario: seleccione esta opción para realizar la revocación de certificados cuando un certificado esté disponible. Si se revoca un certificado y no se ha encontrado ninguna lista de revocación de certificados (CRL) adecuada, rechace la conexión SSL. Si no se ha encontrado ninguna CRL adecuada para averiguar el estado de la revocación del certificado y éste no se ha revocado, acepte la conexión SSL.

  • Solicitado: Selecciónelo para realizar la revocación de certificados cuando esté disponible una CRL. Rechace la conexión SSL si se revoca el certificado. Si no se ha encontrado ninguna CRL adecuada para determinar el estado de la revocación del certificado y éste no se revoca, acepte la conexión SSL.

Requerir Versión SSL (opcional):

En la lista seleccione la versión de SSL. El cliente y el servidor deben utilizar versiones compatibles. Puede seleccionar SSL v3.0 o permitir que se utilice una versión existente o futura de SSL.

Requerir Autenticación de Cliente: (Sólo Servidor)

Esta casilla de control se activa por defecto. Desactívela si no desea solicitar la autenticación del cliente.

Coincidir con Nombre X.509 de Servidor (Sólo Cliente)

De la lista, seleccione si desea ver si el nombre distinguido (DN) del servidor coincide con su nombre de servicio. Si fuerza las verificaciones de coincidencia, SSL asegura que el certificado es del servidor. Si decide no forzarla, SSL realiza la comprobación pero permite la conexión, independientemente de si hay una coincidencia. Si no se fuerza la coincidencia, puede que el servidor falsee su identidad. Seleccione uno de los siguientes valores:

  • Sí: Seleccione esta opción para comprobar el DN del servidor. Si el DN coincide con el nombre de servicio, la conexión es correcta. Si el DN no coincide, la conexión es correcta, pero se registra un error en el archivo sqlnet.log .

  • No: Seleccione esta opción para no comprobar el DN del servidor. Si ignora esta comprobación puede que el servidor falsee su identidad.
