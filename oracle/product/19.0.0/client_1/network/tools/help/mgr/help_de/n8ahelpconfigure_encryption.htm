Verschlüsselung konfigurieren

Bei der Verschlüsselung wird eine Meldung so verschlüsselt, dass nur der beabsichtigte Empfänger sie lesen kann.

So konfigurieren Sie Verschlüsselungsparameter:

  1. Blenden Sie Oracle Net-Konfiguration > Lokal > Profil im Navigator-Fensterbereich ein.

  2. Wählen Sie Oracle Advanced Security aus der Liste im rechten Fensterbereich.

  3. Wählen Sie das Register Verschlüsselung.

  4. Wählen Sie Client oder Server aus der Liste Verschlüsselung.

  5. Wählen Sie einen Wert aus der Liste Verschlüsselungstyp, um das Client- oder Server-Verhalten bei der Verhandlung von Verschlüsselung und Integrität anzugeben. Die vier möglichen Werte sind:

    Element Beschreibung
    Akzeptiert Der Dienst ist aktiv, wenn die andere Seite der Verbindung "obligatorisch" oder "angefordert" angibt und ein kompatibler Algorithmus auf der anderen Seite verfügbar ist. Sonst ist der Dienst inaktiv.
    Zurückgewiesen Der Dienst muss nicht aktiv sein, und eine Verbindung kann nicht erfolgreich aufgebaut werden, wenn die andere Seite "obligatorisch" angibt.
    Angefordert Der Dienst ist aktiv, wenn die andere Seite der Verbindung "akzeptiert", "obligatorisch" oder "angefordert" angibt und ein kompatibler Algorithmus auf der anderen Seite verfügbar ist.
    Obligatorisch Der Dienst muss aktiv sein, und die Verbindung kann nicht erfolgreich aufgebaut werden, wenn die andere Seite "zurückgewiesen" angibt oder wenn kein kompatibler Algorithmus auf der anderen Seite vorhanden ist.

  6. Geben Sie ein Verschlüsselungsschema in das Feld "Verschlüsselungsschema" ein: eine beliebige Zeichenfolge mit maximal 256 Zeichen. Dieses Verschlüsselungsschema wird mit dem Verschlüsselungsalgorithmus verwendet, den Sie im nächsten Schritt auswählen.

  7. Aus der Liste "Verfügbare Methoden" wählen Sie einen Verschlüsselungsalgorithmus und wählen dann die Schaltfläche mit dem Pfeil nach rechts (<), um ihn in die Liste "Ausgewählte Methoden" zu verschieben. Die Reihenfolge der Methoden in der Liste "Ausgewählte Methoden" bestimmt die bevorzugte Reihenfolge für die Negotiation. Die folgenden Algorithmen werden von Oracle Advanced Security unterstützt.

    Element Beschreibung
    RC4_256 256-Bit-Schlüsselgröße
    3DES168 Triple DES mit einer Drei-Schlüssel-(168-Bit-)Option
    RC4_128 128-Bit-Schlüsselgröße
    3DES112 Triple DES mit einer Zwei-Schlüssel-(112-Bit-)Option
    RC4_56 56-Bit-Schlüsselgröße
    DES Standardmäßige 56-Bit-Schlüsselgröße
    RC4_40 40-Bit-Schlüsselgröße
    DES40 40-Bit-Schlüsselgröße

  8. Wählen Sie Datei >Netzwerkkonfiguration speichern.

Hinweis: Alle Verschlüsselungsalgorithmen und Schlüssellängen von Oracle Advanced Security sind nur für den US-Gebrauch verfügbar.
