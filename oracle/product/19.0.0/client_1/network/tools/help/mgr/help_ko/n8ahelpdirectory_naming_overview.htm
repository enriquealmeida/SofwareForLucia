디렉토리 이름 지정 개요

디렉토리 이름 지정은 클라이언트가 데이터베이스에 접속할 때 사용하는 접속 기술자에 대해 접속 식별자를 분석하기 위해 사용하는 기본 이름 지정 방법 중의 하나입니다. 디렉토리 이름 지정은 LDAP 방식 디렉토리 서버에 접속 식별자와 해당 접속 기술자를 저장합니다. 접속 식별자는 Oracle 소프트웨어와 함께 사용될 항목을 포함하는 Oracle Context 하위 트리 아래에 저장됩니다.

일반 사용자는 다음 형식으로 접속 식별자를 포함하는 접속 문자열을 입력합니다.

CONNERCT username/password@connect_identifier

다음 예에서 sales가 디렉토리에서 찾는 접속 식별자입니다.

CONNECT scott/tiger@sales

접속 기술자가 다음 사항을 클라이언트로 반환합니다.

  • 리스너의 프로토콜 주소 위치

  • 데이터베이스 서비스 이름

클라이언트는 프로토콜 주소 정보를 사용하여 리스너에 연결합니다. 동일한 주소 정보로 구성된 리스너는 요청을 수신합니다. 리스너가 접속 기술자에 지정된 내용과 동일한 데이터베이스 서비스 요청을 수신하면 접속이 부여됩니다.

디렉토리 이름 지정을 사용하려면 Oracle Net Services 기능에 대한 디렉토리 서버를 구성한 다음 디렉토리에 액세스할 수 있도록 클라이언트를 구성합니다.

자세한 내용은 Oracle10i Net Administrator's Guide의 Configuration Management Concepts 장을 참조하십시오.
