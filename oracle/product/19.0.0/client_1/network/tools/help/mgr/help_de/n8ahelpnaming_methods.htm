Namensgebung: Methoden

Mit dem Register Methoden können Sie angeben, welche Benennungsmethoden dieser Client-Rechner verwenden soll, sowie deren Prioritätsfolge. Die Benennungsmethode am Anfang der Liste wird als erstes verwendet, um den Connect-Bezeichner, einen Net Service Name oder einen Datenbankdienst aufzulösen. Wenn der Name nicht aufgelöst werden kann, wird die nächste in der Liste angegebene Methode verwendet.

Während einer Verbindungsanforderung nimmt der Client Verbindung mit einer Benennungsmethode auf, um Connect-Bezeichner in einen Verbindungsdeskriptor aufzulösen. Danach leitet der Client die Anforderung an die Listener-Adresse weiter, die im Verbindungsdeskriptor angegeben wird.

Oracle Net unterstützt die folgenden Benennungsmethoden:

  • Lokale Benennung (TNSNAMES)

  • Verzeichnisbenennung (LDAP)

  • Hostbenennung(HOSTNAME)

  • Easy Connect-Benennung (EZCONNECT)

  • Network Information Service(NIS)

Standard-Domain

Geben Sie den Namen der Netzwerk-Domain ein, von der die meisten Clientanforderungen ausgehen. Dies könnte die Domain sein, in der der Client gespeichert ist, oder eine Domain, aus der der Client häufig Netzwerkdienste anfordert.

Nachdem der Domainname festgelegt wurde, wird er automatisch an jedem nicht vollständigen Namen angehängt.
