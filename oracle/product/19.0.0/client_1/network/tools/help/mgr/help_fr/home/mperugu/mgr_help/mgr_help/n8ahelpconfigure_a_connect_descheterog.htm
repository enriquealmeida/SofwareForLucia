Configuration d'un descripteur de connexion pour les services hétérogènes

Les services hétérogènes vous permettent d'accéder à un système autre qu'Oracle à partir du serveur Oracle. Pour initier la connexion à un système non Oracle, le serveur Oracle lance un processus agent via le processus d'écoute sur la passerelle. Pour que la base de données Oracle puisse se connecter à un système non Oracle, vous devez configurer le serveur avec un nom de service réseau.

Pour configurer le serveur pour les services hétérogènes :

  1. Assurez-vous que le processus d'écoute est configuré pour les services hétérogènes, tel que décrit à la rubrique Configuration des services hétérogènes pour le processus d'écoute.

  2. Créez dans le fichier tnsnames.ora un nom de service réseau permettant de connecter un serveur de base de données Oracle à un système non-Oracle. Assurez-vous que les informations du descripteur de connexion correspondent aux informations configurées pour le processus d'écoute.

  3. Sélectionnez le nom de service réseau dans le dossier Local > Résolution de noms de service.

  4. Dans la case Identification de service du panneau de droite, choisissez Options avancées. La boîte de dialogue Options de service avancées apparaît.

  5. Sélectionnez Utilisation pour les services hétérogènes, puis cliquez sur OK.

  6. Dans le panneau de droite, choisissez Appliquer.

  7. Choisissez Fichier > Enregistrer la configuration réseau.
