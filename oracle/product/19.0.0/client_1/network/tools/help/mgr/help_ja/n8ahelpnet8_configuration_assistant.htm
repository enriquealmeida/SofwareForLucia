Oracle Net Configuration Assistant

インストール後に、次のような基本的なネットワーク・コンポーネントを構成するためのツール。

  • リスナー名およびプロトコル・アドレス

  • クライアントが接続識別子の変換に使用するネーミング・メソッド

  • tnsnames.oraファイル内のネット・サービス名

  • ディレクトリ・サーバー使用

Oracleのインストレーション・ガイドに示すとおり、ソフトウェアをインストールした後、Oracle Net Configuration Assistantは自動的に実行されます。Oracle Net Configuration Assistantは、クライアントまたはデータベース・サーバーで使用可能です。

Oracle Net Configuration Assistantをスタンドアロン・モードで実行して、ネーミング・メソッドの使用方法、リスナー、tnsnames.oraファイルのネット・サービス名およびディレクトリ・サーバー使用を構成できます。

次の操作で、Oracle Net Configuration Assistantをスタンドアロン・モードで起動します。

  • UNIXでは、$ORACLE_HOME/binにあるnetmgrを実行します。

  • Windowsオペレーティング・システムでは、「スタート」→「プログラム」→「Oracle - HOME_NAME」→「Configuration and Migration Tools」→「Oracle Net Configuration Assistant」を選択します。
