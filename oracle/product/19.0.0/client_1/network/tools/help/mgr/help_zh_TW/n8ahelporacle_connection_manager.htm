Oracle 連線管理程式

一種路由器，從屬端連線要求可以透過它來傳送至其下一個躍點，或直接傳送至資料庫伺服器。透過「Oracle 連線管理程式」傳遞連線要求的從屬端，可以使用「Oracle 連線管理程式」上設定的存取控制、多工以及協定轉換功能。

「Oracle 連線管理程式」可針對下列情況的環境，提供連線路由：

  • 有大量使用者需要透過單一協定存取單一服務。

  • 從屬端和伺服器上安裝的協定不同，造成典型的連線失敗。

  • 典型的 Oracle Net 連線需要從屬端和伺服器安裝相同協定。

  • 需要控制 TCP/IP 環境中指定之伺服器的從屬端存取。

如需有關「Oracle 連線管理程式」的資訊，請參閱 Oracle9i Net Services Administrator's Guide 中的 Networking Challenges in the Internet Age 章節。
