Lien de base de données

Pointeur définissant un chemin de communication unidirectionnel entre un serveur de base de données Oracle et un autre serveur de base de données. Ce pointeur est en fait une entrée dans une table du dictionnaire de données. Pour y accéder, vous devez être connecté à la base de données locale contenant l'entrée du dictionnaire de données.

Une connexion par lien de base de données est unidirectionnelle en ce sens qu'un client connecté à la base locale A peut utiliser un lien qui y est stocké pour accéder aux données de la base distante B, mais qu'un utilisateur connecté à la base B ne peut pas utiliser ce même lien pour accéder aux données de la base A. Si les utilisateurs de la base B veulent accéder aux données de la base A, ils doivent définir un lien qui sera stocké dans le dictionnaire de données de la base B.

Les types de lien de base de données suivants sont pris en charge :

  • Lien de base de données privé dans un schéma spécifique d'une base de données. Seul le propriétaire du lien peut l'utiliser.

  • Lien de base de données public pour une base de données. Tous les utilisateurs de la base de données peuvent l'utiliser.

Pour plus d'informations sur la création de liens de base de données privés et publics, reportez-vous au Guide de l'administrateur Oracle10i Database.
