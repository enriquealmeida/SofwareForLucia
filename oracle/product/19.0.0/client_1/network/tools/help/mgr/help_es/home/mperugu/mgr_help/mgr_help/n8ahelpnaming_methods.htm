Nomenclatura: Métodos

El separador Métodos permite especificar los métodos de nomenclatura que va a utilizar esta computadora cliente, así como la prioridad. El método del principio de la lista se utiliza primero para resolver el identificador de conexión, un nombre de servicio de red o un servicio de base de datos. Si no se puede resolver el nombre, se utiliza el siguiente método especificado en la lista.

Durante una solicitud de conexión, el cliente intenta entrar en contacto con el primer método de nomenclatura enumerado para resolver los identificadores de conexión en un descriptor de conexión. La solicitud del cliente envía la solicitud a la dirección del listener especificada en el descriptor de conexión.

La Red de Oracle proporciona soporte para los siguientes métodos de nomenclatura:

  • Nomenclatura Local(TNSNAMES)

  • Nomenclatura de Directorios(LDAP)

  • Nomenclatura de Host(HOSTNAME)

  • Nomenclatura de Conexión Sencilla(EZCONNECT)

  • Servicio de Información de Red(NIS)

Lista Métodos Disponibles

Muestra los métodos de nomenclatura que se pueden seleccionar.

Lista Métodos Seleccionados

Muestra los métodos de nomenclatura seleccionados. El orden de los métodos de nomenclatura de la lista determina el orden en el que los prueba la Red de Oracle.

Botón <

Haga clic para mover un método de nomenclatura de la lista Métodos Disponibles a la lista Métodos Seleccionados.

Botón >

Haga clic para mover un método de nomenclatura de la lista Métodos Seleccionados a la lista Métodos Disponibles.

Botón Aumentar

Haga clic para subir un método de nomenclatura seleccionado en la lista Métodos Seleccionados. La Red de Oracle trata de utilizar el primer método para resolver un nombre de servicio de red o de servicio de base de datos. Si no puede utilizar el primer método, utiliza el segundo y así sucesivamente.

Botón Reducir

Haga clic para bajar un método de nomenclatura seleccionado en la lista Métodos Seleccionados.

Dominio por Defecto

Introduzca el nombre del dominio de red desde el que se realizarán la mayoría de las solicitudes del cliente. Puede ser el nombre de dominio en el que está situado el cliente o un dominio del que el cliente suele solicitar servicios de red.

Una vez especificado, el nombre del dominio se agrega automáticamente a todos los nombres no cualificados.
