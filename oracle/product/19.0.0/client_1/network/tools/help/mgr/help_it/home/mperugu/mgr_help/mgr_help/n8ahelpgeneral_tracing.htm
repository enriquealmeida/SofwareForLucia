Generale: Trace

La scheda Trace consente di impostare i parametri di trace per il computer. Se si tratta di un client, utilizzare i campi Informazioni client. Se si tratta di un server, utilizzare i campi Informazioni server.

Sul client, questa funzione consente di tenere trace delle operazioni eseguite dal client; sul server, consente di tenere trace delle operazioni eseguite dal server nel corso di una connessione di rete. Sul server, consente di tenere trace delle operazioni eseguite dal server nel corso di una connessione di rete.

Livello di trace

Dalla lista selezionare il livello di trace.

OFF: la funzione di trace non è abilitata. Off è l'impostazione predefinita.

USER: la funzione di trace viene impostata su un livello appropriato per gli utenti. Consente di identificare le condizioni di errore provocate dagli utenti.

ADMIN: la funzione di trace viene impostata su un livello appropriato per gli amministratori del database. Consente di identificare i problemi specifici dell'installazione.

SUPPORT: la funzione di trace viene impostata su un livello appropriato per il Supporto Oracle.

I file di trace possono raggiungere dimensioni notevoli. Si consiglia di disabilitare la funzione di trace quando non è in corso la risoluzione dei problemi della rete.

Directory di trace

Immettere la directory in cui verranno scritti i file di trace. Anche se i file di trace del client e del server contengono informazioni relative al client o al server, entrambi vengono scritti sul computer locale. Il percorso predefinito è $ORACLE_ HOME/network/trace sulle piattaforme UNIX e ORACLE_HOME\network\trace sulle piattaforme Windows.

File di trace

Immettere il nome del file di trace. Il nome file predefinito è sqlnet.trc per il client e svr_pid.trc per il server.

Nome file di trace univoco

Scegliere questa opzione per creare un nome univoco per ciascun file di trace. Per consentire la coesistenza di nomi diversi, al nome di ciascun file di trace viene aggiunto un identificativo di processo. Deselezionare questa opzione se non si desidera creare file di trace univoci e si preferisce memorizzare tutte le informazioni di trace nello stesso file.
