Oracle Advanced Security: Authentifizierung

Im Register Berechtigungsprüfung können Sie Berechtigungsprüfungsmethoden auswählen und deren Priorität festlegen. Als nächstes sehen Sie eine Liste mit Berechtigungsprüfungsmethoden, die von Oracle Advanced Security unterstützt werden.

KERBEROS5

Unterstützt sichere Single Sign-On-Funktionen in einer verteilten Umgebung mit einer Secret Key-Kryptographie. Kennwörter werden in einem zentralen Repository gespeichert, sodass Kennwörter aus dem Netzwerk nicht mehr gestohlen werden können. Kerberos ermöglicht außerdem die Berechtigungsprüfung von Datenbank-Links und die erweiterte PC-Sicherheit.

CYBERSAFE

Ein Kerberos-basierter Berechtigungsprüfungsserver, der eine sichere Berechtigungsprüfung, basierend auf Schlüsselverwaltung und Shared Secrets, bereitstellt. Der Challenger verfügt über eine Kennwortprüfung und Berechtigungsprüfung mit Token-Security-Cards.

NTS

Ermöglicht die Durchführung der Betriebssystem-Berechtigungsprüfung zwischen einem Client und einem Oracle-Server unter Windows NT. Die systemeigene Berechtigungsprüfungsmethode von Windows NT ermöglicht die Berechtigungsprüfung von Datenbankbenutzern über Windows NT. Dadurch können Client-Rechner sichere Verbindungen zu einer Oracle-Datenbank auf einem Windows NT-Server herstellen. Von einer sicheren Verbindung wird gesprochen, wenn ein Windows NT-Client-Benutzername auf einem Windows NT-Server über die systemeigene Berechtigungsprüfungsmethode von Windows NT abgerufen wird. Der Windows NT-Server lässt dann zu, dass der Benutzername die Datenbankaktionen auf dem Server durchführt.

RADIUS

Unterstützt Authentifizierungsmethoden, die mit dem RADIUS-Standard konform sind, einschließlich Token-Cards und Smart-Cards. Ein RADIUS-Server überträgt Informationen zwischen dem Oracle-Server und den angegebenen Authentifizierungs-Servern. Aus Sicht des Benutzers erfolgt der ganze Authentifizierungsprozess nahtlos und transparent.

Schaltfläche Vorziehen

Klicken Sie auf diese Schaltfläche, um eine ausgewählte Berechtigungsprüfungsmethode in der Liste "Ausgewählte Methoden" zu verschieben. Die Client-/Server-Verhandlung versucht, die Berechtigungsprüfungsmethoden in der Reihenfolge zu verwenden, in der sie aufgeführt wurden. Oracle Net verwendet zuerst die Berechtigungsprüfung am Anfang der Liste. Wenn die erste Methode nicht gefunden werden kann, wird die zweite Methode verwendet usw.

Schaltfläche Zurückstufen

Klicken Sie auf diese Schaltfläche, um eine ausgewählte Berechtigungsprüfungsmethode in der Liste "Ausgewählte Methoden" zu verschieben.

Hinweis: Wenn Sie das Protokoll TCP/IP with SSL konfiguriert haben, wird standardmäßig die Secure Sockets Layer (SSL) konfiguriert. Jede Berechtigungsprüfungsmethode, die Sie in diesem Register wählen, setzt die Berechtigungsprüfungsfunktionen von SSL außer Kraft. Unter Register SSL wird beschrieben, wie zusätzliche SSL-Optionen konfiguriert werden. Wenn Sie das Protokoll TCP/IP mit SSL verwenden, verwenden Sie keine der anderen Berechtigungsprüfungsmethoden: Die Liste "Ausgewählte Methoden" muss leer sein.

Weitere Informationen finden Sie in der Windows NT-Dokumentation oder im Oracle Advanced Security Administrator's Guide.
