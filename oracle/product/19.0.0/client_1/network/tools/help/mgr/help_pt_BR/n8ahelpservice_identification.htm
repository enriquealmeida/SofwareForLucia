Identificação do Serviço

Esta folha de propriedades permite exibir ou modificar as informações de rede para um nome do serviço de rede ou serviço de banco de dados selecionado.

Identificação do Serviço

Use a caixa Identificação de Serviço para exibir ou modificar informações sobre CONNECT_DATA.

Elemento Descrição
Nome do Serviço Digite um nome de serviço caso esteja configurando este cliente para estabelecer conexão com um serviço de banco de dados Oracle8 i ou de versão posterior.Observação: para um serviço de banco de dados armazenado em um servidor de diretórios, este campo só exibe o valor atual. Não serão permitidas modificações.
SID Exibe o SID (Identificador de Sistema Oracle) de um banco de dados Oracle8.
Tipo de Conexão Na lista, selecione se você deseja que este nome do serviço de rede use um servidor dedicado, servidor compartilhado ou qualquer um com o qual o banco de dados tenha sido configurado. Se o servidor compartilhado estiver configurado no arquivo de parâmetros de inicialização, selecione a opção de servidor dedicado para forçar o listener a iniciar um servidor dedicado para ignorar a configuração do servidor compartilhado. Se o servidor compartilhado estiver configurado no arquivo de parâmetros de inicialização e você quiser garantir que essa conexão sempre utilize um servidor compartilhado, selecione Servidor Compartilhado. Se o Pool de Conexões Residente do Banco de Dados estiver ativado no servidor, você poderá selecionar Servidor do Pool para obter uma conexão do pool.
Usar Identificação Compatível com Oracle8 Release 8.0 Selecione esta opção se estiver configurando um cliente para estabelecer conexão com um serviço de banco de dados anterior à release Oracle8i. Desmarque-a se estiver configurando para um serviço de banco de dados Oracle9i ou Oracle8i.
Avançado Escolha para configurar parâmetros CONNECT_DATA avançados.

Configuração de Endereço

Use a caixa Endereço para modificar, adicionar ou deletar um endereço de protocolo de listener.

Se já houver um endereço configurado, suas informações sobre protocolo serão exibidas nos campos.

Elemento Descrição
Protocolo Na lista, selecione o protocolo para o endereço de listener. Dependendo do protocolo escolhido, especifique informações nos campos apropriados.
Botão Mostrar Avançado/Ocultar Avançado Use a alternância Mostrar Avançado/Ocultar Avançado para exibir ou ocultar configurações para especificar o limite de espaço no buffer de E/S para o endereço.

Essas definições são suportadas pelos protocolos TCP/IP, TCP/IP com SSL e SDP.

  • Tamanho Total do Buffer para Envio: especifique, em bytes, o espaço no buffer para operações de envio das sessões.

  • Tamanho Total do Buffer para Recebimento: Especifique, em bytes, o espaço no buffer para operações de recebimento das sessões.

Observação: A especificação dessas configurações anula aquelas existentes em um arquivo sqlnet.ora do lado cliente.
Botão + Escolha para criar um novo endereço de protocolo. É exibida uma nova guia Endereço, na qual você pode selecionar o protocolo e especificar informações sobre protocolo nos campos. O protocolo escolhido deverá estar instalado no cliente e no servidor (no qual o listener reside).
Botão X Escolha para remover a guia Endereço selecionada e as respectivas informações de endereço.
< button Escolha essa opção para mover o protocolo para cima na ordem da lista.
botão > Escolha essa opção para mover o protocolo para baixo na ordem da lista.
Botão Avançado Escolha essa opção para configurar várias opções de endereço, como failover de tempo de conexão e balanceamento de carga do cliente.

Botão Aplicar

Escolha para aplicar as definições.

Botão Reverter

Escolha para restabelecer as definições com seus valores originais.

Para obter mais informações sobre como configurar o espaço de buffer de Entrada/Saída, consulte o capítulo Optimizing Performance no Oracle10i Net Administrator's Guide.
