Configuración de Integridad de Datos

La comprobación de la integridad de los datos es el proceso mediante el que se calcula una cantidad según el contenido de un mensaje. El remitente del mensaje calcula el total de control. El valor de total de control se incluye como parte del paquete del mensaje. El receptor del mensaje vuelve a calcular el total de control del mensaje al recibirlo. Para comprobar que el mensaje no se ha modificado durante el envío, los dos valores de total de control deben coincidir.

Para configurar la integridad de los datos:

  1. En el panel del navegador, amplíe Configuración de Red de Oracle > Local > Perfil.

  2. Seleccione Oracle Advanced Security de la lista del panel de la derecha.

  3. Seleccione el separador Integridad.

  4. Active la integridad de los datos del cliente o del servidor, activando para ello las casillas de control Cliente o Servidor respectivamente.

  5. Especifique el modo de negociación de la integridad de los datos entre el cliente y el servidor. Debe seleccionar Nivel de Total de Control para ambos. A continuación, se describen los niveles:

    Elemento Descripción
    aceptado El servicio estará activo si el otro lado de la conexión especifica "necesario" o "solicitado" y en el otro lado está disponible un algoritmo compatible. De lo contrario, estará inactivo.
    rechazado El servicio no debe estar activo y la conexión fracasará si el otro lado especifica "necesario".
    solicitado El servicio estará activo si el otro lado de la conexión especifica "aceptado", "necesario" o "solicitado" y en el otro lado está disponible un algoritmo compatible.
    necesario El servicio debe estar activo y la conexión fracasará si el otro lado especifica "rechazado" o si no existe un algoritmo compatible en el otro lado.

  6. Haga clic en Archivo > Guardar Configuración de Red.
