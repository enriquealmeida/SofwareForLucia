Présentation de la configuration d'un processus d'écoute

Les sections suivantes sont couvertes dans cette rubrique.

Configuration pendant l'installation

Au cours d'une installation Enterprise Edition ou Standard Edition sur le serveur de base de données, l'assistant Configuration Oracle Net configure un processus d'écoute appelé LISTENER , qui a une adresse de protocole d'écoute TCP/IP pour la base de données Oracle. Au cours d'une installation personnalisée, l'assistant Configuration Oracle Net vous invite à configurer les nom de processus d'écoute et adresse de protocole de votre choix.

En outre, quel que soit le type d'installation, une adresse de protocole d'écoute IPC est automatiquement configurée pour les appels de procédures externes.

L'assistant Configuration Oracle Net configure également automatiquement dans le fichier listener.ora les informations sur les services nécessaires pour les procédures externes.

Configuration à l'aide d'Oracle Net Manager

Oracle Net Manager vous permet d'effectuer les opérations suivantes :

  • Création d'un processus d'écoute

  • Configuration des informations de service pour les bases de données Oracle8 ou antérieures.

  • Configuration des informations de service pour des bases de données autres qu'Oracle, et notamment des procédures externes et des services hétérogènes

  • Activation des fonctions de trace et de journalisation

  • Activation de l'authentification des mots de passe pour l'utilitaire LSNRCTL (Listener Control).

Pour accéder à la configuration d'un processus d'écoute :

Dans le panneau du navigateur, développez Oracle Net Configuration > Local > Processus d'écoute.

Pour plus d'informations, reportez-vous au chapitre Configuring and Administering the Listener dans le Guide de l'administrateur Oracle10i Net.
