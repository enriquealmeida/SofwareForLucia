目錄命名組態簡介

本主題的內容涵蓋「目錄命名」組態詳細資訊。

安裝期間的伺服器組態

自訂伺服器安裝期間，「Oracle Net 組態輔助程式」會提示您：

  • 選取目錄伺服器類型

  • 識別目錄位置

  • 選取或建立伺服器可以查詢、建立及修改連線 ID 的 Oracle Context

  • 設定目錄命名作為偏好的命名方法

如果正確設定目錄用法，「資料庫組態輔助程式」會嘗試連線至目錄並註冊目錄中的資料庫服務項目。資料庫服務項目設定有連線描述區，包含下列網路資訊：

  • 監聽器位址

  • 資料庫服務名稱

安裝期間的從屬端組態

所有從屬端安裝期間，「Oracle Net 組態輔助程式」會提示您設定下列目錄用法設定值：

  • 指定目錄的類型

  • 識別目錄位置

  • 選取從屬端可以查詢連線 ID 的預設 Oracle Context

  • 設定目錄命名作為偏好使用的命名方法

安裝後使用「Oracle Net 組態輔助程式」進行設定

「Oracle Net 組態輔助程式」也可以在獨立模式下執行來：

  • 設定從屬端或伺服器 (如先前所述)

  • 建立 Oracle 綱要

  • 建立 Oracle Context

在獨立模式執行「Oracle Net 組態輔助程式」：

  • 如果是在 UNIX，請執行 $ORACLE_HOME/bin 中的 netca。

  • 如果是在 Windows NT，請選擇開始 > 程式集 > Oracle - HOME_NAME > 網路管理 > Oracle Net 組態輔助程式。

使用 Oracle Net Manager 進行設定

您可以使用 Oracle Net Manager 來執行下列動作：

  • 變更 Oracle Context

  • 變更認證證明資料

  • 建立網路服務名稱

  • 建立網路服務別名

  • 設定透過多個協定位址連線監聽器

  • 設定資料庫服務或網路服務名稱的進階 CONNECT_DATA 資訊

  • 將 tnsnames.ora 檔案中的網路服務名稱項目移轉至目錄伺服器

若要存取目錄命名組態，請從瀏覽窗格，選擇目錄服務命名。

如需詳細資訊，請參閱「Oracle Net 組態輔助程式」說明系統。
