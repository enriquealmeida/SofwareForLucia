listener.ora 파일

다음 사항을 지정하는 리스너 구성 파일입니다.

  • 리스너 이름

  • 접속 요청을 허용하는 프로토콜 주소

  • 수신하는 서비스(데이터베이스 서비스 또는 데이터베이스 서비스 이외의 서비스)

listener.ora 파일은 다음 위치 중 하나에 있어야 합니다.

  1. TNS_ADMIN 환경 변수에 의해 지정된 디렉토리입니다. TNS_ADMIN 환경 변수가 Windows NT에서 변수로 정의되지 않은 경우 레지스트리에 있을 수 있습니다.

  2. 노드의 전역 구성 디렉토리입니다. Sun Solaris의 경우 이 디렉토리는 /var/opt/oracle입니다. Windows NT에는 중앙 디렉토리가 없습니다.

  3. UNIX의 $ORACLE_HOME/network/admin 디렉토리 또는 Windows 운영 체제의 ORACLE_HOME\network\admin 디렉토리입니다.

주: Oracle9i 및 Oracle8i 데이터베이스인 경우 서비스 등록 기능이 있으므로 데이터베이스 서비스 식별이 필요하지 않습니다. 그러나 Oracle Enterprise Manager를 사용하여 데이터베이스를 모니터링하려는 경우 정적 서비스 구성이 필요합니다.
