Criando um Nome de Serviço de Rede

Observação: Somente usuários membros do grupo OracleNetAdmins ou OracleContextAdmins podem criar entradas de nome do serviço de rede em um diretório.

Para adicionar nomes de serviço de rede a um arquivo tnsnames.ora ou a um servidor de diretórios:

  1. No painel do navegador, expanda Configuração do Oracle Net > Diretório ou Local > Nomeação de Serviço.

  2. Escolha o sinal de adição (+) na barra de ferramentas ou escolha Editar > Criar na barra de menus. A página de boas-vindas do Assistente de Nome do Serviço de Rede será exibida.

  3. Informe qualquer nome no campo Nome do Serviço de Rede. O nome do serviço de rede pode ser qualificado com o domínio do cliente. O nome do serviço de rede será automaticamente qualificado pelo domínio se o domínio do cliente for especificado.

  4. Escolha Próximo. A página Protocolo será exibida.

  5. Selecione o protocolo no qual o listener está configurado para atender. Observe que esse protocolo também deve estar instalado no cliente.

  6. Escolha Próximo. A página Definições de Protocolo é exibida.

  7. Nos campos fornecidos, especifique os parâmetros de protocolo apropriados para o protocolo escolhido.

  8. Escolha Próximo. A página Serviço será exibida.

  9. Selecione uma release, informe um serviço de destino e, como opção, selecione um tipo de conexão de banco de dados na lista Tipos de Conexões do Oracle8i ou Versões Mais Recentes.

    Se o serviço de destino for Oracle8i ou versão mais recente, selecione Oracle8i ou versão mais recente, informe um nome de serviço no campo Nome do Serviço e escolha Próximo. Se o serviço de destino for o Oracle8 ou o Oracle7, selecione Oracle8i ou Anterior, informe um Oracle System Identifier no campo SID do Banco de Dados e escolha Próximo.

    A Oracle Corporation recomenda que você use a definição default do Default do Banco de Dados para o tipo de conexão. Se o servidor compartilhado estiver configurado no arquivo de parâmetros de inicialização, você poderá selecionar Servidor Dedicado para forçar o listener a criar um servidor dedicado, ignorando a configuração do servidor compartilhado. Se o Pool de Conexões Residente do Banco de Dados estiver ativado no servidor, você poderá selecionar Servidor do Pool para obter uma conexão do pool. Se o servidor compartilhado estiver configurado no arquivo de parâmetros de configuração e você quiser garantir que a conexão sempre use o servidor compartilhado, selecione Servidor Compartilhado.

  10. Escolha Próximo. A página Teste é exibida.

  11. Escolha Testar para verificar se o nome do serviço de rede funciona, ou escolha Finalizar para salvar sua configuração e fechar o Assistente de Nome do Serviço de Rede.

    Se você escolher Testar, o Oracle Net estabelecerá conexão com o servidor de banco de dados utilizando as informações do descritor de conexão que você configurou. Portanto, o banco de dados e o listener devem estar em execução para que o teste seja realizado com sucesso. Se não estiverem, consulte o Capítulo Establishing a Connection and Testing the Network, no Oracle10i Net Administrator's Guide, para iniciar os componentes. Um teste bem-sucedido resulta na seguinte mensagem:

    Teste de conexão bem-sucedido.

    Se o teste for bem-sucedido, clique em Fechar para sair da caixa de diálogo Teste de Conexão e, em seguida, continue com a Etapa 12.

    Se o teste não for bem-sucedido:

    1. Verifique se o banco de dados e o listener estão sendo executados e escolha Testar.

    2. Escolha Alterar Log-in para alterar o nome de usuário e a senha para a conexão e, em seguida, escolha Testar.

  12. Escolha Finalizar para salvar sua configuração e fechar o Assistente de Nome do Serviço de Rede.

  13. Verifique se o perfil reflete o local (LOCAL) ou o método de nomeação de diretório (LDAP).
