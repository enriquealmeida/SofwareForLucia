Général : Routage

L'onglet Routage vous permet de diriger en permanence les demandes de connexion émanant de cet ordinateur vers un processus spécifique.

Toujours utiliser le serveur dédié

Sélectionnez cette option pour obliger le processus d'écoute à générer dynamiquement un serveur dédié pour toutes les sessions réseau de ce client. Le processus d'écoute effectue cette opération en ajoutant (SERVER=dedicated) à la partie CONNECT_DATA des descripteurs de connexion. Les connexions en provenance de ce client utiliseront des serveurs dédiés même si la configuration a défini un serveur partagé.

Cette option accélère l'établissement des connexions, mais ralentit le serveur. Oracle conseille d'utiliser cette option si vous effectuez des transactions intensives sur la base de données, avec très peu de temps d'inactivité.

Remarque : Si elle est activée, cette option remplace l'option Type de connexion dans la feuille de propriétés Identification du service.

Utiliser les adresses du gestionnaire de connexions

Choisissez cette option pour envoyer le client, chaque fois que possible, vers Oracle Connection Manager. Si aucune adresse d'Oracle Connection Manager n'est disponible, les connexions sont alors acheminées via n'importe quelle adresse de processus d'écoute.
