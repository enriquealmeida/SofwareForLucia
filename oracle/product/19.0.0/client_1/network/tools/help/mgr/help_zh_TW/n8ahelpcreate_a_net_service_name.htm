建立網路服務名稱

注意：只有身為 OracleNetAdmins 或 OracleContextAdmins 群組成員的使用者，才可以在目錄中建立網路服務名稱項目。

在 tnsnames.ora 檔案或目錄伺服器新增網路服務名稱：

  1. 在瀏覽窗格中，展開 Oracle net 組態 > 目錄或區域 > 服務命名。

  2. 選擇工具列中的加號 (+)，或者從功能表列選擇編輯 > 建立。「網路服務名稱精靈」的「歡迎」頁面就會顯示。

  3. 在網路服務名稱欄位輸入任何名稱。網路服務名稱可以加上從屬端的網域名稱。如果指定從屬端的網域，網路服務名稱會自動加上網域名稱。

  4. 選擇下一步。「協定」頁面就會顯示。

  5. 選取監聽器要監聽的協定。請注意，此協定必須也安裝在從屬端。

  6. 選擇下一步。「協定設定值」頁面就會顯示。

  7. 在出現的欄位中輸入選取之協定的適當協定參數。

  8. 選擇「下一步」。「服務」頁面就會顯示。

  9. 選取版本，輸入目的地服務，並 (或) 從 Oracle8i 或更新版本的「連線類型」清單選取資料庫連線類型。

    如果目的地服務是 Oracle8i 或更新的版本，請選取 Oracle8i 或更新的版本，在服務名稱欄位中輸入服務名稱，然後選擇下一步。如果目的地服務是 Oracle8 或 Oracle7，請選取 Oracle8 或先前的版本，在資料庫 SID 欄位中輸入 Oracle 系統 ID，然後選擇下一步。

    Oracle Corporation 建議您使用「資料庫預設值」的預設設定值作為連線類型。如果初始化參數檔中設定了共用伺服器，您可以選取專用伺服器來強制監聽器產生專用伺服器，而略過共用伺服器組態。如果伺服器上已經啟用「資料庫常駐連線集區」，您可以選取集區伺服器，從集區取得連線。如果初始化參數檔中有設定共用伺服器，而您想確保連線永遠使用共用伺服器，請選取共用伺服器。

  10. 選擇下一步。「測試」頁面就會顯示。

  11. 選擇測試來驗證網路服務名稱是否可用，或選擇「完成」來儲存組態並關閉「網路服務名稱精靈」。

    如果選擇「測試」，Oracle Net 會使用您設定的連線描述區連線至資料庫。因此，資料庫和監聽器必須在執行中，才能順利測試。如果沒有在執行中，請參閱 Oracle10i Net Administrator's Guide 中的 Establishing a Connection and Testing the Network 章節來啟動元件。測試成功時會產生下列訊息：

    連線測試成功。

    如果測試成功，請選擇關閉以關閉「連線測試」對話方塊，然後繼續到步驟 12。

    如果測試失敗：

    1. 確定資料庫與監聽器是否在執行，然後選擇測試。

    2. 選擇「變更登入」來變更連線的使用者名稱與密碼，然後選擇測試。

  12. 選擇完成儲存組態並關閉「網路服務名稱精靈」。

  13. 確定設定檔反映區域 (LOCAL) 或目錄命名 (LDAP) 方法。
