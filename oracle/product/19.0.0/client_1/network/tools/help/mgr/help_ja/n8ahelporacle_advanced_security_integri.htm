Oracle Advanced Security: 整合性

「整合性」タブを使用すると、認証方式のデータの整合性を構成できます。データの整合性とは、クライアントとサーバーの両方で、送られたデータを比較することです。

注意: Secure Sockets Layer (SSL)の暗号スイートを構成する場合は、「SSL」タブを使用してください。

整合性

「サーバー」または「クライアント」チェック・ボックスを選択し、現在構成しているコンピュータがクライアントであるかサーバーであるかを指定します。「サーバー」オプションは、このコンピュータの接続先サーバーには適用されません。

チェックサム・レベル

チェックサム・レベルを選択します。次の4つのチェックサム・レベルから選択できます。

要素 説明
適用 一方の接続側で「必要」または「要求」が指定されており、もう一方の接続側で矛盾のないアルゴリズムが使用可能になっている場合、サービスはアクティブになります。それ以外の場合、サービスはアクティブにはなりません。
拒否 サービスはアクティブにはなりません。一方の接続側で「必要」が指定されている場合、接続は失敗します。
要求 一方の接続側で「適用」、「必要」または「要求」が指定されており、矛盾のないアルゴリズムが使用可能になっている場合、サービスはアクティブになります。それ以外の場合、サービスはアクティブにはなりません。
必要 サービスはアクティブになります。一方の接続側で「拒否」が指定されている場合、または両立するアルゴリズムがない場合、接続は失敗します。

「使用可能なメソッド」リスト

チェックサムの作成に使用する整合性アルゴリズムが表示されます。次の2つの方法があります。

アルゴリズム名 有効な値
Secure Hash Algorithm (SHA-1) SHA1
Message Digest 5 (MD5) MD5

「選択メソッド」リスト

選択しているアルゴリズムが表示されます。リスト内のアルゴリズムの順序によって、使用するアルゴリズムの順序が決まります。
