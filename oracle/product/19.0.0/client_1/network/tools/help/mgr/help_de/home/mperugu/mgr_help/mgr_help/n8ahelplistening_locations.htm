Listening-Adressen

Listener sind so konfiguriert, dass sie auf ankommende Verbindungsanforderungen für eine Datenbank auf einer bestimmten Netzwerk-Protokolladresse hören. Mit dem Register Listening-Adressen konfigurieren Sie eine oder mehrere Listening-Adressen, auf denen ein Listener auf Clientverbindungen wartet.

Protokoll

Wählen Sie das Protokoll für die Adresse aus der Liste. Je nach ausgewähltem Protokoll geben Sie Protokollparameter-Informationen in die Felder ein.

Schaltfläche Erweiterte anzeigen/Erweiterte verbergen

Mit dem Schalter "Erweiterte anzeigen/Erweiterte verbergen" können Sie Einstellungen zur Angabe des Grenzwertes für den E/A-Pufferspeicher für den Listener anzeigen oder verbergen.

Diese Einstellungen werden von den Protokollen TCP/IP, TCP/IP with SSL und SDP unterstützt.

  • Gesamte Sende-Puffergröße

Geben Sie die Größe des Puffers in Byte für Sendevorgänge von Sessions an.

Weil der Datenbank-Server Daten in Clients schreibt, ist die Festlegung dieser Einstellung auf Server-Seite sinnvoll. Wenn der Datenbank-Server große Mengen von Anforderungen empfängt, legen Sie auch die Einstellung Gesamte Empfangs-Puffergröße fest.

  • Gesamte Empfangs-Puffergröße

Geben Sie die Größe des Puffers in Byte für Empfangsvorgänge von Sessions an.

Diese Adresse statisch für JServer-Verbindungen dedizieren

Legen Sie fest, dass die Datei listener.ora mit General Inter-Orb Protocol-(GIOP-)Endpunkten für Oracle JServer Release 8.1 Verbindungen konfiguriert werden soll.

Adresse hinzufügen

Wählen Sie diese Option, um eine neue Protokolladresse zu erstellen. Daraufhin wird ein neues Register Adresse angezeigt, in dem Sie das Protokoll auswählen und Protokollinformationen in die Felder eingeben können.

Adresse entfernen

Klicken Sie auf diese Option, um das ausgewählte Register Adresse und die Adressinformationen zu löschen.

Weitere Informationen zur Konfiguration des I/O-Pufferspeicherplatzes finden Sie im Kapitel Optimizing Performance im Oracle10i Net Administrator's Guide.
