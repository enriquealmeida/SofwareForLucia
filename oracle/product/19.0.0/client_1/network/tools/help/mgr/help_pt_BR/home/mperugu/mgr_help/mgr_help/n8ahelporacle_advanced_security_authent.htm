Oracle Advanced Security: Autenticação

A guia Autenticação permite que você selecione e priorize métodos de autenticação. A seguir está uma lista de métodos de autenticação suportados pelo Oracle Advanced Security.

KERBEROS5

Oferece suporte para recursos de sign-on simples e seguros em um ambiente distribuído com a criptografia de chave secreta. As senhas são armazenadas em um repositório central, tornando o roubo de senhas da rede impossível. O Kerberos também oferece autenticação de link de banco de dados e segurança aperfeiçoada para PCs.

CYBERSAFE

Um servidor de autenticação baseado no Kerberos que permite uma autenticação segura baseada em um gerenciamento chave e em segredos compartilhados. O Challenger possui autenticação e verificação de senha com cartões de segurança de token.

NTS

Permite que uma autenticação de sistema operacional seja executada entre um cliente e um servidor Oracle no Windows NT. O método de autenticação Nativa do Windows NT permite uma autenticação do usuário do banco de dados através do Windows NT. Isso permite que os computadores clientes estabeleçam conexões seguras com um banco de dados Oracle em um servidor Windows NT. Uma conexão segura é estabelecida quando um nome de usuário cliente do Windows NT é recuperado em um servidor Windows NT através do método de autenticação Nativo do Windows NT. Depois disso, o servidor Windows NT permite que o nome do usuário execute ações de banco de dados no servidor.

RADIUS

Suporta qualquer método de autenticação que obedeça ao padrão RADIUS, incluindo cartões de token e cartões inteligentes. Um servidor RADIUS passa informações entre o servidor Oracle e o(s) servidores de autenticação designados. Da perspectiva dos usuários, o processo de autenticação inteiro acontece de uma forma imperceptível.

botão <

Escolha para mover um método de autenticação selecionado na lista Métodos Disponíveis para a lista Métodos Selecionados.

botão >

Escolha para mover um método de autenticação selecionado da lista Métodos Selecionados para a lista Métodos Disponíveis.

Botão Promover

Escolha para mover um método de autenticação selecionado para cima na lista de Métodos Selecionados. A negociação Cliente/Servidor tentará usar os métodos de autenticação na ordem em que eles são listados. O Oracle Net utilizará primeiro a autenticação que estiver no topo da lista. Se não conseguir usar o primeiro método, ele utilizará o segundo método e assim por diante.

Botão Rebaixar

Escolha para mover um método de autenticação selecionado para baixo na lista Métodos Selecionados.

Observação: Se você tiver configurado o protocolo TCP/IP com SSL, a SSL (Secure Sockets Layer) será configurada por default. Qualquer método de autenticação escolhido nesta guia substituirá os recursos de autenticação da SSL. Use a guia SSL para configurar opções adicionais de SSL. Se você estiver usando o protocolo TCP/IP com SSL, não use nenhum outro método de autenticação. A lista Métodos Selecionados deverá estar vazia.

Para obter informações, consulte a documentação do Windows NT ou o Oracle Advanced Security Administrator's Guide.
