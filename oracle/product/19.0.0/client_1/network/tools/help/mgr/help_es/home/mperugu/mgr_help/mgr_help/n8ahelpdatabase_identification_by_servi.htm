Identificación de Bases de Datos por Nombres de Servicio

Una base de datos Oracle se representa para los clientes como un servicio, es decir, la base de datos realiza el trabajo en nombre de los clientes. Las bases de datos pueden tener uno o más servicios asociados.

Hasta Oracle8i, los servicios de base de datos Oracle se identificaban únicamente con un identificador del sistema Oracle (SID). El SID también lo utilizaba internamente la base de datos como puntero del Área Global del Sistema (SGA). Los clientes se conectaban a una instancia de la base de datos especificando el SID en el descriptor de conexión. Este esquema de nomenclatura no distinguía los servicios de las instancias.

Debido a que las bases de datos Oracle pueden abarcar varias computadoras, se especifican tanto el servicio en conjunto como cada una de sus instancias en Oracle9i y Oracle8i.

Nombre de Servicio

Las bases de datos se identifican mediante el nombre de servicio. El nombre de servicio se especifica con el parámetro SERVICE_NAMES en el archivo de parámetros de inicialización. SERVICE_NAMES especifica el nombre del nivel superior de la vista del servicio de base de datos Oracle, que puede abarcar instancias y/o nodos. Por defecto, se utiliza SERVICE_NAMES como nombre de la base de datos global, formado por el nombre de la base de datos y el nombre de dominio.

Nombre de Instancia

Las instancias de las bases de datos se identifican mediante un nombre de instancia con el parámetro INSTANCE_NAME en el archivo de parámetros de inicialización. INSTANCE_NAME corresponde al SID de la instancia.

Descriptores de Conexión

Configure los descriptores de conexión con el parámetro SERVICE_NAME (sin S) para conectarse a una base de datos Oracle9i u Oracle8i. Por ejemplo, el siguiente conector de descripción contiene la dirección de un listener ubicado en sales1-sun que recibe las solicitudes de conexión de un servicio de base de datos denominado sales.us.example.com:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)))

El descriptor de conexión también se puede configurar con el parámetro INSTANCE_NAME para conectarse a una instancia determinada de la base de datos. Esto puede que sea necesario si Oracle9i Real Application Clusters tiene varias instancias.

Por ejemplo, el siguiente descriptor de conexión contiene la dirección de un listener ubicado en sales-sun1 que atiende las solicitudes de conexión de una instancia denominada sales1 asociada a la base de datos sales.us.example.com:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)

(INSTANCE_NAME=sales1)))

Para obtener información sobre cómo especificar un nombre de instancia en la sección CONNECT_DATA de un conector de descripción, consulte el capítulo 13 sobre activación de las funciones avanzadas para servicios de red de Oracle, en Oracle10i Net Services Administrator's Guide (Guía del Administrador de Servicios de Red de Oracle10i).
