Configurazione dei banner per l'accesso non autorizzato e l'audit delle azioni utente

È necessario creare e configurare i banner per avvisare gli utenti dell'accesso non autorizzato e del possibile audit delle azioni utente. Gli avvisi sono disponibili per l'applicazione client quando viene eseguito il login al database.

Per configurare la visualizzazione di questi banner, impostare i seguenti parametri sqlnet.ora dal lato server del database in modo che puntino a un file di testo contenente le informazioni sul banner:

  • SEC_USER_UNAUTHORIZED_ACCESS_BANNER. Ad esempio: SEC_USER_UNAUTHORIZED_ACCESS_BANNER = /opt/Oracle/11g/dbs/unauthaccess.txt

  • SEC_USER_AUDIT_ACTION_BANNER. Ad esempio: SEC_USER_AUDIT_ACTION_BANNER = /opt/Oracle/11g/dbs/auditactions.txt

Per impostazione predefinita, questi parametri non sono impostati.

Dopo aver impostato questi parametri, l'applicazione Oracle Call Interface deve utilizzare le API OCI appropriate per recuperare questi banner e visualizzarli all'utente finale.
