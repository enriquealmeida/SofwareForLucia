Paramètres généraux : Général

L'onglet Général vous permet de configurer les paramètres généraux du processus d'écoute.

Nom du module d'écoute

Affiche le nom du processus d'écoute.

Durée d'attente avant le démarrage

Indiquez la durée, en secondes, pendant laquelle le processus d'écoute doit attendre avant de répondre à la commande START de l'utilitaire LSNRCTL (Listener Control).

Remarque : l'utilisation de ce paramètre est déconseillée. Il ne sera plus pris en charge dans les versions suivantes. Si vous avez besoin de ce paramètre pour exécuter le processus d'écoute, signalez-le au support technique Oracle.

Enregistrer la configuration à la fermeture

Choisissez cette option pour que les modifications apportées par la commande SET de LSNRCTL deviennent définitives.

Informations de contact SNMP

Entrez un texte qui sera écrit dans le fichier snmp.ora afin d'être utilisé par Oracle Enterprise Manager. Ces informations peuvent être une chaîne quelconque de caractères. Elles sont envoyées pour toute connexion SNMP . Il n'affecte pas le processus de création des connexions SNMP. En général, ces informations indiquent les contacts de l'utilisateur ou de l'administrateur du client.

ADMIN_RESTRICTIONS_listener_name

Restreint l'administration à l'exécution du processus d'écoute.

Le paramètre par défaut est off. La définition de ADMIN_RESTRICTIONS_listener_name=on désactive la modification à l'exécution des paramètres dans listener.ora. En d'autres termes, le processus d'écoute refuse d'accepter les commandes SET permettant de modifier ses paramètres. Pour modifier ces derniers dans listener.ora, y compris ADMIN_RESTRICTIONS_listener_name, modifiez le fichier listener.ora manuellement et rechargez ses paramètres (à l'aide de la commande RELOAD) pour que les nouvelles modifications soient appliquées sans arrêter et redémarrer explicitement le processus d'écoute.
