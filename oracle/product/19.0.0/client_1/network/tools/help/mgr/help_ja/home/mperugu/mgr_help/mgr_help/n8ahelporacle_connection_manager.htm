Oracle Connection Manager

クライアント接続リクエストを、次のホップ(ルーター)または直接データベース・サーバーに送信するルーター。Oracle Connection Managerを通して接続リクエストを送信するクライアントは、そのOracle Connection Managerに構成されているアクセス制御、多重化およびプロトコル変換機能などが利用できます。

Oracle Connection Managerは、次のような環境のための接続ルートを提供します。

  • 多数のユーザーが1つのプロトコルで1つのサービスにアクセスする必要がある場合

  • クライアントとサーバーに異なるプロトコルがインストールされており、標準的な接続が失敗する場合

  • 標準のOracle Net接続で、クライアントとサーバーに同じプロトコルをインストールする必要がある場合

  • TCP/IP環境で指定されたサーバーへのクライアント・アクセスを制御する必要がある場合

Oracle Connection Managerの詳細は、Oracle9i Net Services管理者ガイドの「インターネット時代の先進的ネットワーキング」の章を参照してください。
