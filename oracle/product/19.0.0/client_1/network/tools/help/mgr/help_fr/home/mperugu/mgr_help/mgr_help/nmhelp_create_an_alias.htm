Création d'un alias de service réseau

Remarques :

  • Seuls les utilisateurs faisant partie du groupe OracleNetAdmins ou du groupe OracleContextAdmins peuvent créer des entrées d'alias de service réseau dans un annuaire.

  • Les alias de service réseau ne sont pas pris en charge avec Microsoft Active Directory.

  • Vous pouvez exporter des objets d'alias existants à partir d'un serveur Oracle Names. Pour plus d'informations, reportez-vous au chapitre Exporting Naming Data to a Directory Server dans le Guide de l'administrateur Oracle10i Net Services.

Vous pouvez créer des alias de service réseau pour des objets de résolution de noms d'annuaire (noms de service réseau ou services de bases de données) sur un serveur d'annuaire. Un alias de service réseau stocke le nom de l'objet auquel il fait référence.

Lorsqu'un client demande une recherche d'alias de service réseau, l'annuaire détermine que l'entrée est un alias de service réseau et effectue la recherche comme s'il s'agissait de l'entrée référencée. Par exemple, l'alias de service réseau salesalias peut être créé pour le nom de service réseau sales. Si salesalias est utilisé pour la connexion à un service de base de données, comme dans CONNECT scott/tiger@salesalias, il est converti et utilise les informations de descripteur de connexion de sales.

Il existe deux utilisations principales pour les alias de service réseau :

  • Les alias de service réseau permettent à des clients de faire référence à un service de base de données ou à un nom de service réseau sous un autre nom.

  • Utiliser un alias de nom de service réseau dans un contexte Oracle pour un service de base de données ou un nom de service réseau dans un contexte Oracle différent. Ceci permet de définir une fois un service de base de données ou un nom de service réseau sur le serveur d'annuaire, et de l'employer dans d'autres contextes Oracle.

Pour créer un alias de service réseau :

  1. Dans le panneau du navigateur, développez Oracle Net Configuration > Annuaire > Résolution de noms de service > Alias.

  2. Sélectionnez plus (+) dans la barre d'outils ou choisissez Modifier > Créer dans la barre de menus. La boîte de dialogue Créer un alias de service réseau apparaît.

  3. Dans le champ Alias de service réseau, entrez le nom de l'alias de service réseau.

  4. Dans la zone Nom de service réseau ou de service de base de données, saisissez les informations suivantes concernant le nom de service réseau ou de service de base de données :

    • Contexte Oracle : indiquez le contexte Oracle correspondant au nom de service réseau ou de service de base de données en le sélectionnant dans la liste ou en le saisissant dans ce champ.

    • Nom : indiquez le nom distinctif (DN) correspondant au nom de service réseau ou de service de base de données en le sélectionnant dans la liste ou en le saisissant dans ce champ.

  5. Cliquez sur Créer.
