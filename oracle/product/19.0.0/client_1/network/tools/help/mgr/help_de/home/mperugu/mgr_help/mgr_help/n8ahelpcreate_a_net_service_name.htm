Net Service-Name erstellen

Hinweis: Nur Benutzer, die Mitglieder der Gruppe OracleNetAdmins oder OracleContextAdmins sind, können Net Service-Name-Einträge in einem Directory erstellen.

So fügen Sie Net Service-Namen zu einer Datei tnsnames.ora oder einem Directory-Server hinzu:

  1. Blenden Sie im Navigator-Fensterbereich Oracle Net-Konfiguration > Verzeichnis oder Lokal >Dienstebenennung ein.

  2. Klicken Sie auf das Pluszeichen (+) in der Symbolleiste, oder wählen Sie Bearbeiten > Erstellen aus der Menüleiste. Daraufhin wird die Willkommen-Seite des Net Service-Name-Assistenten angezeigt.

  3. Geben Sie einen Namen in das Feld Net Service-Name ein. Der Net Service-Name kann mit der Client-Domain qualifiziert werden. Der Net Service-Name ist automatisch die Domain, wenn die Client-Domain angegeben wird.

  4. Klicken Sie auf Weiter. Daraufhin wird die Seite "Protokoll" angezeigt.

  5. Wählen Sie das Protokoll, auf das der Listener gemäß Konfiguration hören soll. Beachten Sie, dass dieses Protokoll auch auf dem Client installiert sein muss.

  6. Wählen Sie Weiter. Daraufhin wird die Seite "Protokolleinstellungen" angezeigt.

  7. Geben Sie die entsprechenden Protokollparameter für das ausgewählte Protokoll in die angegebenen Felder ein.

  8. Wählen Sie "Weiter". Daraufhin wird die Seite "Service" angezeigt.

  9. Wählen Sie ein Release, geben Sie einen Zielservice ein, und wählen Sie optional einen Datenbankverbindungstyp aus der Liste "Verbindungstyp Oracle8i oder höher".

    Wenn der Zielservice Oracle8i oder höher ist, wählen Sie "Oracle8i oder höher", geben einen Servicenamen in das Feld Servicename ein und wählen Weiter. Wenn der Zieldienst Oracle8 oder Oracle7 ist, wählen Sie Oracle8 oder früher, geben einen Oracle System Identifier in das Feld Datenbank-SID ein und wählen Weiter.

    Oracle Corporation empfiehlt Ihnen, die Standardeinstellung "Datenbank-Standardwert" für den Verbindungstyp zu verwenden. Wenn Shared Server in der Initialisierungsparameterdatei konfiguriert ist, können Sie Dedizierter Server wählen, damit der Listener einen dedizierten Server hochfährt und die Shared Server-Konfiguration umgeht. Wenn das datenbankresidente Verbindungspooling auf dem Server aktiviert ist, können Sie Im Pool befindlicher Server wählen, um eine Verbindung aus dem Pool abzurufen. Wenn Shared Server in der Initialisierungsparameterdatei konfiguriert ist und Sie sicherstellen möchten, dass die Verbindung immer einen Shared Server verwendet, wählen Sie Shared Server.

  10. Klicken Sie auf Weiter. Daraufhin wird die Seite "Test" angezeigt.

  11. Klicken Sie auf Test, um zu überprüfen, ob der Net Service-Name einwandfrei funktioniert, oder klicken Sie auf Fertig, um Ihre Konfiguration zu speichern und den Net Service-Name-Assistenten zu beenden.

    Wenn Sie auf "Test" klicken, meldet sich Oracle Net bei dem Datenbank-Server mit den von Ihnen konfigurierten Verbindungsdeskriptor-Informationen an. Deshalb müssen für einen erfolgreichen Test die Datenbank und der Listener hochgefahren sein. Wenn dies nicht der Fall ist, wird im Kapitel Establishing a Connection and Testing the Network im Oracle10i Net Administrator's Guide beschrieben, wie die Komponenten gestartet werden. Bei einem erfolgreichen Test wird folgende Meldung ausgegeben:

    Test erfolgreich.

    Wenn der Test erfolgreich war, klicken Sie auf Schließen, um das Dialogfeld "Verbindungstest" zu schließen, und gehen Sie zu Schritt 12.

    Wenn der Test nicht erfolgreich war:

    1. Prüfen Sie, ob Datenbank und Listener hochgefahren sind, und klicken Sie dann auf Test.

    2. Klicken Sie auf "Anmeldung ändern", um den Benutzernamen und das Kennwort für die Verbindung zu ändern. Danach klicken Sie auf Test.

  12. Klicken Sie auf Fertig, um die Konfiguration zu speichern und den Net Service-Name-Assistenten zu schließen.

  13. Stellen Sie sicher, dass das Profil die lokale (LOCAL) oder Verzeichnisbenennungs- (LDAP) Methode reflektiert.
