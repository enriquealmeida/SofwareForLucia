Oracle Advanced Security: Altri parametri

La scheda Altri parametri consente di configurare ulteriori parametri dei servizi di autenticazione.

Dalla lista Servizio di autenticazione, selezionare il metodo di autenticazione. Dalla lista {1}Servizio di autenticazione{2}, selezionare il metodo di autenticazione e immettere le informazioni appropriate negli appositi campi.

KERBEROS(v5)

Elemento Descrizione
Servizio Immettere una stringa qualsiasi per specificare il nome del servizio Kerberos.
File cache delle credenziali Immettere un percorso valido per specificare il nome del percorso della cache delle credenziali di Kerberos.
File di configurazione Immettere un percorso valido per specificare il nome del file di configurazione di Kerberos.
File di conversione realm Immettere un percorso valido per specificare il nome del file di conversione dei realm di Kerberos.
Tabella delle chiavi Immettere un percorso valido per specificare il nome del file delle chiavi segrete di Kerberos.
Disallineamento clock Immettere un numero intero positivo per specificare la differenza accettabile per quanto riguarda il numero di secondi che possono trascorrere tra l'invio e la ricezione delle credenziali.

CYBERSAFE

Servizio GSSAPI: specificare un principal del servizio CYBERSAFE. Immettere una stringa correttamente formattata relativa al principal del servizio.

NTS

Non sono richiesti parametri aggiuntivi.

RADIUS

Elemento Descrizione
Nome host Immettere il nome del server RADIUS primario.
Numero porta Immettere il numero della porta del server RADIUS primario.
Timeout (secondi) Immettere il numero di secondi che il database server Oracle deve attendere per ottenere una risposta dal server RADIUS primario.
Numero di nuovi tentativi Immettere il numero di tentativi che il database server Oracle deve eseguire per rinviare i messaggi al server RADIUS primario.
File segreto Immettere il nome di percorso del file della chiave segreta.
Invia servizi di accounting Specificare ON per abilitare l'accounting oppure OFF per disabilitarlo.
Challenge Response Immettere ON per abilitare la funzione Challenge/Response oppure OFF per disabilitarla.
Parola chiave predefinita Immettere la parola chiave per la richiesta di challenge dal server RADIUS.
Nome classe interfaccia Immettere il nome della classe creata per gestire la conversazione challenge/response tra il client Oracle e il server RADIUS.

Per ulteriori informazioni, vedere il manuale Oracle Advanced Security Administrator's Guide.
