Oracle Advanced Security: Verschlüsselung

In der Registerkarte Verschlüsselung können Sie die Verschlüsselungsparameter für die ausgewählten Authentifizierungsmethoden konfigurieren.

Hinweis: Verwenden Sie das Register SSL, um Cipher Suites für die Secure Sockets Layer (SSL) zu konfigurieren.

Encryption

Wählen Sie den Rechner, den Sie momentan konfigurieren, aus der Liste: Client oder Server.

Verschlüsselungstyp

Wählen Sie einen Wert, um das Client- oder Server-Verhalten bei Verschlüsselung und Integrität anzugeben. Die vier möglichen Verschlüsselungstypen sind:

Element Beschreibung
Akzeptiert Der Service ist aktiv, wenn die andere Seite der Verbindung "obligatorisch" oder "angefordert" angibt und ein kompatibler Algorithmus auf der anderen Seite verfügbar ist. Sonst ist der Service inaktiv.
Zurückgewiesen Der Service muss nicht aktiv sein, und eine Verbindung kann nicht erfolgreich aufgebaut werden, wenn die andere Seite "obligatorisch" angibt.
Angefordert Der Service ist aktiv, wenn die andere Seite der Verbindung "akzeptiert", "obligatorisch" oder "angefordert" angibt und ein kompatibler Algorithmus auf der anderen Seite verfügbar ist; sonst ist er inaktiv.
Obligatorisch Der Service muss aktiv sein, und die Verbindung kann nicht erfolgreich aufgebaut werden, wenn die andere Seite "zurückgewiesen" angibt oder wenn kein kompatibler Algorithmus auf der anderen Seite vorhanden ist.

EncryptionSeed

Geben Sie ein Verschlüsselungsschema in Form einer beliebigen Zeichenfolge mit maximal 256 zeichen in das Feld Verschlüsselungsschema ein. Mit dem Verschlüsselungsschema werden kryptographische Schlüssel generiert. Dies ist erforderlich, wenn Verschlüsselung oder Integrität aktiviert sind.

Hinweis : Wenn Sie Sonderzeichen, wie z.B. ein Komma [,] oder eine rechte Klammer [)] als Bestandteil des Verschlüsselungsschema-Parameters verwenden möchten, empfiehlt Oracle, dass der Wert in Hochkommata gesetzt wird.

Liste Verfügbare Methoden/Ausgewählte Methoden

Zeigt die Berechtigungsprüfungsmethoden, die ausgewählt werden können, sowie die ausgewählten Verschlüsselungsmethoden an. Die Reihenfolge der Methoden in der Liste bestimmt die bevorzugte Reihenfolge für die Verhandlung.

Mit den Pfeilen nach links (<) und rechts (>) verschieben Sie einen ausgewählten Verschlüsselungsalgorithmus zwischen den Listen. Die folgenden Verschlüsselungsalgorithmen werden von Oracle Advanced Security unterstützt:

Element Beschreibung
AES256 AES 256-Bit-Schlüssel
RC4_256 RC4 256-Bit-Schlüssel
AES192 AES 192-Bit-Schlüssel
3DES168 Triple DES mit einer Drei-Schlüssel-(168-Bit-)Option
AES128 AES 128-Bit-Schlüssel
RC4_128 RC4 128-Bit-Schlüssel
3DES112 Triple DES mit einer Zwei-Schlüssel-(112-Bit-)Option
DES DES 56-Bit-Schlüssel
RC4_40 RC4 40-Bit-Schlüssel
DES40 DES 40-Bit-Schlüssel
