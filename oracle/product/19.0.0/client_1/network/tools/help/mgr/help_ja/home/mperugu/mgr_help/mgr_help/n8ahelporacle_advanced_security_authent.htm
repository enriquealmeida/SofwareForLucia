Oracle Advanced Security: 認証

「認証」タブを使用して、認証方式を選択し優先順位を付けます。次のリストでは、Oracle Advanced Securityでサポートしている認証方式を示します。

KERBEROS5

シークレット・キーの暗号化を使用して、分散環境での安全なシングル・サインオン機能のサポートを提供します。パスワードは集中リポジトリに格納されるため、ネットワークからの盗聴が不可能になります。データベース・リンク認証とPCのセキュリティ強化にも使用されます。

CYBERSAFE

Kerberosベースの認証サーバーで、鍵の管理および秘密鍵の共有を基にして、安全性が確保された認証を提供します。Challengerには、トークン・セキュリティ・カードを使用したパスワードのチェックおよび認証の機能があります。

NTS

OSが行う認証を、Windows NT上のOracleサーバーとクライアントの間で行うことができます。Windows NTシステム固有の認証方式によって、Windows NTを介したデータベース・ユーザー認証が可能になります。これによって、Windows NTサーバー上のOracleデータベースとクライアント・コンピュータを、安全に接続できます。Windows NTシステム固有の認証方式によって、Windows NTクライアント・ユーザー名がWindows NTサーバーで確認されると、安全な接続が確立します。これで、ユーザーはサーバー上のデータベースを操作できます。

RADIUS

トークン・カード、スマートカードなど、RADIUS準拠のデバイスを使用した認証を提供します。RADIUSサーバーは、Oracleサーバーと対象となる認証サーバー間の情報のやりとりを中継します。ユーザーからみると、認証手続きはシームレスに透過的に行われます。

< ボタン

「使用可能なメソッド」リストで選択した認証方式を、「選択メソッド」リストに移動します。

> ボタン

「選択メソッド」リストで選択した認証方式を、「使用可能なメソッド」リストに移動します。

「上へ」ボタン

「選択メソッド」リスト内で、選択した認証方式を上に移動します。クライアント/サーバーのネゴシエーションでは、リストに表示される順に認証方式が試行されます。Oracle Netでは、リストの先頭にある認証が最初に使用されます。最初のメソッドを使用できない場合は、2番目以降を順番に使用します。

「下へ」ボタン

「選択メソッド」リスト内で、選択した認証方式を下に移動します。

注意: SSL付きTCP/IPプロトコルを構成している場合、デフォルトでSecure Sockets Layer (SSL)が構成されます。このタブで選択した認証方式は、SSLの認証方式より優先されます。SSLの追加オプションを構成するには、「SSL」タブを使用します。SSL付きTCP/IPプロトコルを使用する場合は、他の認証方式を使用しないでください。「選択メソッド」リストは必ず空にしてください。

詳細は、Windows NTのマニュアルまたはOracle Advanced Security管理者ガイドを参照してください。
