Configuración de un Descriptor de Conexión para Servicios Heterogéneos

Los servicios heterogéneos permiten acceder a un sistema que no sea de Oracle desde Oracle Server. Para iniciar una conexión a un sistema que no sea de Oracle, Oracle Server inicia un proceso de agente mediante el listener del gateway. Para que la base de datos Oracle se conecte a un sistema que no sea de Oracle, configure el servidor con un nombre de servicio de red.

Para configurar el servidor para servicios heterogéneos:

  1. Compruebe que el listener está configurado para servicios heterogéneos, como se describe en Configuración de Servicios Heterogéneos para el Listener.

  2. Cree un nombre de servicio de red en el archivo tnsnames.ora que se puede utilizar para las conexiones desde el servidor de la base de datos Oracle a un sistema que no sea de Oracle. Asegúrese de que la información del descriptor de conexión coincide con la información configurada para el listener.

  3. Seleccione el nombre de servicio de red en la carpeta Local > Nomenclatura de Servicios.

  4. En el recuadro Identificación de Servicios del panel de la derecha, seleccione Avanzado. Aparece el recuadro de diálogo Opciones de Servicio Avanzadas.

  5. Seleccione Usar para Servicios Heterogéneos y haga clic en Aceptar.

  6. Haga clic en Aplicar en el panel de la derecha.

  7. Seleccione Archivo > Guardar Configuración de Red.
