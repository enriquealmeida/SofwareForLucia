Configuration d'un descripteur de connexion pour les procédures externes

Les procédures externes sont des procédures appelées par un autre programme mais écrites dans un langage différent. Par exemple, c'est le cas pour un programme PL/SQL qui appelle une ou plusieurs routines C nécessaires à la réalisation d'un traitement particulier.

Pour que la base de données Oracle puisse se connecter à une procédure externe, le serveur de base de données doit être configuré avec un nom de service réseau lui donnant accès aux procédures externes.

Remarque : Pendant l'installation, l'assistant Oracle Net Configuration crée automatiquement un nouveau nom de service réseau pour les procédures externes.

Pour configurer un serveur de base de données pour des procédures externes :

  1. Assurez-vous que le processus d'écoute est configuré pour les procédures externes, comme décrit dans Configuration des procédures externes pour le processus d'écoute.

  2. Créez un nom de service réseau dans le fichier tnsnames.ora de la base de données. Assurez-vous que les informations du descripteur de connexion correspondent aux informations configurées pour le processus d'écoute.

  3. Choisissez Fichier > Enregistrer la configuration réseau.
