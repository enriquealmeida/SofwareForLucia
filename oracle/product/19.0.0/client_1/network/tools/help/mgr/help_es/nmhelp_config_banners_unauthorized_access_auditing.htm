Configuración de Rótulos para Acceso No Autorizado y Auditoría de Acciones del Usuario

Debe crear y configurar rótulos para advertir a los usuarios sobre accesos no autorizados o posibles auditorías de acciones del usuario. Los avisos estarán disponibles en la aplicación del cliente cuando se conecten a la base de datos.

Para que aparezcan estos rótulos, defina los siguientes parámetros de sqlnet.ora en el servidor de la base de datos para que apunten a un archivo de texto que contenga la información de rótulo:

  • SEC_USER_UNAUTHORIZED_ACCESS_BANNER. Por ejemplo: SEC_USER_UNAUTHORIZED_ACCESS_BANNER = /opt/Oracle/11g/dbs/unauthaccess.txt

  • SEC_USER_AUDIT_ACTION_BANNER.Por ejemplo: SEC_USER_AUDIT_ACTION_BANNER = /opt/Oracle/11g/dbs/auditactions.txt

Por defecto, estos parámetros no están definidos.

Después de definir estos parámetros, la aplicación Oracle Call Interface deberá utilizar las API de OCI adecuadas para poder recuperar los rótulos y presentarlos al usuario final.
