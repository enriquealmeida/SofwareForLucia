Oracle Connection Manager

Un router attraverso il quale una richiesta di connessione viene inviata dal client all'hop successivo o direttamente al database server. I client che instradano le richieste di connessione con Oracle Connection Manager possono usufruire delle funzioni di controllo dell'accesso, multiplex e conversione dei protocolli configurate in Oracle Connection Manager.

Oracle Connection Manager supporta l'instradamento delle connessioni per ambienti che presentano le caratteristiche descritte di seguito.

  • Molti utenti devono accedere ad un unico servizio mediante un singolo protocollo.

  • Sul client e sul server sono installati protocolli diversi, che non consentono di stabilire una connessione tipica.

  • Le connessioni tipiche di Oracle Net richiedono l'installazione dello stesso protocollo sia sul client che sul server.

  • Il controllo dell'accesso client ai server indicati in un ambiente TCP/IP è necessario.

Per informazioni su Oracle Connection Manager, vedere il capitolo Networking Challenges in the Internet Age del manuale Oracle9i Net Services Administrator's Guide.
