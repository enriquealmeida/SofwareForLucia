Enlace de Base de Datos

Puntero que define una ruta de acceso de comunicación unidireccional de un servidor de base de datos Oracle a otro servidor de base de datos. El puntero de enlace se define realmente como una entrada de una tabla del diccionario de datos. Para acceder al enlace, debe conectarse a la base de datos local que contiene la entrada del diccionario de datos.

Una conexión de enlace de base de datos es unidireccional ya que un cliente conectado a la base de datos local A puede utilizar un enlace almacenado en la base de datos A para acceder a información de la base de datos remota B, pero los usuarios conectados a la base de datos B no pueden utilizar el mismo enlace para acceder a los datos de la base de datos A. Si los usuarios locales de la base de datos B desean acceder a los datos de la base de datos A, deben definir un enlace que se almacena en el diccionario de datos de la base de datos B.

Se soportan los siguientes tipos de enlaces de base de datos:

  • Un enlace de base de datos privado en un esquema concreto de una base de datos. Sólo lo puede utilizar el propietario de dicho enlace.

  • Un enlace de base de datos público para una base de datos. Lo pueden utilizar todos los usuarios de la base de datos.

Para obtener más información sobre la creación de enlaces de bases de datos privados y públicos, consulte Oracle10i Database Administrator's Guide (Guía del Administrador de la Base de Datos Oracle10i).
