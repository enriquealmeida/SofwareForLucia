Oracle Context

Ein Relative Distinguished Name cn=OracleContext in einem Directory Information Tree (DIT) unter einem Benennungskontext oder einem nicht veröffentlichten Verzeichniseintrag. Ein Oracle Context enthält Einträge zur Benutzung mit Oracle-Software, wie z.B. Datenbankdienste und Net Service-Namen zur Benutzung mit der Verzeichnisbenennung. In einem Verzeichnis können ein oder mehrere Oracle Contexts vorhanden sein.

Oracle Internet Directory erstellt automatisch einen Oracle Context im Stamm der DIT-Struktur. Dieser Root Oracle Context hat einen Distinguished Name (DN) (dn:cn=OracleContext).

Wenn Oracle Net Manager gestartet wird, verwendet er den standardmäßigen Oracle Context, der von dem Oracle Net-Konfigurationsassistenten konfiguriert wurde. Sie können diesen Oracle Context in einen anderen Kontext ändern. Bei den meisten Anwendungen müssen Sie zusätzliche Oracle Contexts erstellen.
