Oracle Advanced Security: SSL

SSL 탭을 사용하여 SSL(Secure Sockets Layer) 설정을 수정할 수 있습니다. SSL은 네트워크 통신 보안을 위한 산업 표준 프로토콜로 인증, 비밀번호화, 데이터 무결성을 제공합니다. 클라이언트와 서버 사이의 통신 보안을 유지하려면 SSL을 사용합니다. 특히, SSL을 사용하여 하나 이상의 Oracle 서버에 대해 클라이언트나 서버를 인증하거나 클라이언트에 대해 Oracle 서버를 인증할 수 있습니다.

SSL 구성

클라이언트 또는 서버에 대한 설정을 지정하려면 목록에서 선택합니다.

서버에 구성해야 할 설정은 클라이언트에 대한 설정과 유사합니다. 추가 매개변수로 [클라이언트 인증 필요]라는 체크 박스가 하나 더 있을 뿐입니다.

암호화 모음 구성

기본적으로 몇 가지 SSL 암호화 모음이 설치되어 있습니다. 비밀번호 모음을 직접 추가하면 이러한 기본 비밀번호 모음을 겹쳐 씁니다.

요소 설명
추가 단추 [활성화할 암호화 모음 선택] 대화상자를 엽니다. 대화상자에서 암호화 모음을 선택한 다음 [확인]을 선택합니다. 암호화 모음이 목록 상자에 추가되었습니다. 주: 모든 Oracle Advanced Security 암호화 알고리즘 및 키 길이는 미국 및 기타 지역에서 사용할 수 있습니다.
제거 단추 선택된 암호화 모음을 제거합니다.
올림 단추 선택된 암호화 모음을 목록의 상위 레벨로 이동합니다.
내림 단추 선택된 암호화 모음을 목록의 하위 레벨로 이동합니다.

취소 확인(서버 전용)

인증서에 대한 취소 확인 여부를 지정합니다. 다음 값 중 하나를 선택합니다.

  • 없음: 인증서 취소 확인을 해제하려면 선택합니다.

  • 필요함: 인증서를 사용 가능한 경우 인증서 취소를 수행하려면 선택합니다. 인증서가 취소되고 적합한 CRL(인증서 취소 목록)이 없는 경우 SSL 접속을 거부합니다. 인증서의 취소 상태를 확인하기 위한 적합한 CRL이 없고 인증서가 취소되지 않은 경우 SSL 접속을 승인합니다.

  • 요청됨: CRL이 사용 가능한 경우 인증서 취소를 수행하려면 선택합니다. 인증서가 취소된 경우 SSL 접속을 거부합니다. 인증서의 취소 상태를 확인하기 위한 적합한 CRL이 없고 인증서가 취소되지 않은 경우 SSL 접속을 승인합니다.

SSL 버전 필요(선택 사항)

목록에서 SSL 버전을 선택합니다. 클라이언트와 서버는 호환 가능한 SSL 버전을 사용해야 합니다. 기존 또는 이후 SSL 버전을 사용할 수 있도록 선택할 수 있습니다.

클라이언트 인증 필요(서버 전용)

이 체크 박스는 기본적으로 선택되어 있습니다. 클라이언트측 인증이 필요하지 않으면 이 옵션의 선택을 해제합니다.

서버 X.509 이름 일치(클라이언트 전용)

목록에서 서버의 DN(식별 이름)이 서버의 서비스 이름과 일치하는지 여부를 확인할 것인지 선택합니다. 일치 검증을 강제 수행하는 경우 SSL은 인증서가 서버에서 발급된 것인지 확인합니다. 일치 검증을 강제 수행하도록 선택하지 않은 경우 SSL은 확인을 수행하지만 일치 여부에 관계없이 접속을 허용합니다. 일치 검증을 강제 수행하지 않으면 서버에서 해당 ID를 위조할 가능성이 있습니다. 다음 값 중 하나를 선택합니다.

  • 예: 서버 DN을 확인하도록 선택합니다. DN이 서비스 이름과 일치할 경우 접속은 성공합니다. DN이 서비스 이름과 일치하지 않을 경우 접속은 성공하지만 오류가 sqlnet.log 파일에 기록됩니다.

  • 아니오: 서버 DN을 확인하지 않도록 선택합니다. 이 확인을 무시하면 서버에서 ID를 조작할 수 있습니다.
