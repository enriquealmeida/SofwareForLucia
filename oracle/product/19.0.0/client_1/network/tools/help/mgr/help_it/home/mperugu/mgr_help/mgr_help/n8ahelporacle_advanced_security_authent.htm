Oracle Advanced Security: Autenticazione

La scheda Autenticazione consente di selezionare e assegnare una priorità ai metodi di autenticazione. Di seguito è riportata una lista dei metodi di autenticazione supportati da Oracle Advanced Security.

KERBEROS5

Include funzioni Single Sign-On sicure in un ambiente distribuito con crittografia mediante chiave segreta. Le password sono memorizzate in un repository centrale, pertanto è impossibile risalirvi dalla rete. Kerberos supporta inoltre l'autenticazione dei database link e altre funzioni di sicurezza avanzate per i PC.

CYBERSAFE

Un server di autenticazione basato su Kerberos che fornisce autenticazioni protette mediante la gestione delle chiavi e i segreti condivisi. Il modulo Challenger dispone di funzioni di controllo e autenticazione della password con schede token.

NTS

Supporta l'autenticazione del sistema operativo tra un client e un server Oracle in Windows NT. Il metodo di autenticazione nativa di Windows NT prevede l'autenticazione dell'utente del database tramite Windows NT. In questo modo i computer client sono in grado di stabilire connessioni protette con un database Oracle8 su un server Windows NT. Una connessione è protetta quando il nome utente di un client Windows NT viene recuperato su un server Windows NT tramite il metodo di autenticazione nativa di Windows NT. Il server Windows NT consente quindi all'utente di eseguire sul server tutte le operazioni relative al database.

RADIUS

Supporta tutti i metodi di autenticazione conformi allo standard RADIUS, incluse le schede token e le smart card. Il server RADIUS trasmette le informazioni tra il server Oracle e i server di autenticazione designati. Dal punto di vista dell'utente, l'intero processo di autenticazione avviene in modo uniforme e trasparente.

Pulsante <

Scegliere questo pulsante per spostare un metodo di autenticazione selezionato dalla lista Metodi disponibili alla lista Metodi selezionati.

Pulsante >

Scegliere questo pulsante per spostare un metodo di autenticazione selezionato dalla lista Metodi selezionati alla lista Metodi disponibili.

Pulsante Avanza

Scegliere questo pulsante per spostare un metodo di autenticazione selezionato in alto nella lista Metodi selezionati. La negoziazione client/server tenterà di utilizzare i metodi nell'ordine in cui sono elencati e Oracle Net utilizzerà il primo metodo di autenticazione presente nella lista. Oracle Net utilizzerà il primo metodo di autenticazione presente nella lista. Se il primo metodo non è disponibile, viene utilizzato il secondo e così via.

Pulsante Retrocedi

Scegliere questo pulsante per spostare un metodo di autenticazione selezionato in basso nella lista Metodi selezionati.

Nota: se si è configurato il protocollo TCP/IP con il protocollo SSL (Secure Sockets Layer), quest'ultimo viene configurato per impostazione predefinita. Qualsiasi metodo di autenticazione selezionato in questa scheda avrà la precedenza rispetto alle funzioni di autenticazione SSL. Per configurare altre opzioni SSL, utilizzare la scheda SSL. Se è impostato il protocollo TCP/IP con il protocollo SSL, non occorre utilizzare altri metodi di autenticazione. La lista Metodi selezionati deve essere vuota.

Per ulteriori informazioni, vedere la documentazione di Windows NT o Oracle Advanced Security Administrator's Guide.
