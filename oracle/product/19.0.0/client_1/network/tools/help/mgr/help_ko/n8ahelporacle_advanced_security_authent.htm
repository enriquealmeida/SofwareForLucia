Oracle Advanced Security: 인증

인증 탭을 사용하여 인증 방법을 선택하고 우선 순위를 지정할 수 있습니다. 다음은 Oracle Advanced Security가 지원하는 인증 방법 목록입니다.

KERBEROS5

분산 환경에서 보안 키 암호화와 함께 보안된 Single Sign-On 기능을 지원합니다. 비밀번호는 네트워크 상에서 유출이 불가능한 형태로 중앙 저장소에 저장됩니다. 또한 Kerberos는 데이터베이스 링크 인증과 향상된 PC 보안을 제공합니다.

CYBERSAFE

키 관리 및 공유 기밀 기반의 보안 인증을 제공하는 Kerberos 기반 인증 서버입니다. Challenger는 토큰 보안 카드로 비밀번호를 확인하고 인증합니다.

NTS

Windows NT에서 Oracle 서버와 클라이언트 간에 운영 체제 인증이 수행됩니다. Windows NT 원시 인증 방법을 사용하면 Windows NT를 통해 데이터베이스 사용자를 인증할 수 있습니다. 이렇게 하면 클라이언트 컴퓨터가 Windows NT 서버의 Oracle 데이터베이스에 안전하게 접속할 수 있습니다. Windows NT 원시 인증 방법을 통해 Windows NT 서버에서 Windows NT 클라이언트 도메인 사용자 이름이 검색되면 접속이 안전하게 됩니다. 그러면 Windows NT 서버는 서버에서 해당 사용자의 데이터베이스 작업을 허용합니다.

RADIUS

토큰 카드, 스마트 카드를 포함하여 RADIUS 표준을 준수하는 모든 인증 방법을 지원합니다. RADIUS 서버는 Oracle 서버와 지정된 인증 서버 간에 정보를 전달합니다. 사용자 측면에서 전체 인증 프로세스는 완벽하고 투명하게 수행됩니다.

주: SSL 프로토콜을 사용하는 TCP/IP를 구성한 경우 기본적으로 SSL(Secure Sockets Layer)가 구성됩니다. 현재 탭에서 선택한 인증 방법은 SSL 인증 기능을 무효화합니다. 추가 SSL 옵션을 구성하려면 SSL 탭을 사용합니다. SSL 프로토콜을 사용하는 TCP/IP를 사용하는 경우 다른 인증 방법을 사용하지 마십시오. 선택된 방법 목록은 비어 있어야 합니다.
