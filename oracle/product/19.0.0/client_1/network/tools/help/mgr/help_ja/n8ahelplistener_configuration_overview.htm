リスナー構成の概要

このセクションでは、次のトピックについて説明します。

インストール中の構成

Oracle Net Configuration Assistantでは、データベース・サーバーへのEnterprise EditionまたはStandard Editionのインストール中に、Oracleデータベース用のTCP/IPリスニング・プロトコル・アドレスを持つリスナーLISTENERが構成されます。カスタム・インストール中には、選択したリスナー名とプロトコル・アドレスの構成が要求されます。

また、外部プロシージャ・コールをリスニングするIPCプロトコル・アドレスは、インストール・タイプに関係なく自動的に構成されます。

さらに、Oracle Net Configuration Assistantにより、外部プロシージャのサービス情報もlistener.oraファイルに自動的に構成されます。

Oracle Net Managerを使用した構成

Oracle Net Managerでは、次の操作を実行できます。

  • リスナーの作成

  • Oracle8以下のデータベースのサービス情報の構成

  • 外部プロシージャ、異機種間サービスなど、Oracle以外のデータベース・サービスに関するサービス情報の構成

  • トレースおよびロギング機能の有効化

  • リスナー制御ユーティリティのパスワード認証の有効化

リスナー構成にアクセスする手順は、次のとおりです。

ナビゲータ・ペインで、「Oracle Netの構成」→「ローカル」→「リスナー」を展開します。

詳細は、Oracle10i Net管理者ガイドの「リスナーの構成および管理」の章を参照してください。
