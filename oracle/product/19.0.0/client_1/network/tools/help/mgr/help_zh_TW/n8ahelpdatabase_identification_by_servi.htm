使用服務名稱來識別資料庫

Oracle 資料庫是以服務的方式呈現給從屬端，亦即資料庫會代表從屬端執行工作。一個資料庫可以有一或多個與其關聯的服務。

一直到 Oracle8i 為止，都是使用 Oracle 系統 ID (SID) 來唯一識別 Oracle 資料庫服務。資料庫內部也使用 SID 作為指向「系統整體區域 (SGA)」的指標。從屬端也是藉由在連線描述區中指定 SID，來連線執行處理。此命名綱要並不區分服務和執行處理。

因為 Oracle 資料庫可跨多台電腦執行，所以整體服務和其個別執行處理都是在 Oracle9i 與 Oracle8i 中指定。

服務名稱

資料庫現在是由其服務名稱來識別的。服務名稱則是由初始化參數檔中的 SERVICE_NAMES 參數所指定。SERVICE_NAMES指定了可跨執行處理和 (或) 節點執行之 Oracle 資料庫服務的最高階檢視名稱。SERVICE_NAMES 的預設值是全域資料庫名稱，此名稱是由資料庫名稱和網域名稱所組成。

執行處理名稱

資料庫執行處理是由執行處理名稱 (初始化參數檔中的 INSTANCE_NAME 參數) 來識別的。INSTANCE_NAME對應至執行處理的 SID。

連線描述區

使用 SERVICE_NAME (沒有 S) 參數設定連線至 Oracle9i 或 Oracle8i 資料庫的連線描述區。例如，下列連線描述區包含位於 sales1-sun 之監聽器的位址，該監聽器會監聽 sales.us.example.com 資料庫服務的連線要求：

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)))

或者，您也可以使用INSTANCE_NAME參數設定連線至資料庫特定執行處理的連線描述區。如果您的 Oracle9i Real Application Clusters 有多個執行處理，就需要這項設定。

例如，以下連線描述區含有位於 sales-sun1 的監聽器位址，可以監聽與 sales.us.example.com 資料庫相關的 sales1 執行處理之連線要求：

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)

(INSTANCE_NAME=sales1)))

如需有關在連線描述區的 CONNECT_DATA 區段指定執行處理名稱的詳細資訊，請參閱 Oracle10i Net Services Administrator's Guide 中的 Chapter 13, Enabling Advanced Features for Oracle Net Services。
