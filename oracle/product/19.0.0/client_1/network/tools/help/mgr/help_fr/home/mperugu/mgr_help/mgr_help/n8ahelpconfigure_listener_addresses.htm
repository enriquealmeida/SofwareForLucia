Configuration des adresses de processus d'écoute

Le module d'écoute doit être configuré avec une ou plusieurs adresses de protocole. Il reçoit des demandes de connexion des clients configurés avec les mêmes adresses.

Pour configurer une adresse de protocole pour le processus d'écoute :

  1. Dans le panneau du navigateur, développez Oracle Net Configuration > Local > Processus d'écoute.

  2. Sélectionnez un processus d'écoute.

  3. Dans la liste du panneau de droite, sélectionnez Emplacements d'écoute.

  4. Choisissez Ajouter une adresse. Un nouvel onglet Adresse apparaît.

  5. Sélectionnez un protocole, puis indiquez les informations appropriées sur les paramètres de protocole du protocole sélectionné.

    Pour les clients Oracle Net standard, si vous configurez le module d'écoute pour écouter sur TCP/IP, vous devez choisir le port par défaut 1521. Sinon vous devez configurer le paramètre LOCAL_LISTENER dans le fichier de paramètres d'initialisation et résoudre le nom du processus d'écoute via une méthode de résolution de noms.

  6. Si l'ordinateur a plusieurs adresses IP et que vous voulez que le processus d'écoute écoute sur toutes les adresses disponibles, configurez TCP/IP ou TCP/IP avec SSL et saisissez le nom d'hôte de l'ordinateur dans le champ Hôte.

  7. Eventuellement, cliquez sur Afficher paramètres avancés pour accéder aux paramètres permettant de définir l'espace tampon d'E/S des opérations d'émission et de réception de la session.

  8. Choisissez Fichier > Enregistrer la configuration réseau.

Pour plus d'informations sur la configuration d'un port autre que celui par défaut, reportez-vous au chapitre Configuring and Administering the Listener dans le Guide de l'administrateur Oracle10i Net.

Pour plus d'informations sur la configuration de l'espace tampon d'E/S, reportez-vous au chapitre Optimizing Performance dans le Guide de l'administrateur Oracle10i Net.
