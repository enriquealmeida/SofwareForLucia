일반 사항: 고급

[고급] 탭을 사용하면 클라이언트 또는 서버 컴퓨터에 대한 고급 프로파일 정보를 구성할 수 있습니다.

전송 작업 시간 초과

접속이 설정된 후 데이터베이스 서버가 클라이언트로의 전송 작업을 완료하는 데 걸리는 시간을 초 단위로 지정하는 데 사용됩니다.

클라이언트가 때때로 비정상적으로 종료되는 환경에서는 이 매개변수를 설정하는 것이 좋습니다. 데이터베이스 서버가 지정한 시간 내에 전송 작업을 완료하지 못한 경우 데이터베이스 서버는 ORA-12535:TNS:작업 시간이 초과됨 및 ORA-12608: TNS: 전송 시간 초과 발생 이라고 sqlnet.log 파일에 기록합니다.

이 매개변수가 없는 경우 데이터베이스 서버는 작동이 중지된 컴퓨터 또는 사용 중인 상태로 인해 데이터를 수신할 수 없는 클라이언트에게 응답을 계속 전송하게 됩니다.

또한 클라이언트측에서 이 매개변수를 설정하여, 접속이 설정된 후 클라이언트가 데이터베이스 서버에 대한 전송 작업을 완료하는 데 걸리는 시간을 초 단위로 지정할 수 있습니다. 이 매개변수가 없는 경우 클라이언트는 이미 요청으로 포화 상태가 된 데이터베이스 서버에 요청을 계속 보낼 수 있습니다.

수신 작업 시간 초과

접속이 설정된 후 데이터베이스 서버가 클라이언트 데이터를 기다리는 데 걸리는 시간을 초 단위로 지정하는 데 사용됩니다. 클라이언트는 지정된 시간 간격 내에 데이터를 전송해야 합니다.

클라이언트가 때때로 비정상적으로 종료되는 환경에서는 이 매개변수를 설정하는 것이 좋습니다. 클라이언트가 지정한 시간 내에 데이터를 전송하지 않는 경우 데이터베이스 서버는 ORA-12535:TNS:작업 시간이 초과됨 및 ORA-12609: TNS: 수신 시간 초과 발생 이라고 sqlnet.log 파일에 기록합니다.

이 매개변수가 없는 경우 데이터베이스 서버는 작동이 중지되었거나 문제가 발생한 클라이언트에서 데이터를 계속 기다리게 됩니다. 또한 클라이언트측에서 이 매개변수를 설정하여, 접속이 설정된 후 클라이언트가 데이터베이스 서버에서 응답 데이터를 기다리는 데 걸리는 시간을 초 단위로 지정할 수도 있습니다.

이 매개변수가 없는 경우 클라이언트는 요청으로 포화 상태가 된 데이터베이스 서버로부터 응답을 오랫동안 기다릴 수 있습니다.

총 전송 버퍼 크기

세션의 전송 작업에 대한 버퍼 공간 제한을 지정합니다. 이 매개변수는 TCP/IP, SSL을 사용하는 TCP/IP 및 SDP 프로토콜이 지원합니다.

총 수신 버퍼 크기

세션의 수신 작업에 대한 버퍼 공간 제한을 지정합니다. 이 매개변수는 TCP/IP, SSL을 사용하는 TCP/IP 및 SDP 프로토콜이 지원합니다.

TNS 시간 초과 값

클라이언트/서버 접속이 활성화되어 있는지 확인하기 위해 프로브를 전송하는 시간 간격을 분 단위로 지정합니다. 이 값을 0보다 크게 설정하면 클라이언트의 비정상적인 종료로 인해 접속이 무한히 유지되는 일을 막을 수 있습니다. 프로브는 종료된 접속을 발견하거나 더 이상 사용 중이 아닌 접속을 발견한 경우 오류를 반환하고 해당 서버 프로세스를 종료합니다. 이 설정은 일반적으로 동시에 여러 접속을 처리하는 데이터베이스 서버에 사용됩니다.

종료된 접속 감지 기능 사용에 대한 제한 사항은 다음과 같습니다.

  • bequeathed 접속에서는 허용되지 않습니다.

  • 프로브 패킷은 매우 적기는 하지만 추가의 통신량을 생성하므로 네트워크 성능을 저하시킬 수 있습니다.

  • 사용 중인 운영 체제에 따라 접속 프로브 이벤트와 다른 이벤트를 구별하기 위한 추가 프로세스를 수행해야 합니다. 이러한 추가 프로세스도 네트워크 성능을 저하시킬 수 있습니다.

주: 이 옵션은 서버에서만 사용할 수 있습니다.

클라이언트 등록 ID

이 클라이언트에 대한 고유 식별자를 입력합니다. 이 식별자는 접속 요청과 함께 리스너에 전달됩니다.

리스너는 로그 파일에 이 식별자를 포함합니다. 식별자는 최대 128자의 문자열을 사용할 수 있습니다.

로그온 인증 프로토콜 버전

클라이언트 또는 데이터베이스에 의해 허용되는 인증 프로토콜을 지정하는 데 사용됩니다. 클라이언트와 데이터베이스 서버에 일치하는 버전이 최소 한 개 이상 없으면 오류가 발생하면서 인증이 실패합니다.

지원되는 값은 다음과 같습니다.

  • 10i 인증 프로토콜의 경우 10

  • 9i 인증 프로토콜의 경우 9

  • 8i 인증 프로토콜의 경우 8

  • 10i, 9i 및 8i 인증 프로토콜의 경우 기본값

10 이외의 값을 지정하면 이전 버전의 인증 프로토콜에 존재하던 취약성이 노출될 수 있습니다. 모든 버전에 호환되도록 하려면 로그온에 허용되는 버전 목록을 Default 로 설정하여 시스템에 있는 모든 버전의 데이터베이스를 포함합니다.

UNIX 신호 처리 해제

신호 처리기를 사용 안함으로 설정하여 클라이언트가 하위 프로세스 지우기 결과를 UNIX init 프로세스로 전달하도록 구성합니다.

클라이언트 응용 프로그램은 내부적으로 서버 프로세스를 생성한 후 이 프로세스가 완료될 때 하위 프로세스를 지워야 합니다. 서버 프로세스가 접속 책임을 완료하면 소멸 프로세스가 됩니다. 신호 처리기는 이러한 소멸 프로세스 정리를 담당합니다.

진행 중인 작업 중단 해제

진행 중인 작업 중단을 설정 또는 해제합니다.

선택을 해제하는 경우 Oracle Net은 기본 프로토콜이 제공하는 긴급 데이터를 사용하여 "중지" 메시지를 전송하고 수신할 수 있습니다.

선택하는 경우 기본 프로토콜이 제공하는 긴급 데이터를 사용하여 "중지" 메시지를 전송하거나 수신하는 기능을 사용할 수 없습니다. 이 기능은 일단 사용하면 해당 클라이언트가 사용하는 모든 프로토콜에 적용됩니다.
