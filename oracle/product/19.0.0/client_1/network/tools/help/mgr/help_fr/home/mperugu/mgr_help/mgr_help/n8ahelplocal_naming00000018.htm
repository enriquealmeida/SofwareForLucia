Présentation de la résolution de noms locaux

La méthode de résolution de noms locaux est l'une des principales méthodes de résolution de noms utilisées pour transformer un nom simple, ou nom de service réseau, en descripteur de connexion dont le client se sert pour se connecter à une base de données ou à un service. Cette méthode stocke les noms de service réseau et leurs descripteurs de connexion dans un fichier tnsnames.ora local.

Un utilisateur final entre une chaîne de connexion comportant un nom de service réseau :

CONNECT nom_utilisateur/mot_passe@nom_service_réseau

Dans l'exemple suivant, sales est le nom de service réseau recherché dans le fichier tnsnames.ora :

CONNECT scott/tiger@sales

Le descripteur de connexion renvoyé au client comprend les éléments suivants :

  • Emplacement de l'adresse de protocole du processus d'écoute

  • Nom de service de la base de données

Le client utilise ces informations pour se connecter à un processus d'écoute. Le processus d'écoute, configuré avec les mêmes informations d'adresse, reçoit la demande de connexion. Si le processus est à l'écoute des demandes pour le même service de base de données que celui qui est indiqué dans le descripteur de connexion du client, une connexion est autorisée.

Pour plus d'informations, reportez-vous au chapitre Connectivity Concepts dans le Guide de l'administrateur Oracle10i Net.
