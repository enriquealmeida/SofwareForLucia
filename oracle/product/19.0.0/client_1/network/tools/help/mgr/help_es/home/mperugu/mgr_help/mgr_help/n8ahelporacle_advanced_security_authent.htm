Oracle Advanced Security: Autenticación

El separador Autenticación permite seleccionar y establecer prioridades en los métodos de autenticación. A continuación, se ofrece una lista de los métodos de autenticación que soporta Oracle Advanced Security.

KERBEROS5

Proporciona soporte a las funciones de conexión única y segura en un entorno distribuido con control de criptografía mediante clave secreta. Las contraseñas se almacenan en un repositorio central, por lo que resulta imposible descifrarlas. También proporciona la función de autenticación de enlaces de bases de datos y de seguridad de PC mejorada.

CYBERSAFE

Servidor de autenticación basado en Kerberos que permite la autenticación segura mediante la gestión de claves y secretos compartidos. Challenger dispone de comprobación de contraseña y autenticación con tarjetas de identificación de seguridad.

NTS

Permite realizar la autenticación del sistema operativo entre un cliente y Oracle Server en Windows NT. El método de autenticación nativa de Windows NT permite la autenticación de usuarios de bases de datos a través de Windows NT. Así, las computadoras cliente pueden realizar conexiones seguras a una base de datos Oracle en un servidor Windows NT. Una conexión segura se realiza cuando un nombre de usuario de cliente de Windows NT se restablece en un servidor Windows NT mediante el método de autenticación nativa de Windows NT. El servidor Windows NT permite entonces al nombre de usuario realizar las operaciones de bases de datos en el servidor.

RADIUS

Soporta cualquier método de autenticación compatible con el estándar RADIUS, incluidas tarjetas de identificación y tarjetas inteligentes. Un servidor RADIUS transfiere información entre Oracle Server y los servidores de autenticación designados. Desde el punto de vista del usuario, todo el proceso de autenticación tiene lugar de modo uniforme y transparente.

Botón <

Haga clic para mover un método de autenticación seleccionado de la lista Métodos Disponibles a la lista Métodos Seleccionados.

Botón >

Haga clic para mover un método de autenticación seleccionado de la lista Métodos Seleccionados a la lista Métodos Disponibles.

Botón Aumentar

Haga clic para subir un método de autenticación seleccionado en la lista Métodos Seleccionados. La negociación Cliente/Servidor intentará utilizar los métodos de autenticación en el orden en que aparecen. La Red de Oracle utilizará primero la autenticación del principio de la lista. Si no puede utilizar el primer método, utiliza el segundo y así sucesivamente.

Botón Reducir

Haga clic para bajar un método de autenticación seleccionado en la lista Métodos Seleccionados.

Nota: si ha configurado el protocolo TCP/IP con SSL, SSL (Secure Sockets Layer) se configurará por defecto. Cualquier método de autenticación que seleccione en este separador sustituirá las funciones de SSL. Utilice el separador SSL para configurar otras opciones de SSL. Si utiliza el protocolo TCP/IP con SSL, no utilice ningún otro método de autenticación. La lista Métodos Seleccionados debe estar vacía.

Para obtener más información, consulte la documentación de Windows NT u Oracle Advanced Security Administrator's Guide (Guía del Administrador de Oracle Advanced Security).
