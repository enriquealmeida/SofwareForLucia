Database link

Un puntatore che definisce un percorso di comunicazione monodirezionale da un server di database Oracle a un altro server di database. Il puntatore viene definito come una voce in una tabella del dizionario dati. Per accedere al link, è necessario connettersi al database locale che contiene la voce del dizionario dati.

Una connessione tramite database link è monodirezionale nel senso che un client connesso al database locale A può utilizzare un link memorizzato nel database A per accedere alle informazioni contenute nel database remoto B, ma gli utenti connessi al database B non possono servirsi dello stesso link per accedere ai dati del database A. Gli utenti locali del database B che desiderano accedere ai dati nel database A devono definire un link da memorizzare nel dizionario dati del database B.

Sono supportati i seguenti database link:

  • Un database link privato in uno schema specifico di un database. È utilizzabile unicamente dal proprietario.

  • Un database link pubblico. È utilizzabile da tutti gli utenti.

Per ulteriori informazioni sulla creazione di database link privati e pubblici, vedere il manuale Oracle10i Database Administrator's Guide.
