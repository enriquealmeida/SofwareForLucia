Oracle Advanced Security : Intégrité

L'onglet Intégrité vous permet de configurer la fonction d'intégrité des données pour les méthodes d'authentification. Cette fonction compare les données envoyées dans le cadre de la connexion, telles qu'elles sont vues par le client et par le serveur.

Remarque :utilisez l'onglet SSL en vue de configurer les mécanismes de cryptage pour la couche SSL (Secure Sockets Layer).

Intégrité

Sélectionnez la case Serveur ou Client pour indiquer le type d'ordinateur que vous configurez. L'option Serveur ne s'applique pas aux serveurs auxquels cet ordinateur est connecté.

Niveau de checksum

Sélectionnez le niveau de checksum. Il existe quatre niveaux :

Elément Description
accepté Le service n'est actif que si l'autre machine participant à la connexion indique "requis" ou "demandé", et qu'elle dispose d'un algorithme compatible.
rejeté Le service ne doit pas être actif : la connexion échoue si l'autre machine indique "requis".
demandé Le service n'est actif que si l'autre machine participant à la connexion indique "accepté", "requis" ou "demandé", et qu'elle dispose d'un algorithme compatible.
requis Le service doit être actif et la connexion échoue si l'autre machine indique "rejeté" ou ne dispose pas d'un algorithme compatible.

Liste des méthodes disponibles

Affiche les algorithmes d'intégrité utilisables pour créer des checksums. Il en existe deux :

Nom d'algorithme Valeurs autorisées
Secure Hash Algorithm (SHA-1) SHA1
Message Digest 5 (MD5) MD5

Liste des méthodes sélectionnées

Affiche les algorithmes sélectionnés. L'ordre des algorithmes dans la liste détermine l'ordre dans lequel ils sont utilisés.
