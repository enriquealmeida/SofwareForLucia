外部プロシージャの接続記述子の構成

外部プロシージャは、別のプログラムからコールされるプロシージャで、異なる言語で記述されています。たとえば、特殊な処理の実行に必要な1つ以上のCルーチンをコールするPL/SQLプログラムはその一例です。

Oracleデータベースから外部プロシージャに接続するには、外部プロシージャにアクセスするためのネット・サービス名を指定してデータベース・サーバーを構成する必要があります。

注意: Oracle Net Configuration Assistantはインストール時、外部プロシージャのネット・サービス名を自動的に作成します。

外部プロシージャ用にデータベース・サーバーを構成する手順は、次のとおりです。

  1. 「リスナーの外部プロシージャの構成」の説明に従って、リスナーが外部プロシージャ用に構成されていることを確認します。

  2. データベースのtnsnames.oraファイル内に、ネット・サービス名を作成します。接続記述子情報が、リスナーに構成された情報と一致することを確認します。

  3. 「ファイル」→「ネットワーク構成の保存」を選択します。
