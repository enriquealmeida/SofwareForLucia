Protocole

Ensemble de règles définissant le mode de transmission des données dans les réseaux. Oracle Net Manager prend en charge les protocoles suivants :

IPC

Le protocole IPC (Interprocess Communication) est utilisé pour les applications client résidant sur le même ordinateur que le processus d'écoute. Avec IPC, les connexions locales sont plus rapides qu'avec TCP/IP.

Named Pipes

Named Pipes est utilisé sur des PC qui ne font pas appel au protocole TCP/IP. Named Pipes est disponible uniquement sous Windows NT.

TCP/IP

Le protocole TCP/IP (Transmission Control Protocol/Internet Protocol) est le protocole Ethernet constituant une norme de fait pour les communications client/serveur de base de données sur le réseau. TCP/IP permet à une application Oracle résidant sur un client de communiquer avec des bases Oracle distantes (à condition que ces dernières se trouvent sur des systèmes hôtes prenant en charge TCP/IP).

TCP/IP est le protocole réseau standard. Il est très utilisé par les réseaux UNIX et PC. C'est également le protocole de connexion Internet.

TCP/IP avec SSL

Le protocole TCP/IP avec SSL (Secure Sockets Layer) permet à une application Oracle résidant sur un client de communiquer avec des bases Oracle distantes (à condition que ces dernières se trouvent sur des systèmes hôtes prenant en charge TCP/IP avec SSL). TCP/IP avec SSL n'est utilisable qu'avec la méthode d'authentification SSL.

Remarque : Votre système d'exploitation peut prendre en charge d'autres protocoles.
