Oracle Advanced Security-Authentifizierungsmethoden konfigurieren

Mit zentralisierten, sicheren Authentifizierungsmethoden können Sie hohes Vertrauen in die Identität von Benutzern, Clients und Servern in verteilten Umgebungen haben. Die Netzwerk-Authentifizierungsmethoden bieten auch den Vorteil des Single Sign-On für Benutzer.

Mit Oracle Net Manager konfigurieren Sie die folgenden Authentifizierungsadapter:

  • Kerberos

  • CyberSafe

  • RADIUS

  • SSL

  • Systemeigene Windows NT-Berechtigungsprüfung

Hinweis: Wenn Sie das Protokoll TCP/IP with SSL konfiguriert haben, wird die Secure Sockets Layer (SSL) standardmäßig konfiguriert. Jede Berechtigungsprüfungsmethode, die Sie in diesem Register wählen, setzt die Funktionen zur Berechtigungsprüfung von SSL außer Kraft. Unter SSL-Register wird beschrieben, wie zusätzliche SSL-Optionen konfiguriert werden. Wenn Sie das Protokoll TCP/IP with SSL verwenden, verwenden Sie keine der anderen Berechtigungsprüfungsmethoden.

So konfigurieren Sie Authentifizierungsmethoden:

  1. Blenden Sie Oracle Net-Konfiguration > Lokal > Profil im Navigator-Fensterbereich ein.

  2. Wählen Sie Oracle Advanced Security aus der Liste im rechten Fensterbereich.

  3. Wählen Sie das Register Authentifzierung.

  4. Wählen Sie die gewünschte Berechtigungsprüfungsmethode aus der Liste Verfügbare Methoden, und klicken Sie dann auf den Pfeil nach rechts (>).

  5. Ordnen Sie die Berechtigungsprüfungsmethoden nach der Reihenfolge an, in der die Methoden verwendet werden sollen. Wählen Sie die Berechtigungsprüfungsmethode in der Liste Ausgewählte Methoden, und verschieben Sie die Auswahl mit den Schaltflächen Vorziehen oder Zurückstufen in der Liste nach oben oder unten.

  6. Wählen Sie das Register Andere Parameter, um zusätzliche Parameter für folgende Authentifzierungsmethoden zu konfigurieren:

    • Kerberos

    • CyberSAFE

    • RADIUS

  7. Wählen Sie Datei >Netzwerkkonfiguration speichern.
