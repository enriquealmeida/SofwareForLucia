Mehrere Adressoptionen konfigurieren

Wenn auf einen Datenbankdienst von mehreren Listener-Protokolladressen zugegriffen werden kann, muss unbedingt die Reihenfolge angegeben werden, in der die Protokolladressen verwendet werden sollen. Die Protokolladressen können z.B. zufällig oder folgerichtig ausgewählt werden.

Wenn mehrere Protokolladressen für einen Net Service-Namen oder Datenbankdienst konfiguriert wurden, können folgende Optionen konfiguriert werden:

  • Failover bei Anmeldezeit, um die erste Protokolladresse zu wählen und dann zu der nächsten Protokolladresse zu wechseln, wenn der Versuch mit der ersten Protokolladresse nicht erfolgreich verläuft.

  • Client-Load Balancing, um eine Protokolladresse zufällig auszuwählen.

  • Source Routing, um alle Protokolladressen sequentiell zu verwenden.

So konfigurieren Sie mehrere Adressoptionen:

  1. Blenden Sie im Navigator-Fensterbereich Oracle Net-Konfiguration> Verzeichnis> Lokal oder Dienstebenennung ein.

  2. Wählen Sie einen Net Service-Namen oder einen Datenbankdienst. Im rechten Fensterbereich werden der aktuelle Zieldienst und die Adressliste angezeigt.

  3. Klicken Sie im Feld Adresskonfiguration auf Erweitert. Daraufhin wird das Dialogfeld Adresslistenoptionen angezeigt.

  4. Löschen Sie die Option Mit Net 8.0-Clients kompatible Optionen verwenden, wenn diese Konfiguration für einen Oracle9i - oder Oracle8i-Client vorgenommen wird.

  5. Wählen Sie die für die Protokolladressliste zutreffende Option, und klicken Sie auf OK.

  6. Klicken Sie auf Anwenden im rechten Fensterbereich.

  7. Wenn Sie diese Änderungen an dem Ordner "Lokal" vornehmen, wählen Sie Datei > Netzwerkkonfiguration speichern.
