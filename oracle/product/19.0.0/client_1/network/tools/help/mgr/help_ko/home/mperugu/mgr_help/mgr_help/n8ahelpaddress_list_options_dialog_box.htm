주소 목록 옵션 대화상자

주소 목록 옵션 대화상자를 사용하여 접속할 때 접속 기술자의 프로토콜 주소가 사용되는 순서와 방법을 지정할 수 있습니다.

성공할 때까지 각 주소를 순서대로 시도

Oracle9i 또는 Oracle8i 클라이언트에 대해 접속 시간 복구를 활성화하려면 선택합니다. 첫번째 주소에 도달하지 못하면 이 복구 기능은 클라이언트가 다음 리스너 프로토콜 주소를 사용하도록 합니다. 8.0 또는 이전 릴리스 클라이언트에 대해 소스 경로 지정을 비활성화하려면 이 옵션 및 Net8 8.0 클라이언트와 호환되는 옵션 사용 옵션을 선택합니다.

성공할 때까지 각 주소를 임의의 순서로 시도

Oracle9i 또는 Oracle8i 클라이언트에 대해 클라이언트 로드 밸런싱 및 접속 시간 복구를 사용으로 설정하려면 선택합니다.

임의로 선택된 주소 시도

Oracle9i 또는 Oracle8i 클라이언트에 대해 클라이언트 로드 밸런싱 기능을 사용으로 설정하려면 선택합니다.

대상에 도달할 때까지 각 주소를 순서대로 사용

Oracle8i 또는 Oracle8 클라이언트에 대해 소스 경로 지정을 사용으로 설정하려면 선택합니다.

이 옵션은 대상에 도달할 때까지 둘 이상의 중간 주소가 요구될 때 필요합니다. 이 기능은 보통 Oracle Connection Manager 기능을 사용으로 설정하기 위해 사용됩니다.

Oracle9i 클라이언트에는 이 매개변수가 필요하지 않습니다.

첫번째 주소만 사용

소스 경로 지정, 접속 시간 복구 및 클라이언트 로드 밸런싱 기능을 사용하지 않으려면 이 단추를 선택합니다.

Net8 8.0 클라이언트와 호환되는 옵션 사용

릴리스 8.0 클라이언트를 구성 중인 경우 선택합니다. 이 경우 다음 옵션만 선택할 수 있습니다.

-성공할 때까지 각 주소를 순서대로 시도

-대상에 도달할 때까지 각 주소를 순서대로 사용

Oracle9i 또는 Oracle8i 클라이언트를 구성하는 경우에는 이 옵션의 선택을 해제합니다.
