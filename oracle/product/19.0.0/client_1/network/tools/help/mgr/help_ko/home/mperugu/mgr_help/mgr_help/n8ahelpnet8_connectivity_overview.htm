Oracle Net 접속 개요

Oracle Net을 사용하여 오라클 데이터베이스, 타사 데이터베이스, 게이트웨이 및 외부 프로시저 등의 여러 서비스에 접속할 수 있습니다.

사용자는 접속할 서비스에 서비스 식별자와 함께 사용자 이름 및 비밀번호를 전달하여 접속 요청을 시작합니다. 접속 식별자는 다음을 식별합니다.

  • 대상 서비스

  • 네트워크에서 해당 서비스를 찾기 위한 경로, 즉 네트워크 경로

접속 식별자는 다양한 방법으로 지정됩니다. 가장 일반적인 방법 중 하나는 접속 기술자에 매핑되는 네트 서비스 이름(서비스의 다른 이름)을 사용하는 것입니다. 접속 기술자에는 대상 서비스와 네트워크 경로 정보가 포함됩니다. Oracle9i 및 Oracle8i 데이터베이스의 경우에는 서비스 이름을 사용하고, Oracle8 또는 Oracle7 데이터베이스의 경우에는 Oracle SID(시스템 식별자)를 사용하여 원하는 서비스가 표시됩니다. 네트워크 경로는 네트워크 주소를 통해 최소한의 리스너 위치를 제공합니다.

다음은 접속 기술자에 매핑되는 sales라는 네트 서비스 이름을 보여줍니다.

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-server1)(PORT=1521))

(CONNECT_DATA=(SERVICE_NAME=sales.us.example.com))

sales1-server에 있는 리스너는 포트 1521의 TCP/IP 프로토콜을 통해 sales.us.example.com이라는 데이터베이스 서비스에 대한 접속 요청을 수신합니다.

접속 식별자와 해당 접속 기술자는 최소한 하나의 이름 지정 방법에 저장됩니다. 접속 요청 동안 클라이언트는 이름 지정 방법을 통해 접속 기술자에 대한 접속 식별자(이전 예제의 sales)를 분석합니다. 클라이언트는 접속 기술자에 지정된 리스너에 접속합니다.

리스너는 프로토콜을 통해 클라이언트 접속을 받아들입니다. 리스너는 클라이언트 정보를 자체 구성 파일, listener.ora에 저장된 정보 및 서비스 등록을 통해 데이터베이스 서비스에서 수신한 정보와 비교합니다. 정보가 일치하면 접속이 허용됩니다.
