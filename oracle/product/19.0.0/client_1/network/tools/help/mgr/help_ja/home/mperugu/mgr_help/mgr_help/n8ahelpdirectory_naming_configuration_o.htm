ディレクトリ・ネーミング構成の概要

このトピックでは、ディレクトリ・ネーミング構成の詳細を説明します。

インストール時のサーバーの構成

サーバーのカスタム・インストール時、Oracle Net Configuration Assistantにより次の事項が要求されます。

  • ディレクトリ・サーバーのタイプの選択

  • ディレクトリ位置の指定

  • サーバーが接続記述子を参照、作成および修正できる場所である、Oracleコンテキストの選択または作成

  • 使用するネーミング・メソッドとしての、ディレクトリ・ネーミングの構成

ディレクトリ使用について正しく構成されている場合、Database Configuration Assistantにより、そのディレクトリへの接続、およびディレクトリへのデータベース・サービス・エントリの登録が試行されます。データベース・サービス・エントリは、次のネットワーク情報を含む接続記述子を使用して構成されます。

  • リスナーのアドレス

  • データベース・サービスの名前

インストール中のクライアントの構成

どのクライアントをインストールする場合でも、Oracle Net Configuration Assistantにより、ディレクトリ使用について次の設定を要求するプロンプトが表示されます。

  • ディレクトリ・タイプの指定

  • ディレクトリ位置の指定

  • クライアントが接続記述子を参照できる場所、デフォルトのOracleコンテキストの選択

  • 使用するネーミング・メソッドとしてのディレクトリ・ネーミング

インストール後のOracle Net Configuration Assistantでの構成

Oracle Net Configuration Assistantをスタンドアローン・モードで実行して、次のことを行えます。

  • クライアントまたはサーバーの設定(前述を参照)

  • Oracleスキーマの作成

  • Oracleコンテキストの作成

Oracle Net Configuration Assistantをスタンドアロン・モードで実行する手順は、次のとおりです。

  • UNIXでは、$ORACLE_HOME/binにあるnetcaを実行します。

  • Windows NTでは、「スタート」→「プログラム」→「Oracle - HOME_NAME」→「Network Administration」→「Oracle Net Configuration Assistant」を選択します。

Oracle Net Managerを使用した構成

Oracle Net Managerでは、次の操作を実行できます。

  • Oracleコンテキストの変更

  • 認証資格証明の変更

  • ネット・サービス名の作成

  • ネット・サービス別名の作成

  • 複数のプロトコル・アドレスを通るリスナーへの複数ルートの構成

  • データベース・サービスまたはネット・サービス名の詳細CONNECT_DATA情報の構成

  • tnsnames.oraファイルからディレクトリ・サーバーへのネット・サービス名エントリの移行

ナビゲータ・ペインからディレクトリ・ネーミング構成にアクセスする場合は、「ディレクトリ・サービス・ネーミング」を選択します。

詳細は、Oracle Net Configuration Assistantのヘルプ・システムを参照してください。
