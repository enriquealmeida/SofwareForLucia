Listener-Adressen konfigurieren

Der Listener muss mit einer oder mehreren Protokolladressen konfiguriert werden. Der Listener empfängt Verbindungsanforderungen von Clients mit denselben Adressen.

So konfigurieren Sie eine Protokolladresse für den Listener:

  1. Blenden Sie Oracle Net-Konfiguration > Lokal> Listener im Navigator-Fensterbereich ein.

  2. Wählen Sie einen Listener.

  3. Wählen Sie Listening-Adressen aus der Liste im rechten Fensterbereich.

  4. Wählen Sie Adresse hinzufügen. Daraufhin wird ein neues Register Adresse angezeigt.

  5. Wählen Sie ein Protokoll, und geben Sie die entsprechenden Protokollparameter-Informationen für das ausgewählte Protokoll ein.

    Wenn Sie den Listener konfigurieren, damit er bei Oracle Net-Clients auf TCP/IP hört, müssen Sie den Standardport oder 1521 auswählen. Sonst müssen Sie den Parameter LOCAL_LISTENER in der Initialisierungsparameterdatei konfigurieren und den Listener-Namen über eine Benennungsmethode auflösen.

  6. Wenn Ihr Rechner über mehr als eine IP-Adresse verfügt und der Listener auf alle verfügbaren IP-Adressen hören soll, konfigurieren Sie TCP/IP oder TCP/IP with SSL und geben den Hostnamen des Rechners in das Feld Host ein.

  7. Optional klicken Sie auf Erweiterte anzeigen, um Einstellungen zur Angabe des E/A-Pufferspeichers für Senden- und Empfangenvorgänge der Session anzugeben.

  8. Wählen Sie Datei >Netzwerkkonfiguration speichern.

Weitere Informationen zur Konfiguration eines nicht-standardmäßigen Ports finden Sie in Kapitel Configuring and Administering the Listener im Oracle10i Net Administrator's Guide.

Weitere Informationen zur Konfiguration des I/O-Pufferspeicherplatzes finden Sie im Kapitel Optimizing Performance im Oracle10i Net Administrator's Guide.
