Exportando Entradas do Arquivo tnsnames.ora para um Servidor de Diretórios

Nomes de serviço de rede já configurados em um arquivo tnsnames.ora podem ser exportados para um servidor de diretórios. A exportação é feita por um domínio de cada vez.

A exportação de um arquivo tnsnames.ora cria um nome de serviço de rede no Oracle Context atualmente selecionado no diretório. Se você precisar alterar o Oracle Context, escolha Comando > Servidor de Diretórios > Alterar o Oracle Context.

Para exportar nomes de serviço de rede armazenados em um arquivo tnsnames.ora para um servidor de diretórios:

  1. Escolha Comando > Diretório > Exportar Nomes de Serviço. O Assistente de Migração de Diretório será iniciado.

  2. Escolha Próximo.

    Se vários domínios forem detectados, a página Selecionar Domínio será exibida. Continue na etapa 3.

    Se os nomes de serviços de rede não forem qualificados por domínio, a página Selecionar Nomes de Serviço de Rede será exibida. Passe para a Etapa 4.

  3. Selecione o domínio de rede cujos nomes de serviço de rede você deseja exportar e, em seguida, escolha Próximo. A página Selecionar Nomes de Serviço de Rede será exibida.

  4. Selecione os nomes de serviço de rede a serem exportados na lista e escolha Próximo. A página Selecionar Contexto de Destino será exibida.

    Na página Selecionar Contexto de Destino, faça o seguinte:

    a. Na lista Contexto de Nomeação de Diretório, selecione o contexto de nomeação de diretório que contém o Oracle Context. Esse contexto faz parte de uma subárvore de diretórios que contém um ou mais Oracle Contexts com entradas Oracle.

    b. Na lista Oracle Context, selecione o Oracle Context para o qual deseja exportar os nomes de serviço de rede selecionados.

    c. Escolha Próximo. A página Atualização de Servidor de Diretório é exibida com o status da operação de exportação.

  5. Escolha Finalizar para fechar o Assistente de Migração do Servidor de Diretórios.
