Creazione di un nome di servizio di rete

Nota: solo gli utenti che sono membri del gruppo OracleNetAdmins o OracleContextAdmins possono creare voci di nomi di servizi di rete in una directory.

Per aggiungere nomi di servizi di rete a un file tnsnames.ora o a un server delle directory, effettuare le operazioni riportate di seguito.

  1. Nel riquadro del Navigator, espandere Configurazione di Oracle Net > Directory o Locale > Denominazione dei servizi.

  2. Fare clic sul segno più (+) nella barra degli strumenti oppure scegliere Modifica > Crea nella barra dei menu. Viene visualizzata la pagina Benvenuti della Creazione guidata dei nomi di servizi di rete.

  3. Immettere un nome qualsiasi nel campo Nome servizio di rete. Tale nome può essere qualificato con il dominio del client. Tale nome può essere qualificato con il dominio del client, opzione che è automaticamente applicata se il dominio del client è specificato.

  4. Scegliere Successivo. Viene visualizzata la pagina Protocollo.

  5. Selezionare il protocollo di ascolto per il quale è configurato il listener. Tale protocollo deve essere installato anche sul client.

  6. Scegliere Successivo. Viene visualizzata la pagina Impostazioni protocollo.

  7. Immettere i parametri di protocollo appropriati per il protocollo scelto negli appositi campi.

  8. Scegliere Successivo. Viene visualizzata la pagina Servizio.

  9. Selezionare una release, immettere un servizio di destinazione e, facoltativamente, selezionare un tipo di connessione di database dalla lista Tipo di connessione per Oracle8i o successivo.

    Se il servizio di destinazione è Oracle8i o successivo, selezionare Oracle8i o successivo, immettere un nome di servizio nel campo Nome servizio e scegliere Successivo. Se il servizio di destinazione è Oracle8 o Oracle7, selezionare Oracle8 o precedente, immettere un SID (ID di sistema) Oracle nel campo SID database e scegliere Successivo.

    La Oracle Corporation consiglia di utilizzare l'impostazione predefinita Impostazione predefinita database per il tipo di connessione. Se nel file dei parametri di inizializzazione è stato configurato un server condiviso, è possibile selezionare Server dedicato per indurre il listener a generare dinamicamente un server dedicato, ignorando la configurazione del server condiviso. Se sul server è abilitato il connection pooling residente del database, è possibile selezionare Server in pool per ottenere una connessione dal pool. Se nel file dei parametri di inizializzazione è stato configurato un server condiviso e si desidera garantire che la connessione utilizzi sempre un server condiviso, selezionare Server condiviso.

  10. Scegliere Successivo. Viene visualizzata la pagina Test.

  11. Scegliere Test per verificare il funzionamento del nome di servizio di rete oppure scegliere Fine per salvare la configurazione e uscire dalla Creazione guidata dei nomi di servizi di rete.

    Se si sceglie Test, Oracle Net esegue la connessione al database server utilizzando le informazioni del descrittore di connessione configurato. Il test potrà riuscire soltanto se il database e il listener sono in esecuzione. In caso contrario, per avviare i componenti vedere il capitolo Establishing a Connection and Testing the Network del manuale Oracle10i Net Administrator's Guide. Se il test ha esisto positivo, viene visualizzato il seguente messaggio:

    Test della connessione completato.

    Se il test ha dato esito positivo, scegliere Chiudi per uscire dalla finestra di dialogo Test di connessione e continuare con il passo 12.

    Se l'esito del test non è stato positivo, effettuare le operazioni riportate di seguito.

    1. Accertarsi che il database e il listener siano in esecuzione e scegliere Test.

    2. Scegliere Cambia login per modificare il nome utente e la password di connessione, quindi scegliere Test.

  12. Scegliere Fine per salvare la configurazione e uscire dalla Creazione guidata dei nomi dei servizi di rete.

  13. Accertarsi che il profilo rifletta il metodo locale (LOCAL) o di denominazione delle directory (LDAP).
