Configurando Banners para Acesso Não Autorizado e Auditando Ações do Usuário

Você deve criar e configurar banners para advertir os usuários sobre o acesso não autorizado e a possível auditoria das ações do usuário. Os avisos estão disponíveis para a aplicação do cliente quando ela efetuar log-in no banco de dados.

Para configurar estes banners para exibição, defina os parâmetros do sqlnet.ora no servidor do banco de dados para que o arquivo de texto que contém as informações do banner seja apontado.

  • SEC_USER_UNAUTHORIZED_ACCESS_BANNER. Por exemplo: SEC_USER_UNAUTHORIZED_ACCESS_BANNER = /opt/Oracle/11g/dbs/unauthaccess.txt

  • SEC_USER_AUDIT_ACTION_BANNER. Por exemplo: SEC_USER_AUDIT_ACTION_BANNER = /opt/Oracle/11g/dbs/auditactions.txt

Por default, estes parâmetros não estão definidos.

Após a definição destes parâmetros, a aplicação do Oracle Call Interface deve usar as APIs do OCI adequadas para a recuperação destes banners e apresentá-los ao usuário final.
