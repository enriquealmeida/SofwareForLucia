Panoramica sulla connettività di Oracle Net

Oracle Net consente di stabilire connessioni a diversi servizi, ad esempio i database Oracle, i database non Oracle, i gateway e le procedure esterne.

Gli utenti avviano una richiesta di connessione inserendo un nome utente e una password con un identificativo per il servizio a cui desiderano connettersi. L'identificativo, denominato identificativo di connessione, specifica:

  • il servizio di destinazione;

  • il percorso o l'instradamento di rete per quel servizio.

È possibile specificare un identificativo di connessione in diversi modi. Uno dei più comuni è l'utilizzo di un nome di servizio di rete, ovvero un altro nome per il servizio mappato a un descrittore di connessione. Un descrittore di connessione contiene le informazioni sul servizio di destinazione e l'instradamento di rete. Il servizio desiderato è indicato tramite il nome del servizio stesso per i database Oracle9i o Oracle8i oppure tramite il SID (ID di sistema) Oracle per i database Oracle8 e Oracle7. L'instradamento di rete fornisce almeno la posizione del listener mediante l'uso di un indirizzo di rete.

Di seguito viene riportato un nome di servizio di rete denominato sales mappato a un descrittore di connessione:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-server1)(PORT=1521))

(CONNECT_DATA=(SERVICE_NAME=sales.us.example.com))

Il listener, che si trova su sales1-server, effettua le operazioni di ascolto tramite il protocollo TCP/IP sulla porta 1521 per le richieste di connessione per un servizio di database denominato sales.us.example.com.

L'identificativo di connessione e il descrittore di connessione corrispondente sono memorizzati in almeno un metodo di denominazione. Durante una richiesta di connessione, il client utilizza un metodo di denominazione per risolvere l'identificativo di connessione (sales nell'esempio precedente) in un descrittore di connessione. Quindi, il client contatta il listener specificato nel descrittore di connessione.

Il listener accetta la connessione del client tramite un protocollo. Confronta le informazioni client con quelle ricevute dal servizio di database attraverso la registrazione del servizio, così come le informazioni memorizzate nel suo stesso file di configurazione, vale a dire listener.ora. Se le informazioni corrispondono, la connessione viene assegnata.
