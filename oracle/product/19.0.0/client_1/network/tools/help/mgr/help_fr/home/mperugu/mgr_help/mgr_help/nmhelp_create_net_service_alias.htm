Boîte de dialogue Création d'un alias de service réseau

La boîte de dialogue Création d'un alias de service réseau permet de créer un alias de service réseau dans un serveur d'annuaire.

Un alias de service réseau est un autre nom désignant un nom de service réseau ou un service de base de données. Une entrée d'alias de service réseau ne comporte pas d'informations de descripteur de connexion. Il ne référence que l'emplacement de l'objet dont il est l'alias. Lorsqu'un client demande une recherche d'alias de service réseau, l'annuaire détermine que l'entrée est un alias de service réseau et effectue la recherche comme s'il s'agissait de l'entrée référencée. Par exemple, un alias de service réseau db1alias peut être créé pour le service de base de données db1 . Si db1alias est utilisé pour la connexion à un service de base de données, comme dans CONNECT scott/tiger@db1alias, il est converti et ce sont les informations de descripteur de connexion de db1 qui sont utilisées.

Il existe deux utilisations principales des alias de service réseau :

  • Utilisation d'un alias de service réseau par les clients pour faire référence à un service de base de données ou à un nom de service réseau sous un autre nom.

  • Utilisation d'un alias de service réseau dans un contexte Oracle pour un service de base de données ou un nom de service réseau dans un contexte Oracle différent. Ceci permet qu'un service de base de données ou un nom de service réseau soit défini une seule fois dans le serveur d'annuaire, et référencé par les clients qui utilisent d'autres contextes Oracle.

Alias de service réseau

Saisissez l'alias du service réseau.

Nom de service réseau ou service de base de données

Utilisez la zone Nom de service réseau ou de service de base de données pour entrer les informations suivantes concernant le nom de service réseau ou de service de base de données :

Contexte Oracle : indiquez le contexte Oracle correspondant au nom de service réseau ou de service de base de données en le sélectionnant dans la liste ou en le saisissant dans ce champ.

Nom : indiquez le nom distinctif (DN) correspondant au nom de service réseau ou de service de base de données en le sélectionnant dans la liste ou en le saisissant dans ce champ.
