Configuration des options de plusieurs adresses

Lorsque plusieurs adresses de protocole de processus d'écoute peuvent accéder à un service de base de données, vous devez indiquer l'ordre dans lequel ces adresses seront utilisées. Par exemple, les adresses de protocole peuvent être choisies au hasard ou essayées de manière séquentielle.

Lorsque plusieurs adresses de protocole ont été configurées pour un service de base de données ou un nom de service réseau, vous pouvez configurer les options suivantes :

  • Reprise sur délai de connexion pour choisir la première adresse de protocole, et reprendre à l'adresse de protocole suivante si la première échoue.

  • Répartition de la charge du client pour choisir une adresse de protocole au hasard.

  • Routage de la source pour utiliser toutes les adresses de protocole de manière séquentielle.

Pour configurer les options de plusieurs adresses :

  1. Dans le panneau du navigateur, développez Oracle Net Configuration > Annuaire > Local ou Résolution de noms de service.

  2. Sélectionnez un nom de service réseau ou un service de base de données. Le panneau de droite affiche le service de destination et la liste d'adresses courants.

  3. Dans la case Configuration des adresses, choisissez Avancé. La boîte de dialogue Options des listes d'adresses apparaît.

  4. Désactivez Utiliser des options compatibles avec les clients Net8 8.0 si cette configuration porte sur un client Oracle9 i ou Oracle8 i.

  5. Sélectionnez l'option appropriée à la liste d'adresses de protocole, puis cliquez sur OK.

  6. Dans le panneau de droite, choisissez Appliquer.

  7. Si vous effectuez ces modifications dans le dossier Local, choisissez Fichier > Enregistrer la configuration réseau.
