Identificazione del database tramite il nome di servizio

Un database Oracle viene rappresentato come un servizio per i client, nel senso che esegue delle attività per loro conto. A un database possono essere associati uno o più servizi.

Fino alla release Oracle8i, un servizio del database Oracle veniva identificato in modo univoco mediante il SID (ID di sistema) Oracle. Lo stesso SID veniva utilizzato internamente dal database come puntatore all'area SGA (System Global Area). I client si connettevano a un'istanza del database specificando il SID nel descrittore di connessione. Questo schema di denominazione non distingueva i servizi dalle istanze.

Poiché un database Oracle può estendersi su più computer, in Oracle9i e Oracle8i vengono specificati sia il servizio complessivo che le singole istanze che lo compongono.

Nome servizio

Un database è identificato dal nome di servizio. Un database viene specificato dal parametro SERVICE_NAMES nel file dei parametri di inizializzazione. SERVICE_NAMES specifica il nome della vista di livello più alto del servizio del database Oracle, che può coinvolgere più istanze e/o nodi. Per impostazione predefinita, SERVICE_NAMES è impostato sul nome del database globale, un nome costituito dai nomi del database e del dominio.

Nome istanza

Le istanze di database sono identificate da un nome di istanza specificato con il parametro INSTANCE_NAME nel file di inizializzazione dei parametri. INSTANCE_NAME corrisponde al SID dell'istanza.

Descrittori di connessione

Configurare i descrittori di connessione con il parametro SERVICE_NAME (senza la S) per stabilire una connessione a un database Oracle9i o Oracle8i. Ad esempio, il descrittore di connessione riportato di seguito contiene l'indirizzo di un listener nella posizione sales1-sun che ascolta le richieste di connessione per un servizio di database denominato sales.us.example.com:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)))

È anche possibile configurare i descrittori di connessione con il parametro INSTANCE_NAME per stabilire la connessione a una determinata istanza del database. Questa operazione potrebbe rivelarsi necessaria nel caso in cui si disponga di un sistema Oracle9i Real Application Clusters con più istanze.

Ad esempio, il descrittore di connessione riportato di seguito contiene l'indirizzo di un listener nella posizione sales-sun1, che ascolta le richieste di connessione per un'istanza denominata sales1 associata al database sales.us.acme.com:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)

(INSTANCE_NAME=sales1)))

Per informazioni su come specificare un nome di istanza nella sezione CONNECT_DATA di un descrittore di connessione, vedere il capitolo 13, Enabling Advanced Features for Oracle Net Services, nel manuale Oracle10i Net Services Administrator's Guide.
