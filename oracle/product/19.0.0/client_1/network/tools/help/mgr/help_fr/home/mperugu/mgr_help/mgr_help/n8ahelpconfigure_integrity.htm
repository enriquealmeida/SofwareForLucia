Configuration de l'intégrité des données

Le processus de vérification de l'intégrité des données consiste à calculer une quantité en fonction du contenu d'un message. L'expéditeur du message calcule le checksum. La valeur du checksum est incluse dans le paquet du message. A la réception, le destinataire du message recalcule le checksum du message. Ces deux totaux doivent correspondre pour que le message soit considéré comme n'ayant pas été modifié pendant la transmission.

Pour configurer l'intégrité des données :

  1. Dans le panneau du navigateur, développez Oracle Net Configuration > Local > Profil.

  2. Dans la liste du panneau de droite, sélectionnez Oracle Advanced Security.

  3. Sélectionnez l'onglet Intégrité.

  4. Activez l'intégrité des données sur le client ou le serveur en cochant la case correspondante.

  5. Indiquez comment le client et le serveur négocieront l'intégrité des données. Vous devez sélectionner Niveau de checksum pour le client et pour le serveur. Les niveaux sont décrits ci-après :

    Elément Description
    accepté Le service n'est actif que si l'autre machine participant à la connexion indique "requis" ou "demandé", et qu'elle dispose d'un algorithme compatible.
    rejeté Le service ne doit pas être actif : la connexion échoue si l'autre machine indique "requis".
    demandé Le service n'est actif que si l'autre machine participant à la connexion indique "accepté", "requis" ou "demandé", et qu'elle dispose d'un algorithme compatible.
    requis Le service doit être actif et la connexion échoue si l'autre machine indique "rejeté" ou ne dispose pas d'un algorithme compatible.

  6. Cliquez sur Fichier > Enregistrer la configuration réseau.
