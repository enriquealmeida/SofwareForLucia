Oracle Net 連線簡介

Oracle Net 可以連線至不同的服務，例如 Oracle 資料庫、非 Oracle 資料庫、閘道以及外部程序。

使用者只要傳送使用者名稱、密碼與及要連線之服務的 ID，即可起始連線要求。該 ID (稱為連線 ID) 可識別：

  • 目的地服務

  • 透過網路以到達該服務的路徑 (或稱網路路由)

您可以用多種不同的方式指定連線 ID。其中一個最常用的方法是透過使用網路服務名稱，也就是對應至連線描述區的另一個服務名稱。連線描述區內含目的地服務和網路路由資訊。指示想要的服務時，在 Oracle9i 或 Oracle8i 資料庫是使用其服務名稱來指示，在 Oracle8 或 Oracle7 資料庫則是使用「Oracle 系統 ID (SID)」來指示。網路路由至少需透過使用網路位址來提供監聽器的位置。

以下顯示對應至連線描述區，名為 sales 的網路服務名稱：

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-server1)(PORT=1521))

(CONNECT_DATA=(SERVICE_NAME=sales.us.example.com))

監聽器位於 sales1-server 上，並在連接埠 1521 上使用 TCP/IP 協定監聽 sales.us.example.com 資料庫服務的連線要求。

連線 ID 與連線描述區至少儲存在一個命名方法中。連線要求期間，從屬端會透過命名方法，將連線 ID (前一個範例中的 sales) 解析成連線描述區。然後，從屬端會連繫指定在連線描述區中的監聽器。

監聽器是透過協定來接受從屬端連線。它會比較從屬端資訊與透過服務註冊從資料庫服務接收的資訊，以及儲存在其本身組態檔 listener.ora 中的資訊。如果資訊相符，即允許連線。
