Dialogfeld "Adresslistenoptionen"

Im Dialogfeld "Adresslistenoptionen" können Sie die Reihenfolge und Verwendung der Protokolladressen in Verbindungsdeskriptoren bei Verbindungen angeben.

Nehmen Sie den Versuch folgerichtig mit jeder Adresse vor, bis ein Versuch erfolgreich ist

Wählen Sie diese Option, um Failover bei der Anmeldung für einen Oracle9i- oder Oracle8i-Client zu aktivieren. Wenn auf die erste Adresse nicht zugegriffen werden kann, kann der Client mit Failover die nächste Listener-Protokolladresse verwenden. Wählen Sie diese Option und die Option "Mit Net8 8.0-Clients kompatible Optionen verwenden'", um das Source-Routing für einen Release 8.0-Client oder früher auszuschalten.

Nehmen Sie den Versuch beliebig mit jeder Adresse vor, bis ein Versuch erfolgreich ist

Wählen Sie diese Option, um das Client-Load Balancing und das Failover bei Verbindung für einen Oracle9i oder Oracle8i Client zu aktivieren.

Nehmen Sie den Versuch mit einer beliebig ausgewählten Adresse vor

Wählen Sie diese Option, um die Funktion für das Client-Load Balancing für einen Oracle9i oder Oracle8i Client zu aktivieren.

Verwenden Sie jede Adresse in Folge, bis ein Ziel erreicht ist

Wählen Sie diese Option, um das Source Routing für einen Oracle8i oder Oracle8-Client zu aktivieren.

Diese Option ist erforderlich, wenn das Erreichen des Ziels mehr als einen Adress-Stopp erfordert. Diese Funktion wird im Allgemeinen verwendet, um die Funktionen von Oracle Connection Manager zu aktivieren.

Dieser Parameter ist bei einem Oracle9i Client nicht erforderlich.

Nur erste Adresse verwenden

Wählen Sie diese Option, um Source-Routing, Failover bei Verbindung und Client-Load Balancing zu deaktivieren.

Mit Net8 8.0 Clients kompatible Optionen verwenden

Wählen Sie diese Option, wenn Sie einen Client Release 8.0 konfigurieren. Sie können nur unter folgenden Optionen auswählen:

-- Nehmen Sie den Versuch folgerichtig mit jeder Adresse vor, bis ein Versuch erfolgreich ist

Verwenden Sie jede Adresse in Folge, bis ein Ziel erreicht ist

Löschen Sie diese Option, wenn Sie einen Oracle9i- oder Oracle8i-Client konfigurieren.
