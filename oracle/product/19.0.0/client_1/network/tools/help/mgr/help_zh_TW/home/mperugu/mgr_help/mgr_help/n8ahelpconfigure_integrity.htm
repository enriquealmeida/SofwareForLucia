設定資料完整性

資料完整性檢查是指根據訊息的內容計算數量的處理作業。訊息寄件者會計算總和檢查。總和檢查的值會包含在訊息封包中。訊息收件者則在收件時重新計算總和檢查。這二個總和檢查值必須相符，才表示訊息傳輸時未遭到更改。

設定資料完整性：

  1. 在瀏覽窗格中，展開 Oracle Net 組態 > 區域 > 設定檔。

  2. 從右方窗格中的清單選取 Oracle Advanced Security。

  3. 選取完整性頁籤。

  4. 藉由分別選擇「從屬端」或「伺服器」核取方塊來啟動從屬端或伺服器資料完整性。

  5. 指定從屬端和伺服器交涉資料完整性的方式。從屬端和伺服器都必須選取總和檢查層次。總和檢查層次描述如下：

    元素 描述
    接受 如果連線的另一端為「必要」或「要求」，並且也有可用的相容演算法，則服務會在作用中；否則，服務將會是非作用中。
    拒絕 服務必定不是作用中，如果另一端為「必要」，連線將會失敗。
    要求 如果連線的另一端指定為「接受」、「必要」或「要求」，並且也有可用的相容演算法，則服務會在作用中；否則，服務將為非作用中。
    必要 服務必須為作用中，如果另一端為「拒絕」，或者沒有可用的相容演算法，連線將會失敗。

  6. 按一下檔案 > 儲存網路組態。
