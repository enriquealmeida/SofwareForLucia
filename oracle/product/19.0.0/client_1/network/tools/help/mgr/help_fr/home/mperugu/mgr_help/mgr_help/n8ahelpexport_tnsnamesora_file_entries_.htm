Export d'entrées de fichier tnsnames.ora vers un serveur d'annuaire

Les noms de service réseau déjà configurés dans un fichier tnsnames.ora peuvent être exportés vers un serveur d'annuaire. L'export est effectué pour un domaine à la fois.

L'export d'un fichier tnsnames.ora crée un nom de service réseau dans le contexte Oracle sélectionné dans le répertoire. Si vous devez modifier le contexte Oracle, choisissez Commande > Serveur d'annuaire > Modifier le contexte Oracle.

Pour exporter des noms de service réseau stockés dans un fichier tnsnames.ora file vers un serveur de répertoires :

  1. Choisissez Commande > Annuaire > Exporter des noms de service. L'assistant Migration d'annuaire démarre.

  2. Cliquez sur Suivant.

    Si plusieurs domaines sont détectés, la page Sélectionner le domaine s'affiche. Passez à l'étape 3.

    Si les noms de service réseau ne sont pas qualifiés d'un domaine, la page Noms de service réseau apparaît. Passez à l'étape 4.

  3. Sélectionnez le domaine de réseau dont vous souhaitez exporter les noms de service réseau, puis cliquez sur Suivant. La page Sélectionner des noms de service réseau apparaît.

  4. Sélectionnez dans la liste les noms de service réseau à exporter, puis cliquez sur Suivant. La page Sélectionner un contexte de destination apparaît.

    Procédez alors comme suit :

    a. Dans la liste Contexte de résolution de noms d'annuaire, sélectionnez le contexte de résolution de noms d'annuaire contenant le contexte Oracle. Le contexte de résolution de noms d'annuaire fait partie d'un sous-arbre qui comprend un ou plusieurs contextes Oracle contenant des entrées Oracle.

    b. Dans la liste Contexte Oracle, sélectionnez le contexte Oracle dans lequel vous souhaitez exporter les noms de service réseau sélectionnés.

    c. Cliquez sur Suivant. La page Mise à jour du serveur d'annuaire apparaît et indique le statut de l'export.

  5. Choisissez Fin pour quitter l'assistant Migration du serveur d'annuaire.
