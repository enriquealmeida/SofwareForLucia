Visão Geral da Nomeação Local

A nomeação local é um dos principais métodos de nomeação usados para resolver um nome simples, um nome de serviço de rede, como um descritor de conexão, que o cliente utiliza para se conectar a um banco de dados ou serviço. A nomeação local armazena os nomes de serviços de rede e seus descritores de conexão em um arquivo tnsnames.ora local.

Um usuário final informa uma string de conexão que inclui o nome do serviço de rede no seguinte formato:

CONNECT nome do usuário/senha@nome_do_serviço de_rede

No exemplo a seguir, sales é o nome do serviço de rede pesquisado no arquivo tnsnames.ora:

CONNECT scott/tiger@sales

O descritor de conexão retornado ao cliente inclui:

  • A localização do endereço de protocolo do listener

  • O nome de serviço do banco de dados

O cliente utiliza essas informações para se conectar a um listener. O listener, configurado com as mesmas informações de endereço, recebe a solicitação. Se o listener estiver atendendo a solicitações para o mesmo serviço de banco de dados especificado no descritor de conexão do cliente, uma conexão será concedida.

Para obter mais informações, consulte o capítulo Connectivity Concepts, no Oracle10i Net Administrator's Guide.
