수신 위치

리스너는 특정 네트워크 프로토콜 주소에서 데이터베이스에 대한 접속 요청을 수신하도록 구성됩니다. 수신 위치 탭을 사용하여 리스너가 클라이언트 접속을 대기할 수신 위치 주소를 여러 개 구성할 수 있습니다.

프로토콜

목록에서 해당 주소에 대한 프로토콜을 선택합니다. 선택한 프로토콜에 따라 해당 필드에 프로토콜 매개변수 정보를 입력합니다.

고급 표시/고급 숨기기 단추

고급 표시/고급 숨기기 토글 단추를 사용하여 리스너에 대한 I/O 버퍼 공간 제한을 지정하는 설정을 표시하거나 숨깁니다.

이러한 설정은 TCP/IP, SSL을 사용하는 TCP/IP 및 SDP 프로토콜에 의해 지원됩니다.

  • 총 전송 버퍼 크기

세션의 전송 작업에 필요한 버퍼 공간을 바이트 단위로 지정합니다.

데이터베이스 서버는 데이터를 클라이언트에 기록하기 때문에 일반적으로 이 설정은 서버측에서 설정하는 것이 좋습니다. 데이터베이스 서버가 다량의 요청을 수신할 경우 총 수신 버퍼 크기 설정도 설정합니다.

  • 총 수신 버퍼 크기

세션의 수신 작업에 필요한 버퍼 공간을 바이트 단위로 지정합니다.

JServer 접속을 위해 이 주소를 정적으로 전용화

Oracle JServer 릴리스 8.1 접속에 대한 GIOP(General Inter-Orb Protocol) 끝점을 가진 listener.ora 파일을 구성하도록 선택합니다.

주소 추가

새 프로토콜 주소를 생성합니다. 프로토콜을 선택하고 필드에 프로토콜 정보를 입력할 수 있는 새 주소 탭이 나타납니다.

주소 제거

선택한 주소 탭과 해당 주소 정보를 삭제합니다.

I/O 버퍼 공간 구성에 대한 자세한 내용은 Oracle10i Net Administrator's Guide의 Optimizing Performance 장을 참조하십시오.
