Pool de connexions résidant en base de données

Le pool de connexions résidant en base de données permet de regrouper en pool les connexions dédiées de divers processus ou applications client. Une connexion dédiée désigne un serveur dédié et la session de base de données correspondante. Utilisez le pool de connexions résidant en base de données quand plusieurs applications client (comme des niveaux intermédiaires (middle tier)) doivent rester connectées en permanence à la base de données ou lorsque des ressources de serveur (telles que la mémoire) doivent être optimisées.

Dans le pool de connexions résidant en base de données, les clients sont connectés à un processus en arrière-plan appelé broker de connexion, qui assure la fonctionnalité de regroupement en pool.
