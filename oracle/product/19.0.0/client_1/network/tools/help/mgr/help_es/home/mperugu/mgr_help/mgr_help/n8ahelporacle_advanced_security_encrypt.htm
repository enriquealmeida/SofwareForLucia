Oracle Advanced Security: Cifrado

El separador Cifrado permite configurar parámetros de cifrado para los métodos de autenticación seleccionados.

Nota: Utilice el separador SSL si desea configurar Series Cipher para SSL (Secure Sockets Layer).

Cifrado

En la lista, seleccione la computadora que esté configurando: cliente o servidor.

Tipo de Cifrado

Seleccione un valor para especificar el comportamiento del cliente o servidor al negociar el cifrado y la integridad. A continuación, se describen los cuatro valores posibles:

Elemento Descripción
aceptado El servicio estará activo si el otro lado de la conexión especifica "necesario" o "solicitado" y en el otro lado está disponible un algoritmo compatible. De lo contrario, estará inactivo.
rechazado El servicio no debe estar activo y la conexión fracasará si el otro lado especifica "necesario".
solicitado El servicio estará activo si el otro lado de la conexión especifica "aceptado", "necesario" o "solicitado" y en el otro lado está disponible un algoritmo compatible.
necesario El servicio debe estar activo y la conexión fracasará si el otro lado especifica "rechazado" o si no existe un algoritmo compatible en el otro lado.

EncryptionSeed

Escriba un cifrado inicial, una cadena aleatoria con una longitud máxima de 256 caracteres en el campo Cifrado Inicial. El cifrado inicial se utiliza para generar claves de cifrado. Es un dato necesario cuando se activan las opciones de cifrado o de integridad.

Nota: Si opta por utilizar caracteres especiales, como por ejemplo, una coma [,] o un paréntesis derecho [)] como parte del parámetro Cifrado Inicial, Oracle recomienda que el valor esté entre comillas simples.

Lista Métodos Disponibles/Métodos Seleccionados

Muestra los métodos de autenticación disponibles para la selección y los métodos de cifrado seleccionados. El orden de los métodos en la lista determina el orden preferido para la negociación.

Utilice el botón de flecha a la izquierda (<) y a la derecha (>) para mover un algoritmo de cifrado seleccionado de una lista a la otra. Oracle Advanced Security soporta los siguientes algoritmos de cifrado:

Elemento Descripción
AES256 Clave AES de 256 bits
RC4_256 Clave RC4 de 256 bits
AES192 Clave AES de 192 bits
3DES168 Opción DES triple con clave triple (168 bits)
AES128 Clave AES de 128 bits
RC4_128 Clave RC4 de 128 bits
3DES112 Opción DES triple con clave doble (112 bits)
DES Clave DES de 56 bits
RC4_40 Clave RC4 de 40 bits
DES40 Clave DES de 40 bits
