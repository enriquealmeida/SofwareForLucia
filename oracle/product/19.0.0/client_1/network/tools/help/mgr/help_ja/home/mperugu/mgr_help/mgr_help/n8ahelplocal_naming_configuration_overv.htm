ローカル・ネーミング構成の概要

このセクションでは、次のトピックについて説明します。

「管理者」または「ランタイム」インストール

「管理者」または「ランタイム」のクライアント・インストールでは、ディレクトリ使用が構成されていない場合、Oracle Net Configuration Assistantによりtnsnames.oraファイル内にネット・サービス名を構成するように求めるプロンプトが表示されます。

「カスタム」インストール

「カスタム」のクライアントまたはサーバー・インストール中に、ネーミング・メソッドとしてローカル・ネーミングを選択した場合は、tnsnames.oraファイルにネット・サービス名を構成するように求めるプロンプトが表示されます。

Oracle Net Managerを使用した構成

Oracle Net Managerでは、ネット・サービス名の構成のみではなく、次の作業も実行できます。

  • Oracle以外のデータベース・サービスのネット・サービス名の作成

  • 複数のプロトコル・アドレスを通るリスナーへの複数ルートの構成

  • データベース・サービスまたはネット・サービス名の詳細CONNECT_DATA情報の構成

ローカル・ネーミング構成にアクセスする手順は、次のとおりです。

ナビゲータ・ペインから「ローカル」→「サービス・ネーミング」フォルダを選択し、ネット・サービス名を作成するか接続記述子情報を変更します。

詳細は、Oracle10i Net管理者ガイドの「ネーミング・メソッドの構成」の章を参照してください。
