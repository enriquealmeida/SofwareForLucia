Identificação de Banco de Dados por Nome de Serviço

Um banco de dados Oracle é representado para os clientes como um serviço, isto é, o banco de dados realiza o trabalho para os clientes. Um banco de dados pode ter um ou mais serviços associados a ele.

Antes do Oracle8i, um serviço de banco de dados Oracle era identificado de forma exclusiva por um Oracle SID (System Identifier). O SID também era usado internamente pelo banco de dados como um ponteiro para a SGA (System Global Area). Os clientes eram conectados a uma instância do banco de dados por meio da especificação do SID no descritor de conexão. Esse esquema de nomeação não distinguia os serviços das instâncias.

Já que um banco de dados Oracle pode abranger vários computadores, o serviço (como um todo) e cada uma de suas instâncias são especificados no Oracle9i e Oracle8i.

Nome do Serviço

Um banco de dados é agora identificado pelo seu nome do serviço. O nome de serviço é especificado pelo parâmetro SERVICE_NAMES no arquivo de parâmetros de inicialização. SERVICE_NAMES especifica o nome da view de nível mais alto do serviço de banco de dados Oracle, que pode abranger instâncias e/ou nós. O default de SERVICE_NAMES é o nome do banco de dados global, um nome composto pelo nome do banco de dados e pelo nome do domínio.

Nome da Instância

As instâncias do banco de dados são identificadas por um nome de instância com o parâmetro INSTANCE_NAME no arquivo de parâmetros de inicialização. INSTANCE_NAME corresponde ao SID da instância.

Descritores de Conexão

Configure descritores de conexão com o parâmetro SERVICE_NAME (sem S) para estabelecer conexão com um banco de dados Oracle9i ou Oracle8i. Por exemplo, o descritor de conexão a seguir contém o endereço de um listener localizado em sales1-sun que atende a solicitações de conexão de um serviço de banco de dados denominado sales.us.example.com:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)))

Opcionalmente, os descritores de conexão podem ser configurados com o parâmetro INSTANCE_NAME para estabelecer conexão com uma instância específica do banco de dados. Isso poderá ser necessário se você tiver um Oracle9i Real Application Clusters com várias instâncias.

Por exemplo, o descritor de conexão a seguir contém o endereço de um listener localizado em sales-sun1 que está atendendo a solicitações de conexão para uma instância denominada sales1, associada ao banco de dados sales.us.example.com:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)

(INSTANCE_NAME=sales1)))

Para obter informações sobre como especificar um nome de instância na seção CONNECT_DATA de um descritor de conexão, consulte o Capítulo 13, Enabling Advanced Features para Oracle Net Services, no Oracle10i Net Services Administrator's Guide.
