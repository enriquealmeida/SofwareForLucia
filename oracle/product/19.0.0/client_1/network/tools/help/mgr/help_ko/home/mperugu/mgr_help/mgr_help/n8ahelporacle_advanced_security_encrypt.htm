Oracle Advanced Security: 암호화

암호화 탭에서는 선택한 인증 방법에 대한 암호화 매개변수를 구성할 수 있습니다.

주: SSL 탭을 사용하여 SSL(Secure Sockets Layer)에 대한 암호화 모음을 구성합니다.

암호화

현재 구성 중인 컴퓨터가 클라이언트인지 또는 서버인지를 목록에서 선택합니다.

암호화 유형

암호화와 무결성을 조정할 때 클라이언트 또는 서버 동작을 지정하는 값을 선택합니다. 다음 4가지 암호화 유형을 사용할 수 있습니다.

요소 설명
승인됨 상대편 접속이 "필요함" 또는 "요청됨"을 지정하고 사용 가능한 호환 알고리즘이 있으면 서비스를 사용할 수 있습니다. 그렇지 않으면 서비스를 사용할 수 없습니다.
거부됨 서비스를 사용할 수 없으며 상대편 접속이 "필요함"을 지정해도 접속에 실패합니다.
요청됨 상대편 접속이 "승인됨", "필요함", "요청됨" 중 하나를 지정하고 사용 가능한 호환 알고리즘이 있으면 서비스를 사용할 수 있습니다. 그렇지 않으면 서비스를 사용할 수 없습니다.
필요함 서비스를 사용할 수 있으며, 상대편 접속이 "거부됨"을 지정하거나 호환 알고리즘이 없으면 접속에 실패하게 됩니다.

암호화 시드

최대 256자의 임의 문자열인 암호화 시드를 암호화 시드 필드에 입력합니다. 비밀번호화 시드는 비밀번호 키를 생성하는 데 사용되며 비밀번호화 및 무결성을 사용할 때 필요합니다.

주: 콤마[,] 또는 오른쪽 괄호[)]와 같은 특수 문자를 암호화 시드 매개변수의 일부로 사용하는 경우 Oracle은 값을 작은 따옴표로 묶을 것을 권장합니다.

사용 가능한 방법/선택된 방법 목록

선택 가능한 인증 방법과 선택된 암호화 방법을 표시합니다. 목록에 나열된 순서에 따라 조정의 우선 순위가 결정됩니다.

왼쪽 화살표(<) 단추와 오른쪽 화살표(>) 단추를 사용하여 선택된 암호화 알고리즘을 목록 사이에서 이동합니다. 다음은 Oracle Advanced Security에서 지원하는 암호화 알고리즘입니다.

요소 설명
AES256 AES 256비트 키
RC4_256 RC4 256비트 키
AES192 AES 192비트 키
3DES168 3키(168비트)를 사용하는 Triple DES 옵션
AES128 AES 128비트 키
RC4_128 RC4 128비트 키
3DES112 2키(112비트)를 사용하는 Triple DES 옵션
DES DES 56비트 키
RC4_40 RC4 40비트 키
DES40 DES 40비트 키
