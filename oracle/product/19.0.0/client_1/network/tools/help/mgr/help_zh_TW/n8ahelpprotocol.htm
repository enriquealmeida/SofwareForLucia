協定

一組定義如何在網路傳輸資料的規則。Oracle 在 Oracle Net Manager 支援的協定包括：

IPC

跨處理作業通訊 (IPC) 協定是用於與監聽器處於相同電腦的從屬端應用程式。IPC 可以提供比 TCP/IP 更快速的區域連線。

Named Pipes

Named Pipes 主要用在未使用 TCP/IP 的個人電腦上。Named Pipes 只能在 Windows NT 上使用。

TCP/IP

Transmission Control Protocol/Internet Protocol (TCP/IP) 是從屬端/資料庫進行網路通訊時所使用的實際標準 Ethernet 協定。TCP/IP 可以讓從屬端的 Oracle 應用程式透過 TCP/IP 與遠端 Oracle 資料庫進行通訊 (如果 Oracle 資料庫是在支援使用 TCP/IP 網路通訊的主機系統上執行)。

TCP/IP 是標準的網路協定。UNIX 與個人電腦網路全面使用此協定。它也是網際網路連線時所使用的協定。

TCP/IP 與 SSL

TCP/IP 與 Secure Sockets Layer (SSL) 協定可以讓從屬端上的 Oracle 應用程式透過 TCP/IP 與 SSL 與遠端 Oracle 資料庫通訊 (如果 Oracle 資料庫在支援使用 TCP/IP 與 SSL 網路通訊的主機系統上執行)。TCP/IP 與 SSL 僅用於 SSL 認證方法。

注意：您的作業系統可能支援其他的協定。
