데이터베이스 링크

Oracle 데이터베이스 서버에서 다른 데이터베이스로 전달되는 단방향 통신 경로를 정의하는 포인터입니다. 링크 포인터는 실제로 데이터 딕셔너리 테이블에 있는 항목으로 정의됩니다. 링크에 액세스하려면 데이터 딕셔너리 항목을 포함하는 로컬 데이터베이스에 접속되어 있어야 합니다.

로컬 데이터베이스 A에 접속된 클라이언트는 데이터베이스 A에 저장된 링크를 사용하여 원격 데이터베이스 B에 있는 정보에 액세스할 수 있지만 데이터베이스 B에 접속된 사용자는 동일한 링크를 사용하여 데이터베이스 A에 있는 데이터에 액세스할 수 없다는 점에서 데이터베이스 링크 접속은 단방향입니다. 데이터베이스 B의 로컬 사용자가 데이터베이스 A에 있는 데이터에 액세스하려는 경우 데이터베이스 B의 데이터 딕셔너리에 저장된 링크를 정의해야 합니다.

다음 데이터베이스 링크 유형이 지원됩니다.

  • 데이터베이스 특정 스키마의 전용 데이터베이스 링크 전용 데이터베이스 링크의 소유자만 이 링크를 사용할 수 있습니다.

  • 데이터베이스에 대한 공용 데이터베이스 링크 데이터베이스의 모든 사용자가 사용할 수 있습니다.

전용 및 공용 데이터베이스 링크를 생성하는 방법은 Oracle10i Database Administrator's Guide를 참조하십시오.
