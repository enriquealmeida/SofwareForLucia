目錄命名簡介

目錄命名是主要的命名方法之一，用來將連線 ID 解析為從屬端連線至資料庫時所使用的連線描述區 。目錄命名會將連線 ID 及其連線描述區儲存在 LDAP 相容的目錄伺服器中。連線 ID 的儲存位置，是在內含與 Oracle 軟體搭配使用之項目的 Oracle Context 子樹狀結構底下。

一般使用者以下列格式輸入包括連線 ID 的連線字串：

CONNERCT username/password@connect_identifier

在以下的範例中，sales 即是目錄中查詢的連線 ID：

CONNECT scott/tiger@sales

傳回從屬端的連線描述區包括：

  • 監聽器的協定位址位置

  • 資料庫的服務名稱

從屬端使用協定位址資訊來連線至監聽器。而使用相同位址資訊設定的監聽器會接收到連線要求。如果監聽器監聽到與連線描述區中指定之資料庫服務相同的要求，就會授權連線。

若要使用目錄命名，請設定 Oracle Net Services 功能的目錄伺服器，並設定從屬端存取該目錄。

如需詳細資訊，請參閱 Oracle10i Net Administrator's Guide 中的 Configuration Management Concepts 章節。
