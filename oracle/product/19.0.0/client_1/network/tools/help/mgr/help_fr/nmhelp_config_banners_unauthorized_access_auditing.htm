Configuration des bannières d'accès non autorisé et d'actions utilisateur d'audit

Vous devez créer et configurer des bannières afin d'avertir les utilisateurs d'un accès non autorisé et des possibes actions utilisateur d'audit. Ces avis sont mis à la disposition de l'application client lorsqu'elle se connecte à la base de données.

Pour configurer les bannières à afficher, définissez les paramètres sqlnet.ora suivants au niveau du serveur de base de données afin de pointer vers un fichier texte qui contient des informations sur les bannières :

  • SEC_USER_UNAUTHORIZED_ACCESS_BANNER. Par exemple : SEC_USER_UNAUTHORIZED_ACCESS_BANNER = /opt/Oracle/11g/dbs/unauthaccess.txt

  • SEC_USER_AUDIT_ACTION_BANNER. Par exemple : SEC_USER_AUDIT_ACTION_BANNER = /opt/Oracle/11g/dbs/auditactions.txt

Par défaut, ces paramètres ne sont pas définis.

Une fois que vous avez défini ces paramètres, l'application Oracle Call Interface doit utiliser les API OCI appropriés pour extraire ces bannières et les présenter à l'utiliser final.
