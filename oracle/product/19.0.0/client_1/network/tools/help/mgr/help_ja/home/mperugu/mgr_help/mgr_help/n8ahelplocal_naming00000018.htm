ローカル・ネーミングの概要

ローカル・ネーミングは、単純名であるネット・サービス名を、クライアントでデータベースまたはサービスへの接続に使用する接続記述子に変換するための、主要なネーミング・メソッドの1つです。ローカル・ネーミングでは、ネット・サービス名とその接続記述子がローカルのtnsnames.oraファイルに格納されます。

エンド・ユーザーは、次の形式でネット・サービス名を含む接続文字列を入力します。

CONNECT username/password@net_service_name

次の例では、salesがtnsnames.oraファイル内で参照されるネット・サービス名です。

CONNECT scott/tiger@sales

クライアントに戻される接続記述子には、次のものがあります。

  • リスナーのプロトコル・アドレス位置

  • データベースのサービス名

クライアントは、この情報を使用してリスナーに接続します。これと同じアドレス情報で構成されたリスナーが、リクエストを受信します。リスナーが、クライアントの接続記述子に指定されたデータベース・サービスと同じデータベース・サービスへのリクエストをリスニングする場合に、接続が許可されます。

詳細は、Oracle10i Net管理者ガイドの「接続性の概念」の章を参照してください。
