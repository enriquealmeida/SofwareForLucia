ディレクトリ・ネーミングの概要

ディレクトリ・ネーミングは、接続識別子を、クライアントでデータベースへの接続に使用される接続記述子に変換するための、主要なネーミング・メソッドの1つです。ディレクトリ・ネーミングでは、接続識別子とその接続記述子は、LDAP準拠のディレクトリ・サーバーに格納されます。接続識別子は、Oracleソフトウェアで使用するためのエントリを含むOracleコンテキストのサブツリーに格納されます。

エンド・ユーザーは、次の形式で接続識別子を含む接続文字列を入力します。

CONNERCT username/password@connect_identifier

次の例では、salesがディレクトリ内で参照される接続識別子です。

CONNECT scott/tiger@sales

クライアントに戻される接続記述子には、次のものがあります。

  • リスナーのプロトコル・アドレス位置

  • データベースのサービス名

クライアントは、プロトコル・アドレス情報を使用してリスナーに接続します。これと同じアドレス情報で構成されたリスナーが、リクエストを受信します。リスナーが、接続記述子で指定されているのと同じデータベース・サービスに対するリクエストをリスニングしている場合は、接続権限が付与されます。

ディレクトリ・ネーミングを有効にするためには、ディレクトリ・サーバーをOracle Net Services用に構成し、クライアントをディレクトリにアクセスできるように構成します。

詳細は、Oracle10i Net管理者ガイドの「構成管理の概念」の章を参照してください。
