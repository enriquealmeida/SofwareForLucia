Creación de un Nombre de Servicio de Red

Nota: Sólo los usuarios que son miembros de los grupos OracleNetAdmins u OracleContextAdmins pueden crear entradas de nombre de servicio de red en un directorio.

Para agregar nombres de servicio de red a un archivo tnsnames.ora o a un servidor de directorios:

  1. En el panel del navegador, amplíe Configuración de Red de Oracle > Directorio o Local > Nomenclatura de Servicios.

  2. Haga clic en el signo más (+) en la barra de herramientas o seleccione Editar > Crear de la barra de menús. Aparece la página de bienvenida del Asistente de Nombre de Servicio de Red.

  3. En el campo Nombre de Servicio de Red, introduzca un nombre. Puede que el nombre del servicio de red se cualifique con el dominio del cliente. El nombre del servicio de red se cualifica automáticamente si se ha especificado el dominio del cliente.

  4. Haga clic en Siguiente. Aparece la página Protocolo.

  5. Seleccione el protocolo en el que se ha configurado el listener. Tenga en cuenta que este protocolo debe estar instalado también en el cliente.

  6. Haga clic en Siguiente. Aparece la página Valores del Protocolo.

  7. Introduzca los parámetros de protocolo correspondientes al protocolo seleccionado en los campos proporcionados.

  8. Haga clic en Siguiente. Aparece la página Servicio.

  9. Seleccione una versión, introduzca un servicio de destino y, si lo desea, seleccione un tipo de conexión a la base de datos de la lista Tipo de Conexión a Oracle8i o Posterior.

    Si el servicio de destino es Oracle8i o posterior, seleccione Oracle8i o posterior, introduzca un nombre de servicio en el campo Nombre de Servicio y haga clic en Siguiente. Si el servicio de destino es Oracle8 u Oracle7, haga clic en Oracle8 o anterior, introduzca un identificador del sistema Oracle en el campo SID de Base de Datos y haga clic en Siguiente.

    Oracle Corporation recomienda que utilice el valor por defecto de Valor por Defecto de la Base de Datos para el tipo de conexión. Si el servidor compartido está configurado en el archivo de parámetros de inicialización, puede seleccionar Servidor Dedicado para hacer que el listener origine un servidor dedicado, ignorando la configuración de servidor compartido. Si la opción Pool de Conexiones Residentes de la Base de Datos está activada en el servidor, puede seleccionar Servidor de Pool para obtener una conexión desde el pool. Si el servidor compartido está configurado en el archivo de parámetros de inicialización y desea asegurarse de que esta conexión siempre utiliza un servidor compartido, seleccione Servidor Compartido.

  10. Haga clic en Siguiente. Aparece la página Prueba.

  11. Haga clic en Probar para comprobar que el nombre de servicio de red funciona o Terminar para guardar la configuración y salir del Asistente de Nombre de Servicio de Red.

    Si hace clic en Probar, Red de Oracle se conecta al servidor de la base de datos utilizando la información del descriptor de conexión configurado. Por lo tanto, la base de datos y el listener deben estar en ejecución para que la prueba sea correcta. Si no están en ejecución, consulte el capítulo sobre establecimiento de una conexión y comprobación de la red, en Oracle10i Net Administrator's Guide (Guía del Administrador de Red de Oracle10i) para iniciarlos. Si la prueba se realiza correctamente, aparece el siguiente mensaje:

    La prueba de conexión se ha realizado correctamente.

    Si la prueba ha sido correcta, haga clic en Cerrar para salir del recuadro de diálogo Prueba de Conexión y continúe con el Paso 12.

    Si no ha sido correcta:

    1. Asegúrese de que el listener y la base de datos se están ejecutando y haga clic en Probar.

    2. Haga clic en Cambiar Conexión para cambiar el nombre de usuario y la contraseña de la conexión y, a continuación, haga clic en Probar.

  12. Haga clic en Terminar para guardar la configuración y salir del Asistente de Nombre de Servicio de Red.

  13. Asegúrese de que el perfil refleja el método de nomenclatura local (LOCAL) o de directorios (LDAP).
