서비스 식별

이 등록정보 시트를 사용하여 선택된 네트 서비스 이름이나 데이터베이스 서비스에 대한 네트워크 정보를 보거나 수정할 수 있습니다.

서비스 식별

[서비스 식별] 상자를 사용하여 CONNECT_DATA 정보를 보거나 수정할 수 있습니다.

요소 설명
서비스 이름 Oracle8 i 이상 데이터베이스 서비스에 접속하도록 이 클라이언트를 구성 중인 경우 서비스 이름을 입력합니다. 주: 디렉토리 서버에 저장된 데이터베이스 서비스의 경우 이 필드만 현재 값을 표시합니다. 이 값은 수정할 수 없습니다.
SID Oracle8 데이터베이스에 대한 Oracle SID(시스템 식별자)를 표시합니다.
접속 유형 목록에서 이 네트 서비스 이름이 전용 서버, 공유 서버 또는 데이터베이스가 구성된 기타 서버를 사용하도록 하려면 선택합니다. 초기화 매개변수 파일에 공유 서버가 구성되어 있는 경우 리스너가 전용 서버를 시작하여 공유 서버 구성을 통과하도록 하려면 [전용 서버]를 선택합니다. 초기화 매개변수 파일에 공유 서버가 구성되어 있고 이 접속에서 항상 공유 서버를 사용하도록 하려면 [공유 서버]를 선택합니다. 서버에서 [데이터베이스 상주 접속 풀링]이 사용으로 설정된 경우 풀링 서버를 선택하여 풀에 접속할 수 있습니다.
Oracle8 릴리스 8.0 호환 ID 사용 Oracle8i 이전 버전 데이터베이스 서비스에 클라이언트를 접속하도록 구성하려는 경우 이 옵션을 선택합니다. Oracle9i 또는 Oracle8i 데이터베이스 서비스에 대해 구성하는 경우에는 이 옵션의 선택을 해제합니다.
고급 고급 CONNECT_DATA 매개변수를 구성합니다.

주소 구성

[주소] 상자를 사용하여 리스너 프로토콜 주소를 수정, 추가 또는 삭제합니다.

주소가 이미 구성되어 있으면 해당 프로토콜 정보가 필드에 나타납니다.

요소 설명
프로토콜 목록에서 리스너 주소에 대한 프로토콜을 선택합니다. 선택한 프로토콜에 따라 해당하는 정보를 필드에 입력합니다.
고급 표시/고급 숨기기 단추 고급 표시/고급 숨기기 토글 단추를 사용하여 주소에 대한 I/O 버퍼 공간 제한을 지정하는 설정을 표시하거나 숨깁니다.

이러한 설정은 TCP/IP, SSL을 사용하는 TCP/IP 및 SDP 프로토콜에 의해 지원됩니다.

  • 총 전송 버퍼 크기: 세션의 전송 작업을 위한 버퍼 공간을 바이트 단위로 지정합니다.

  • 총 수신 버퍼 크기: 세션의 수신 작업을 위한 버퍼 공간을 바이트 단위로 지정합니다.

주: 여기서 설정된 값은 클라이언트측 sqlnet.ora 파일의 해당 설정 값보다 우선 적용됩니다.
+ 단추 새 프로토콜 주소를 생성합니다. 프로토콜을 선택하고 필드에 프로토콜 정보를 입력할 수 있는 새 [주소] 탭이 나타납니다. 선택한 프로토콜은 클라이언트와 리스너가 상주하는 서버 모두에 설치되어야 합니다.
X 단추 선택된 [주소] 탭과 해당 주소 정보를 제거합니다.
< 단추 목록 순서에서 해당 프로토콜 주소를 위로 이동합니다.
> 단추 목록 순서에서 해당 프로토콜 주소를 아래로 이동합니다.
고급 단추 접속 시간 복구 및 클라이언트 로드 밸런싱과 같은 다중 주소 옵션을 구성합니다.
