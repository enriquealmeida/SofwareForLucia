General: Avanzado

El separador Avanzado permite configurar información avanzada sobre el perfil para una computadora cliente o un servidor.

Timeout de Operaciones de Envío

Permite especificar el tiempo, en segundos, que tarda un servidor de bases de datos en realizar una operación de envío a clientes a partir del establecimiento de la conexión.

Para entornos en los que los clientes cierran de vez en cuando o de forma anormal, se recomienda la definición de este parámetro. Si el servidor de bases de datos no puede terminar una operación de envío en el tiempo especificado, registrará ORA-12535:TNS:timeout de la operación y ORA-12608: TNS: Se ha producido un timeout de envío en el archivo sqlnet.log .

Sin este parámetro, el servidor de bases de datos sigue enviando respuestas a clientes que no pueden recibir datos debido a una computadora inactiva o a que está en estado ocupado.

También puede definir este parámetro en el cliente para especificar el tiempo, en segundos, que tarda un cliente en terminar las operaciones de envío al servidor de bases de datos a partir del establecimiento de la conexión. Sin este parámetro, el cliente puede seguir enviando solicitudes a un servidor de bases de datos que ya esté saturado con solicitudes.

Timeout de Operaciones de Recepción

Permite especificar el tiempo, en segundos, durante el que un servidor de bases de datos esperará los datos del cliente a partir del establecimiento de la conexión. Un cliente debe enviar datos dentro del intervalo de tiempo.

Para entornos en los que los clientes cierran de vez en cuando o de forma anormal, se recomienda la definición de este parámetro. Si un cliente no envía ningún dato en el tiempo especificado, el servidor de bases de datos registrará ORA-12535:TNS:timeout de la operación y ORA-12609: TNS: Se ha producido un timeout de recepción en el archivo sqlnet.log .

Sin este parámetro, el servidor de bases de datos sigue esperando datos de clientes que puede que estén caídos o que experimenten dificultades. También puede utilizar este valor en el cliente para especificar el tiempo, en segundos, durante el que un cliente esperará los datos de respuesta del servidor de bases de datos a partir del establecimiento de la conexión.

Sin este parámetro, puede que el cliente espere durante un largo período de tiempo una respuesta de un servidor de bases de datos saturado con solicitudes.

Tamaño Total de Buffer de Envío

Especifique el límite de espacio en buffer para operaciones de envío de sesiones. Este parámetro está soportado por los protocolos TCP/IP, TCP/IP con SSL y SDP.

Tamaño Total de Buffer de Recepción

Especifique el límite de espacio en buffer para operaciones de recepción de sesiones. Este parámetro está soportado por los protocolos TCP/IP, TCP/IP con SSL y SDP.

Valor de Timeout de TNS

Especifique el intervalo de tiempo, en minutos, para enviar una sonda para verificar que las conexiones del cliente/servidor están activas. Si se define un valor mayor que 0, se garantiza que las conexiones no permanecen abiertas indefinidamente cuando el cliente haya finalizado la conexión de forma anormal. Si la sonda encuentra una conexión terminada o que ya no está en uso, devuelve un error, que da lugar a la terminación de la conexión y la salida del proceso del servidor. Este valor está dirigido al servidor de bases de datos, que suele gestionar varias conexiones a la vez.

Las limitaciones de uso de la función de detección de conexión terminada son:

  • No está permitida en conexiones Bequeath.

  • El paquete de sonda genera un tráfico adicional, aunque muy pequeño, que puede reducir el rendimiento de la red.

  • Según el sistema operativo que utilice, puede que el servidor necesite realizar un procesamiento adicional para distinguir el evento de sonda de conexión de los demás eventos que se produzcan. También puede reducir el rendimiento de la red.

Nota: Esta opción sólo se puede utilizar en el servidor.

ID Registro de Cliente

Introduzca un identificador único para este cliente. Este identificador se transfiere al listener con cualquier solicitud de conexión.

El listener incluirá este identificador en el archivo log. El identificador puede ser cualquier cadena con una longitud máxima de 128 caracteres.

Versión del Protocolo de Autenticación de Conexión

Permite especificar los protocolos de autenticación permitidos por el cliente o la base de datos. Si el cliente y el servidor de bases de datos no tienen al menos una versión coincidente, la autenticación falla con un error.

Los valores soportados incluyen:

  • 10 para protocolos de autenticación de 10i

  • 9 para protocolos de autenticación de 9i

  • 8 para protocolos de autenticación de 8i

  • Por Defecto para protocolos de autenticación de 10i, 9i y 8i

Cualquier valor diferente a 10 podría mostrar las vulnerabilidades que han podido existir en versiones anteriores de los protocolos de autenticación. Para obtener una compatibilidad total, defina la lista de versiones permitidas para la conexión para que Por Defecto incluya todas las versiones de la base de datos en el sistema.

Desactivar Manejo de Señales de UNIX

Seleccione esta opción para configurar el cliente de forma que transfiera la eliminación de los procesos secundarios al proceso init de UNIX, desactivando los manejadores de señales.

Puesto que la aplicación cliente origina internamente un proceso de servidor, la aplicación cliente es responsable de eliminar el proceso secundario cuando finalice. Cuando el proceso del servidor finaliza sus responsabilidades de conexión, se convierte en un proceso difunto. Los manejadores de señales son los responsables de eliminar dichos procesos difuntos.

Desactivar Ruptura Fuera de Banda

Seleccione esta opción para desactivar o activar la ruptura fuera de banda.

Si no se selecciona, la Red de Oracle puede enviar y recibir mensajes de "ruptura" utilizando datos urgentes proporcionados por el protocolo subyacente.

Si se selecciona, desactiva la posibilidad de enviar y recibir mensajes de "ruptura" utilizando datos urgentes proporcionados por el protocolo subyacente. Una vez activada, esta función se aplica a todos los protocolos utilizados por este cliente.
