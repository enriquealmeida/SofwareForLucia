Posizioni di ascolto

I listener sono configurati in modo da ascoltare le richieste di connessione in entrata per un database su un indirizzo di protocollo di rete specifico. La scheda Posizioni di ascolto consente di configurare uno o più indirizzi di posizioni di ascolto sulle quali il listener attende le connessioni client.

Protocollo

Dall'apposita lista, selezionare il protocollo per l'indirizzo. Nei campi immettere i parametri appropriati in base al protocollo selezionato.

Pulsante Mostra avanzate/Nascondi avanzate

Utilizzare il pulsante Mostra avanzate/Nascondi avanzate per visualizzare o nascondere le impostazioni che consentono di specificare il limite di spazio del buffer di I/O per il listener.

Queste impostazioni sono supportate dai protocolli TCP/IP, TCP/IP con SSL e SDP.

  • Dimensione totale del buffer di invio

Specificare, in byte, lo spazio del buffer per le operazioni di invio delle sessioni.

Poiché il database server scrive i dati sui client, l'impostazione di questa opzione sul lato server in genere rappresenta la scelta appropriata. Se il database server riceve un volume elevato di richieste, impostare anche l'opzione Dimensione totale del buffer di ricezione.

  • Dimensione totale del buffer di ricezione

Specificare, in byte, lo spazio del buffer per le operazioni di ricezione delle sessioni.

Dedicare in maniera statica questo indirizzo alle connessioni JServer

Scegliere questa opzione per configurare il file listener.ora con gli endpoint GIOP (General Inter-Orb Protocol) per le connessioni relative a Oracle JServer release 8.1.

Aggiungi indirizzo

Scegliere Aggiungi indirizzo per creare un nuovo indirizzo di protocollo. Viene visualizzata una nuova scheda Indirizzo in cui è possibile selezionare il protocollo e immettere le informazioni sul protocollo negli appositi campi.

Rimuovi indirizzo

Scegliere questa opzione per eliminare la scheda Indirizzo selezionata e le relative informazioni sull'indirizzo.

Per ulteriori informazioni sulla configurazione del spazio del buffer di I/O, vedere il capitolo Optimizing Performance del manuale Oracle10i Net Administrator's Guide.
