Generale: Avanzate

La scheda Avanzate consente di configurare le informazioni avanzate sul profilo per un computer client o server.

Timeout delle operazioni di invio

Consente di specificare il tempo, in secondi, richiesto da un database server per il completamento di un'operazione di invio ai client dopo che è stata stabilita la connessione.

Per gli ambienti in cui i client vengono chiusi in modo occasionale o anomalo, si consiglia di impostare questo parametro. Se il database server non è in grado di completare un'operazione di invio nell'intervallo di tempo specificato, registra i messaggi ORA-12535:TNS: timeout dell'operazione e ORA-12608: TNS: si è verificato il timeout dell'invio nel file sqlnet.log .

Senza questo parametro, il database server continua a inviare le risposte ai client che non sono in grado di ricevere i dati a causa di un computer inattivo o di uno stato occupato.

È possibile anche impostare questo parametro sul lato client per specificare il tempo, in secondi, richiesto da un client per il completamento delle operazioni di invio al database server dopo che è stata stabilita la connessione. Senza questo parametro, è possibile che il client continui a inviare le richieste a un database server già saturo di richieste.

Timeout delle operazioni di ricezione

Consente di specificare il tempo, in secondi, per l'attesa dei dati client da parte di un database server dopo che è stata stabilita la connessione. Un client deve inviare alcuni dati nell'intervallo di tempo.

Per gli ambienti in cui i client vengono chiusi in modo occasionale o anomalo, si consiglia di impostare questo parametro. Se un client non invia alcun dato nell'intervallo di tempo specificato, il database server registra i messaggi ORA-12535:TNS: timeout dell'operazione e ORA-12609: TNS: si è verificato il timeout della ricezione nel file sqlnet.log .

Senza questo parametro, il database server continua ad attendere i dati dai client che potrebbero essere inattivi o presentare difficoltà. È possibile utilizzare questa impostazione anche sul lato client per specificare il tempo, in secondi, di un client per l'attesa dei dati di risposta dal database server dopo che è stata stabilita la connessione.

Senza questo parametro, il client potrebbe attendere molto tempo prima di ottenere una risposta da un database server saturo di richieste.

Dimensione totale del buffer di invio

Specificare il limite di spazio del buffer per le operazioni di invio delle sessioni. Questo parametro è supportato dai protocolli TCP/IP, TCP/IP con SSL e SDP.

Dimensione totale del buffer di ricezione

Specificare il limite di spazio del buffer per le operazioni di ricezione delle sessioni. Questo parametro è supportato dai protocolli TCP/IP, TCP/IP con SSL e SDP.

Valore di timeout TNS

Specificare l'intervallo di tempo, in minuti, per l'invio di una prova che consenta di verificare che le connessioni client/server siano attive. L'impostazione di un valore maggiore di 0 assicura che le connessioni non vengano lasciate aperte per un periodo di tempo indeterminato a causa di interruzioni client anomale. Se la prova trova una connessione interrotta o inutilizzata, restituisce un errore, determinando la chiusura del processo server. Questa impostazione riguarda il database server, che in genere gestisce più connessioni contemporaneamente.

I limiti relativi all'utilizzo della funzione di rilevamento della connessione interrotta sono i seguenti:

  • Non è consentita nelle connessioni di trasmissione.

  • Sebbene sia di dimensioni ridotte, un pacchetto di prova genera traffico aggiuntivo che può ridurre le prestazioni della rete.

  • A seconda del sistema operativo in uso, possono essere necessarie ulteriori elaborazioni del server per distinguere l'evento di prova della connessione dagli altri eventi che si verificano. Ciò può ulteriormente ridurre le prestazioni della rete.

Nota: questa opzione può essere utilizzata solo sul server.

ID registrazione client

Immettere un identificativo univoco per il client. L'identificativo viene inoltrato al listener con ogni richiesta di connessione.

Il listener include l'identificativo nel file di log. L'identificativo può essere una qualsiasi stringa con lunghezza massima di 128 caratteri.

Versione del protocollo di autenticazione per il collegamento

Consente di specificare i protocolli di autenticazione consentiti dal client o dal database. Se il client e il database server non hanno almeno una versione corrispondente, l'autenticazione non riesce e restituisce un errore.

I valori supportati includono i seguenti:

  • 10 per i protocolli di autenticazione 10i

  • 9 per i protocolli di autenticazione 9i

  • 8 per i protocolli di autenticazione 8i

  • Valore predefinito per i protocolli di autenticazione 10i, 9i e 8i

Qualsiasi valore diverso da 10 può esporre i punti deboli della versione precedente dei protocolli di autenticazione. Per una compatibilità completa, impostare questa lista di versioni consentite per il collegamento sul valore predefinito per includere tutte le versioni del database nel sistema.

Disabilita gestione segnale UNIX

Scegliere questa opzione per disabilitare gli handler dei segnali e configurare il client in modo che i processi secondari siano cancellati dal processo init di UNIX.

Poiché è l'applicazione client a generare dinamicamente all'interno un processo server, è sempre l'applicazione client a dover cancellare il processo secondario quando questo giunge al termine. Quando il processo server completa le funzioni di connessione di propria competenza, diventa inattivo. Gli handler dei segnali sono responsabili della cancellazione dei processi inattivi.

Disabilita interruzione fuori banda

Scegliere questa opzione per disabilitare o abilitare le interruzioni fuori banda.

Se questa opzione non è selezionata, consente a Oracle Net di inviare e ricevere i messaggi di "interruzione" utilizzando i dati urgenti forniti dal protocollo sottostante.

Se questa opzione è selezionata, non è possibile inviare e ricevere messaggi di "interruzione" che utilizzano dati urgenti forniti dal protocollo sottostante. Una volta abilitata, la funzione si applica a tutti i protocolli utilizzati dal client.
