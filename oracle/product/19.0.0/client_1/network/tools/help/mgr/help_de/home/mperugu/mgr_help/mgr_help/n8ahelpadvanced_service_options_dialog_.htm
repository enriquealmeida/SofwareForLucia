Dialogfeld "Erweiterte Serviceoptionen"

Im Dialogfeld "Erweiterte Dienstoptionen" können Sie erweiterte CONNECT_DATA- Parameter konfigurieren.

Zusätzliche Serviceeinstellungen

Element Beschreibung
Instanzname Geben Sie die Namen der Datenbankinstanz an, auf die zugegriffen werden soll. Der Instanzname kann aus dem Parameterwert INSTANCE_NAME in der Initialisierungsparameterdatei ermittelt werden.
Sessiondateneinheit (SDU) Geben Sie die Größe der Sessiondateneinheit ein, um die Leistungseigenschaften für Pakete zu ändern, die über das Netzwerk gesendet werden. Legen Sie die SDU-Größe als ein Vielfaches der normalen Transportrahmengröße fest. Die normale Größe für Ethernet-Rahmen beträgt 1500, der verwendbare Teil beträgt jedoch nur 1460. Wenn die Verschlüsselung nicht verwendet wird, wird ein Byte reserviert. Der zu verwendende Algorithmus ist ein Vielfaches von 1460 plus 1, wenn die Verschlüsselung nicht verwendet wird, oder ein Vielfaches von 1460 bei Ethernet.
Für heterogene Dienste verwenden Klicken Sie auf diese Option, damit ein Oracle-Server über Heterogene Dienste auf ein fremdes System zugreifen kann.

Oracle Rdb-Einstellungen:

Element Beschreibung
Oracle Rdb-Einstellungen: Geben Sie den Dateinamen einer Oracle Rdb-Datenbank an.
Diensttyp Geben Sie den Diensttyp für eine Oracle Rdb-Datenbank an.
Globaler Datenbankname Geben Sie einen Bezeichner für die Oracle Rdb-Datenbank an.
