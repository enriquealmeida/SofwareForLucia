추가 프로토콜 주소 생성

데이터베이스 서비스는 두 개 이상의 네트워크 경로 또는 프로토콜 주소를 통해 액세스할 수 있습니다.

기존 네트 서비스 이름 또는 데이터베이스 서비스에 네트워크 프로토콜 주소를 추가하려면 다음과 같이 하십시오.

  1. 네비게이터 창에서 Oracle Net 구성 > 디렉토리 또는 로컬 > 서비스 이름 지정을 확장합니다.

  2. 네트 서비스 이름 또는 데이터베이스 서비스를 선택합니다. 오른쪽 창에 현재 대상 서비스와 주소 목록이 표시됩니다.

  3. 주소 구성 상자에서 플러스 기호(+)를 선택합니다. 새 주소 탭이 나타납니다.

  4. 프로토콜을 선택하고 선택한 프로토콜에 해당하는 프로토콜 매개변수 정보를 입력합니다.

  5. 왼쪽 또는 오른쪽 화살표 단추를 사용하여 주소 목록에서 순서대로 주소를 정렬합니다. 다중 주소 옵션이 구성되어 있지 않으면 목록의 첫번째 주소에만 접속됩니다. 다른 순서를 구성하려면 다중 주소 옵션 구성을 참조하십시오.

  6. 오른쪽 창에서 적용을 선택합니다.

  7. [로컬] 폴더를 이렇게 변경하는 경우 파일 > 네트워크 구성 저장을 선택합니다.
