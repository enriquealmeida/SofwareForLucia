Datenbankidentifikation nach Servicename

Eine Oracle-Datenbank stellt sich Clients als ein Dienst dar, d.h. die Datenbank führt Funktionen für die Clients aus. Mit einer Datenbank können ein oder mehrere Dienste verknüpft sein.

Bis zu Oracle8i wurde ein Oracle-Datenbankdienst eindeutig mit einem Oracle System Identifier (SID) identifiziert. Der SID wurde auch intern von der Datenbank als Zeiger zu der System Global Area (SGA) verwendet. Clients haben sich bei einer Datenbankinstanz mit Angabe der SID im Verbindungsdeskriptor angemeldet. Bei diesem Benennungsschema wurden Dienste nicht von Instanzen unterschieden.

Da eine Oracle-Datenbank über mehrere Rechner hinweg verwendet werden kann, werden sowohl der Dienst als Ganzes als auch jede seiner Instanzen in Oracle9i und Oracle8i angegeben.

Servicename

Eine Datenbank wird jetzt mit ihrem Service Name identifiziert. Der Servicename wird mit dem Parameter SERVICE_NAMES in der Initialisierungsparameterdatei angegeben. SERVICE_NAMES gibt den Namen der View der höchsten Ebene des Oracle-Datenbankdienstes an, der Instanzen und/oder Knoten umfassen kann. SERVICE_NAMES entspricht standardmäßig dem globalen Datenbanknamen, einem Namen, der aus Datenbankname und Domainname zusammengesetzt ist.

Instanzname

Datenbankinstanz werden von einem Instanznamen mit dem Parameter INSTANCE_NAME in der Initialisierungsparameterdatei identifiziert. INSTANCE_NAME entspricht dem SID der Instanz.

Verbindungsdeskriptoren

Konfigurieren Sie Verbindungsdeskriptoren mit dem Parameter SERVICE_NAME (ohne S), um sich bei einer Oracle9i- oder Oracle8i Datenbank anzumelden. Beispiel: Der folgende Verbindungsdeskriptor enthält die Adresse eines Listeners auf sales1-sun , der auf Verbindungsanforderungen für einen Datenbankservice mit der Bezeichnung sales.us.example.com horcht:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)))

Optional können Verbindungsdeskriptoren auch mit dem Parameter INSTANCE_NAME konfiguriert werden, wenn die Verbindung zu einer bestimmten Instanz der Datenbank hergestellt werden soll. Dies ist möglicherweise bei Oracle9i Real Application Clusters mit mehreren Instanzen erforderlich.

Beispiel: Der folgende Verbindungsdeskriptor enthält die Adresse eines Listeners auf sales-sun1, der auf Verbindungsanforderungen für eine Instanz mit der Bezeichnung sales1 hört, die mit der Datenbank sales.us.example.com verknüpft ist:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)

(INSTANCE_NAME=sales1)))

Weitere Informationen über die Angabe eines Instanznamens in dem Abschnitt CONNECT_DATA eines Verbindungsdeskriptors finden Sie in Kapitel 13 "Enabling Advanced Features for Oracle Net Services" im Oracle10i Net Services Administrator's Guide.
