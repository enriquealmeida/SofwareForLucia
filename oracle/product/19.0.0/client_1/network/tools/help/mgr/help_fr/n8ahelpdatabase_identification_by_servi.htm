Identification de la base de données par un nom de service

Une base de données Oracle constitue un service pour les clients, c'est-à-dire qu'elle effectue des tâches pour ces derniers. Un ou plusieurs services peuvent lui être associés.

Jusqu'à Oracle8i, un service de base de données Oracle était identifié de façon unique par un identificateur système (SID) Oracle. Le SID était également utilisé en interne par la base de données comme pointeur de la zone SGA (System Global Area). Pour se connecter à une instance de base de données, les clients indiquaient le SID dans le descripteur de connexion. Ce schéma ne faisait pas la différence entre les services et les instances.

Une base de données Oracle pouvant être répartie sur plusieurs ordinateurs, le service dans son ensemble et chacune de ses instances sont définis dans Oracle9i et Oracle8i.

Nom de service

Une base de données est désormais identifiée par son nom de service. Celui-ci est indiqué par le paramètre SERVICE_NAMES dans le fichier de paramètres d'initialisation. Le paramètre SERVICE_NAMES permet d'indiquer le nom de la vue de niveau supérieur du service de base de données Oracle pouvant intégrer des instances et/ou des noeuds. Ce paramètre prend par défaut le nom global de base de données, c'est-à-dire un nom composé du nom de la base de données et du nom du domaine.

Nom d'instance

Les instances de base de données sont identifiées par un nom d'instance via le paramètre INSTANCE_NAME dans le fichier de paramètres d'initialisation. Le paramètre INSTANCE_NAME correspond au SID de l'instance.

Descripteurs de connexion

Configurez les descripteurs de connexion à l'aide du paramètre SERVICE_NAME (sans S) pour assurer la connexion à une base de données Oracle9i ou Oracle8i. Par exemple, le descripteur de connexion ci-dessous contient l'adresse d'un processus d'écoute situé sur sales1-sun et écoutant les demandes de connexion pour un service de base de données appelé sales.us.example.com :

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)))

Les descripteurs de connexion peuvent éventuellement être configurés via le paramètre INSTANCE_NAME pour assurer une connexion à une instance spécifique de la base de données. Ceci peut être nécessaire si vous disposez d'Oracle9i Real Application Clusters avec instances multiples.

Par exemple, le descripteur de connexion ci-dessous contient l'adresse d'un processus d'écoute, situé sur sales-sun1 qui écoute les demandes de connexion pour une instance sales1 associée à la base de données sales.us.example.com :

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-sun1)(PORT=1521))

(CONNECT_DATA=

(SERVICE_NAME=sales.us.example.com)

(INSTANCE_NAME=sales1)))

Pour plus d'informations sur la spécification d'un nom d'instance dans la section CONNECT_DATA d'un descripteur de connexion, reportez-vous au chapitre 13, Enabling Advanced Features for Oracle Net Services, dans le Guide de l'administrateur Oracle10i Net Services.
