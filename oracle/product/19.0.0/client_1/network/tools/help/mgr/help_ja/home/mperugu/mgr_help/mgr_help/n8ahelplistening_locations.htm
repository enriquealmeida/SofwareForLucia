リスニング位置

リスナーは、特定のネットワーク・プロトコル・アドレス上のデータベースに対する着信接続リクエストをリスニングするように構成されます。「リスニング位置」タブを使用して、リスナーがクライアント接続を待つリスニング位置の1つ以上のアドレスを構成します。

プロトコル

リストから、このアドレス用のプロトコルを選択します。選択したプロトコルごとに、各フィールドに適切なプロトコル・パラメータ情報を入力します。

「詳細の表示/詳細の非表示」ボタン

「詳細の表示/詳細の非表示」ボタンを使用して、リスナーのI/Oバッファ容量制限を指定するための設定を表示または非表示にします。

これらの設定は、TCP/IP、SSL付きTCP/IPおよびSDPプロトコルでサポートされます。

  • 合計送信バッファ・サイズ

セッションの送信操作のためのバッファ領域をバイト単位で指定します。

データベース・サーバーはクライアントにデータを書き込むため、通常は、これをサーバー側で設定すれば十分です。データベース・サーバーが大量のリクエストを受信する場合は、「合計受信バッファ・サイズ」も設定します。

  • 合計受信バッファ・サイズ

セッションの受信操作のためのバッファ領域をバイト単位で指定します。

JServer接続用としてこのアドレスを静的に指定します

Oracle JServerリリース8.1接続のGeneral Inter-ORB Protocol (GIOP)のエンドポイントを、listener.oraファイルに構成します。

アドレスの追加

プロトコル・アドレスを新規作成します。新しい「アドレス」タブが表示され、プロトコルを選択して各フィールドにプロトコル情報を入力できます。

アドレスの削除

選択した「アドレス」タブとそのアドレス情報を削除します。

I/Oバッファ容量の構成の詳細は、Oracle10i Net管理者ガイドの「パフォーマンスの最適化」の章を参照してください。
