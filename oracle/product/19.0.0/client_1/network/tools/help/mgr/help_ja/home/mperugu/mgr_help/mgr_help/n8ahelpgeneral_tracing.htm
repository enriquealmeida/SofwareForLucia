一般: トレース

「トレース」タブでは、このコンピュータのトレース・パラメータを設定できます。このコンピュータがクライアントの場合は、「クライアント情報」フィールドを使用します。このコンピュータがサーバーの場合は、「サーバー情報」フィールドを使用します。

クライアント・トレースでは、クライアントが実行する操作が追跡されます。サーバー・トレースでは、ネットワーク接続の間にサーバーが実行する操作が追跡されます。

トレース・レベル

リストから、トレース・レベルを選択します。

OFF: トレースが使用禁止になります。デフォルトは「オフ」です。

USER: トレースがユーザーに適したレベルに設定されます。ユーザーが発生させたエラー状態を識別します。

ADMIN: トレースがデータベース管理者に適したレベルに設定されます。インストール固有の問題を識別します。

SUPPORT: トレースがカスタマ・サポートに適したレベルに設定されます。

トレース・ファイルがかなり大きくなる場合があります。ネットワーク上の問題のトラブルシューティングを行わないときには、トレースをオフにしておくことをお薦めします。

トレース・ディレクトリ

トレース・ファイルを書き込むディレクトリを入力します。クライアントとサーバーのトレース・ファイルには、クライアントまたはサーバーに関する情報が含まれますが、両方の情報がローカル・コンピュータに書き込まれます。デフォルトは、UNIXの場合は$ORACLE_ HOME/network/trace、Windowsの場合はORACLE_HOME\network\traceになります。

トレース・ファイル

トレース・ファイル名を入力します。デフォルト・ファイル名は、クライアントの場合はsqlnet.trc、サーバーの場合はsvr_pid.trcです。

一意のトレース・ファイル名

各トレース・ファイルの一意名を作成します。複数のトレース・ファイルが共存できるように、各トレース・ファイル名にはプロセス識別子が追加されます。一意のトレース・ファイルを作成せず、すべてのトレース情報を同じトレース・ファイルに格納する場合は、このオプションを選択しないでください。
