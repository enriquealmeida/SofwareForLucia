리스너 주소 구성

리스너는 하나 이상의 프로토콜 주소로 구성되어야 합니다. 리스너는 동일한 주소로 구성된 클라이언트에서 연결 요청을 받습니다.

리스너에 대한 프로토콜 주소를 구성하려면 다음과 같이 하십시오.

  1. 네비게이터 창에서 Oracle Net 구성 > 로컬> 리스너를 확장합니다.

  2. 리스너를 선택합니다.

  3. 오른쪽 창의 목록에서 수신 위치를 선택합니다.

  4. 주소 추가를 선택합니다. 새 주소 탭이 나타납니다.

  5. 프로토콜을 선택하고 선택한 프로토콜에 해당하는 프로토콜 매개변수 정보를 입력합니다.

    Oracle Net 클라이언트에 대해 TCP/IP로 수신하도록 리스너를 구성하려면 기본 포트 또는 1521을 선택해야 합니다. 그렇지 않은 경우 초기화 매개변수 파일의 LOCAL_LISTENER 매개변수를 구성하고 이름 지정 방법을 통해 리스너 이름을 분석해야 합니다.

  6. 컴퓨터에 IP 주소가 두 개 이상 있고 리스너가 사용 가능한 모든 IP 주소에서 수신되도록 하려면 TCP/IP 또는 SSL을 사용하는 TCP/IP를 구성한 후 호스트 필드에 컴퓨터의 호스트 이름을 입력합니다.

  7. 필요에 따라 고급 표시를 눌러 세션의 전송 및 수신 작업에 대한 I/O 버퍼 공간 설정을 지정할 수도 있습니다.

  8. 파일 > 네트워크 구성 저장을 선택합니다.

기본값이 아닌 포트 구성에 대한 자세한 내용은 Oracle10i Net Administrator's Guide의 Configuring and Administering the Listener 장을 참조하십시오.

I/O 버퍼 공간 구성에 대한 자세한 내용은 Oracle10i Net Administrator's Guide의 Optimizing Performance 장을 참조하십시오.
