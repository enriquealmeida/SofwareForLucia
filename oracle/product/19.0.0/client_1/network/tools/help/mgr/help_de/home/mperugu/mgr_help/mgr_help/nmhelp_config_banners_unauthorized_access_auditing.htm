Banner für nicht autorisierten Zugriff und Auditing-Benutzeraktionen konfigurieren

Sie sollten Banner erstellen und konfigurieren, die Benutzer vor nicht autorisiertem Zugriff und möglichem Auditing von Benutzeraktionen warnen. Die Hinweise sind für die Clientanwendung bei der Anmeldung bei der Datenbank verfügbar.

Um die Anzeige dieser Banner zu konfigurieren, legen Sie die folgenden sqlnet.ora-Parameter auf Datenbankserverseite so fest, dass sie auf eine Textdatei verweisen, die die Bannerinformationen enthält:

  • SEC_USER_UNAUTHORIZED_ACCESS_BANNER. Beispiel: SEC_USER_UNAUTHORIZED_ACCESS_BANNER = /opt/Oracle/11g/dbs/unauthaccess.txt

  • SEC_USER_AUDIT_ACTION_BANNER. Beispiel: SEC_USER_AUDIT_ACTION_BANNER = /opt/Oracle/11g/dbs/auditactions.txt

Standardmäßig sind diese Parameter nicht gesetzt.

Nachdem Sie diese Parameter gesetzt haben, muss die Oracle Call Interface-Anwendung die entsprechenden OCI-APIs verwenden, um diese Banner abzurufen und für den Endbenutzer anzuzeigen.
