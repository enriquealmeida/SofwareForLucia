一般：追蹤

您可以使用追蹤頁籤來設定電腦的追蹤參數。如果此電腦是從屬端，則使用從屬端資訊欄位。如果此電腦是伺服器，則使用伺服器資訊欄位。

從屬端追蹤會追蹤從屬端執行的作業。伺服器追蹤會追蹤伺服器在網路連線時執行的作業。

追蹤層次

從清單中選取追蹤層次：

OFF -- 未啟用追蹤。OFF 是預設值。

USER -- 將追蹤設定成適用於使用者的層次。透過追蹤來識別使用者導致的錯誤條件。

ADMIN -- 將追蹤設定成適用於資料庫管理員的層次。追蹤來識別與安裝有關的問題。

SUPPORT -- 將追蹤設定成適用於客戶支援的層次。

追蹤檔可能會變得非常大。Oracle Corporation 建議您在未進行網路問題疑難排解時，將追蹤關閉。

追蹤目錄

輸入追蹤檔會寫入的目錄。雖然從屬端與伺服器追蹤檔記錄從屬端或伺服器的相關資訊，但是二者會寫入區域電腦。預設值是$ORACLE_ HOME/network/trace(UNIX) 以及ORACLE_HOME\network\trace(Windows 平台)。

追蹤檔

輸入追蹤檔名稱。預設檔案名稱是 sqlnet.trc (從屬端) 以及 svr_pid.trc (伺服器)。

唯一的追蹤檔名稱

選擇此選項可為每一個追蹤檔建立唯一名稱。處理作業 ID 會附加至每一個追蹤檔的名稱，因此數個追蹤檔可以共存。如果不想建立唯一追蹤檔而且您想要將所有追蹤資訊儲存至相同的追蹤檔，請清除此選項。
