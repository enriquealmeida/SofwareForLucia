Création d'un nom de service réseau

Remarque : seuls les utilisateurs faisant partie du groupe OracleNetAdmins ou OracleContextAdmins peuvent créer des entrées de nom de service réseau dans un annuaire.

Pour ajouter des noms de service réseau à un fichier tnsnames.ora ou à un serveur d'annuaire :

  1. Dans le panneau du navigateur, développez Oracle Net Configuration > Annuaire ou Local > Résolution de noms de service.

  2. Sélectionnez plus (+) dans la barre d'outils ou choisissez Modifier > Créer dans la barre de menus. La page Bienvenue de l'assistant Nom de service réseau apparaît.

  3. Entrez un nom dans le champ Nom de service réseau. Le nom de service réseau peut être qualifié par le domaine du client. Le nom de service réseau est automatiquement qualifié par un domaine si le domaine client est indiqué.

  4. Cliquez sur Suivant. La page Protocole apparaît.

  5. Sélectionnez le protocole pour lequel le processus d'écoute est configuré. Veuillez noter que ce protocole doit également être installé sur le client.

  6. Cliquez sur Suivant. La page Paramètres de protocole apparaît.

  7. Indiquez les paramètres de protocole correspondant au protocole choisi dans les champs appropriés.

  8. Cliquez sur Suivant. La page Service apparaît.

  9. Sélectionnez une version, entrez le service de destination et sélectionnez, si vous le souhaitez, un type de connexion de base de données dans la liste Type de connexion Oracle8i ou version ultérieure.

    Si le service de destination est Oracle8i ou supérieur, sélectionnez Oracle8i ou supérieur, entrez un nom de service dans le champ Nom de service, puis sélectionnez Suivant. Si le service de destination est Oracle8 ou Oracle7, sélectionnez Oracle8 ou version antérieure, entrez un identificateur système Oracle dans le champ SID de base de données, puis cliquez sur Suivant.

    Oracle recommande d'utiliser la valeur par défaut de la base de données pour le type de connexion. Si le serveur partagé est configuré dans le fichier de paramètres d'initialisation, vous pouvez sélectionner Serveur dédié pour forcer le processus d'écoute à générer dynamiquement un serveur dédié, en passant outre à la configuration en serveur partagé. Si le pool de connexions résidant en base de données est activé sur le serveur, vous pouvez sélectionner Serveur en pool pour obtenir une connexion du pool. Si un serveur partagé est configuré dans le fichier de paramètres d'initialisation et que vous voulez que la connexion fasse toujours appel à un serveur partagé, sélectionnez Serveur partagé.

  10. Cliquez sur Suivant. La page Test apparaît.

  11. Choisissez Tester pour vérifier le fonctionnement du nom de service réseau, ou bien Fin pour enregistrer votre configuration et fermer l'assistant Nom de service réseau.

    Si vous choisissez Tester, Oracle Net se connecte au serveur de base de données en utilisant les informations du descripteur de connexion que vous avez configurées. Par conséquent, la base de données et le processus d'écoute doivent être en cours d'exécution pour que le test réussisse. S'ils ne le sont pas, reportez-vous au chapitre Establishing a Connection and Testing the Network dans le Guide de l'administrateur Oracle10i Net pour démarrer les composants. Si le test réussit, le message suivant apparaît :

    Le test a réussi.

    Dans ce cas, choisissez Fermer pour quitter la boîte de dialogue de test de connexion, puis passez à l'étape 12.

    Si le test échoue :

    1. Assurez-vous que la base de données et le processus d'écoute sont en cours d'exécution, puis choisissez Tester.

    2. Sélectionnez Changer de connexion pour modifier le nom utilisateur et le mot de passe de la connexion, puis Tester.

  12. Choisissez Fin pour enregistrer votre configuration et quitter l'assistant Nom de service réseau.

  13. Assurez-vous que le profil reflète la méthode locale (LOCAL) ou la méthode de résolution de noms d'annuaire (LDAP).
