로컬 이름 지정

로컬 이름 지정은 간단한 이름인 네트 서비스 이름을 클라이언트가 데이터베이스나 서비스에 접속하기 위해 사용하는 접속 기술자로 분석할 때 사용하는 기본 이름 지정 방법입니다. 로컬 이름 지정은 네트 서비스 이름과 해당 접속 기술자를 로컬 tnsnames.ora 파일에 저장합니다.

일반 사용자는 네트 서비스 이름을 포함한 접속 문자열을 다음 형식으로 입력합니다.

CONNECT username/password@net_service_name

다음 예에서 sales는 tnsnames.ora 파일에서 조회되는 네트 서비스 이름입니다.

CONNECT scott/tiger@sales

접속 기술자가 다음 사항을 클라이언트로 반환합니다.

  • 리스너의 프로토콜 주소 위치

  • 데이터베이스 서비스 이름

클라이언트는 이 정보를 사용하여 리스너에 접속합니다. 동일한 주소 정보로 구성된 리스너는 요청을 수신합니다. 리스너가 클라이언트 접속 기술자에 지정된 데이터베이스 서비스 요청을 수신하면 접속을 허용합니다.
