네트 서비스 별칭 생성

주:

  • OracleNetAdmins 또는 OracleContextAdmins 그룹의 구성원인 사용자만 디렉토리에 네트 서비스 별칭 항목을 생성할 수 있습니다.

  • Microsoft Active Directory에서는 네트 서비스 별칭이 지원되지 않습니다.

  • Oracle Names Server에서 기존 별칭 객체를 익스포트할 수 있습니다 . 자세한 내용은 Oracle10i Net Services Administrator's Guide의 Exporting Naming Data to a Directory Server 장을 참조하십시오.

디렉토리 이름 지정 객체(네트 서비스 이름 또는 데이터베이스 서비스)에 대해 네트 서비스 별칭을 디렉토리 서버에 생성할 수 있습니다. 네트 서비스 별칭은 참조하는 객체의 이름을 저장합니다.

클라이언트가 네트 서비스 별칭의 디렉토리 조회를 요청하면 디렉토리는 항목이 네트 서비스 별칭이고 별칭이 참조된 항목인 것처럼 조회를 완료하는지 확인합니다. 예를 들어, 네트 서비스 이름인 sales에 네트 서비스 별칭 salesalias를 생성할 수 있습니다. CONNECT scott/tiger@salesalias처럼 salesalias가 데이터베이스 서비스에 접속하는 데 사용되는 경우 실제로 sales에 대한 접속 기술자 정보로 분석되고 사용됩니다.

네트 서비스 이름 별칭에 대한 두 가지 기본 사용:

  • 클라이언트가 데이터베이스 서비스 또는 다른 이름으로 네트 서비스 이름을 참조하기 위한 방법으로 네트 서비스 이름 별칭을 사용합니다.

  • 다른 Oracle Context에 있는 데이터베이스 서비스나 네트 서비스 이름에 대해 특정 Oracle Context에 있는 네트 서비스 별칭을 사용합니다. 이렇게 하면 데이터베이스 서비스 또는 네트 서비스 이름을 디렉토리 서버에서 한 번 정의하여 다른 Oracle Context 외부에서 사용할 수 있습니다.

네트 서비스 이름 별칭을 생성하려면 다음과 같이 하십시오.

  1. 네비게이터 창에서 Oracle Net 구성 > 디렉토리 > 서비스 이름 지정 > 별칭을 확장합니다.

  2. 도구 모음에서 플러스 기호(+)를 선택하거나 메뉴 표시줄에서 편집 > 생성을 선택합니다. 네트 서비스 별칭 생성 대화상자가 나타납니다.

  3. 네트 서비스 별칭 필드에 네트 서비스 별칭에 대한 이름을 입력합니다.

  4. [네트 서비스] 또는 [데이터베이스 서비스] 상자에 네트 서비스 이름이나 데이터베이스 서비스에 대한 다음 정보를 입력합니다.

    • Oracle Context: 목록에서 하나를 선택하거나 필드에 직접 입력하여 데이터베이스 서비스 또는 네트 서비스 이름의 Oracle Context를 지정합니다.

    • 이름: 목록에서 하나를 선택하거나 필드에 직접 입력하여 데이터베이스 서비스 또는 네트 서비스 이름의 DN(식별 이름)을 지정합니다.

  5. 생성을 선택합니다.
