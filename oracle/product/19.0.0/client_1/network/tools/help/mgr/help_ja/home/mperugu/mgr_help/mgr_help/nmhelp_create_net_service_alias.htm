ネット・サービス別名の作成ダイアログ・ボックス

「ネット・サービス別名の作成」ダイアログ・ボックスを使用すると、ディレクトリ・サーバーにネット・サービス別名(net service alias)を作成できます。

ネット・サービス別名は、ネット・サービス名またはデータベース・サービスの代替名です。ネット・サービス別名のエントリには、接続記述子の情報はありません。かわりに、別名に対するオブジェクトの位置の参照のみを行います。クライアントがネット・サービス別名のディレクトリ検索を要求すると、ディレクトリは、エントリがネット・サービス別名であることを判断し、参照エントリであるかのように検索を完了します。たとえば、db1というデータベース・サービスに対して、db1aliasというネット・サービス別名を作成できます。データベース・サービスに接続する場合に、CONNECT scott/tiger@db1aliasのようにdb1aliasを使用すると、db1の接続記述子情報が解決および使用されます。

ネット・サービス別名には次の2つの主要な使用方法があります。

  • ネット・サービス別名を使用して、クライアントに別の名前でデータベース・サービスまたはネット・サービス名を参照させます。

  • 異なるOracleコンテキストのデータベース・サービスまたはネット・サービス名に対して、1つのOracleコンテキストのネット・サービス別名を付けます。このようにした場合、データベース・サービスまたはネット・サービス名をディレクトリ・サーバーに定義すると、他のOracleコンテキストからも使用できます。

ネット・サービス別名

ネット・サービスの別名を入力します。

ネット・サービス名またはデータベース・サービス

「ネット・サービス名またはデータベース・サービス」ボックスに、データベース・サービスまたはネット・サービス名についての次の情報を入力します。

Oracleコンテキスト: リストから選択するか、フィールドに入力して、データベース・サービスまたはネット・サービス名のOracleコンテキストを指定します。

名前: リストから選択するか、フィールドに入力して、データベース・サービスまたはネット・サービス名の識別名(DN)を指定します。
