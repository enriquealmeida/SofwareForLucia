Visão Geral da Conectividade do Oracle Net

O Oracle Net permite que sejam estabelecidas conexões com vários serviços, como bancos de dados Oracle, bancos de dados não Oracle, gateways e procedimentos externos.

Os usuários iniciam uma solicitação de conexão especificando um nome de usuário e uma senha com o identificador do serviço ao qual desejam se conectar. Esse identificador, denominado identificador de conexão, identifica:

  • O serviço de destino

  • O caminho na rede, ou rota de rede, desse serviço

Um identificador de conexão pode ser especificado de diversas maneiras. Uma das mais comuns consiste na utilização de um nome de serviço de rede, outro nome do serviço, que é mapeado para um descritor de conexão. Um descritor de conexão contém informações sobre o serviço de destino e a rota da rede. O serviço desejado é indicado através de seu nome de serviço para bancos de dados Oracle9i e Oracle8i ou de seu Oracle SID (System Identifier) para bancos de dados Oracle8 ou Oracle7. A rota de rede fornece, no mínimo, a localização do listener através da utilização de um endereço de rede.

É mostrado a seguir o nome do serviço de rede sales mapeado para um descritor de conexão:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-server1)(PORT=1521))

(CONNECT_DATA=(SERVICE_NAME=sales.us.example.com))

O listener, localizado em sales1-server , faz listening com o protocolo TCP/IP na porta 1521 para solicitações de conexão de um serviço de banco de dados denominado sales.us.example.com.

O identificador de conexão e o respectivo descritor de conexão são armazenados em pelo menos um método de nomeação. Durante uma solicitação de conexão, o cliente resolve o identificador de conexão (sales no exemplo anterior) para um descritor de conexão através de um método de nomeação. Em seguida, o cliente entra em contato com o listener especificado no descritor de conexão.

O listener, através de um protocolo, aceita a conexão do cliente. Ele compara as informações do cliente com aquelas que recebeu do serviço de banco de dados através do registro de serviço e também com as informações que armazenou em seu próprio arquivo de configuração, o listener.ora. Se as informações coincidirem, uma conexão será concedida.
