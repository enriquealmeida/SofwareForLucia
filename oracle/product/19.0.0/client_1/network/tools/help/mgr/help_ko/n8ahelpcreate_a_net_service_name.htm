네트 서비스 이름 생성

주: OracleNetAdmins 또는 OracleContextAdmins 그룹의 구성원인 사용자만 디렉토리에 네트 서비스 이름 항목을 생성할 수 있습니다.

네트 서비스 이름을 tnsnames.ora 파일 또는 디렉토리 서버에 추가하려면 다음과 같이 하십시오.

  1. 네비게이터 창에서 Oracle Net 구성 > 디렉토리 또는 로컬 > 서비스 이름 지정을 확장합니다.

  2. 도구 모음에서 플러스 기호(+)를 선택하거나 메뉴 표시줄에서 편집 > 생성을 선택합니다. 네트 서비스 이름 마법사의 시작 페이지가 나타납니다.

  3. 네트 서비스 이름 필드에 이름을 입력합니다. 네트 서비스 이름은 클라이언트 도메인으로 한정될 수 있습니다. 클라이언트 도메인이 지정되어 있는 경우 네트 서비스 이름은 자동으로 도메인으로 한정됩니다.

  4. 다음을 선택합니다. [프로토콜] 페이지가 나타납니다.

  5. 리스너가 수신하도록 구성된 프로토콜을 선택합니다. 이 프로토콜도 클라이언트에 설치되어 있어야 합니다.

  6. 다음을 선택합니다. [프로토콜 설정] 페이지가 나타납니다.

  7. 선택한 프로토콜에 해당하는 프로토콜 매개변수를 제공된 필드에 입력합니다.

  8. [다음]을 선택합니다. [서비스] 페이지가 나타납니다.

  9. 릴리스를 선택하고 대상 서비스를 입력한 다음 필요에 따라 [Oracle8i 또는 이후 버전 접속 유형] 목록에서 데이터베이스 접속 유형을 선택합니다.

    대상 서비스가 Oracle8i 이상인 경우 Oracle8i 또는 이후 버전을 선택하고 서비스 이름 필드에 서비스 이름을 입력한 후 다음을 선택합니다. 대상 서비스가 Oracle8 또는 Oracle7인 경우 Oracle8 또는 이전을 선택하고 데이터베이스 SID 필드에 Oracle 시스템 식별자를 입력한 후 다음을 선택합니다.

    Oracle은 접속 유형에 데이터베이스 기본값의 기본 설정을 사용할 것을 권장합니다. 초기화 매개변수 파일에 공유 서버가 구성되어 있는 경우 전용 서버를 선택하여 리스너가 공유 서버 구성을 우회하여 전용 서버를 생성하도록 할 수 있습니다. 서버에서 [데이터베이스 상주 접속 풀링]이 사용 가능으로 설정된 경우 풀링 서버를 선택하여 풀에서 접속할 수 있습니다. 초기화 매개변수 파일에 공유 서버가 구성되어 있고 접속 시 항상 공유 서버를 사용하려는 경우 공유 서버를 선택합니다.

  10. 다음을 선택합니다. [테스트] 페이지가 나타납니다.

  11. 테스트를 선택하여 네트 서비스 이름이 작동하는지 확인하거나 [완료]를 선택하여 구성을 저장하고 네트 서비스 이름 마법사를 종료합니다.

    테스트를 선택하면 구성된 접속 기술자 정보를 사용하여 Oracle Net가 데이터베이스 서버에 접속됩니다. 따라서 테스트를 성공하려면 데이터베이스와 리스너가 실행 중이어야 합니다. 그렇지 않은 경우 Oracle10i Net Administrator's Guide의 Establishing a Connection and Testing the Network 장을 참조하여 구성 요소를 시작하십시오. 테스트를 성공하면 다음 메시지가 나타납니다.

    접속 테스트를 성공했습니다.

    테스트를 성공하면 닫기를 선택하여 [접속 테스트] 대화상자를 종료한 다음 단계 12로 진행합니다.

    테스트를 실패하면 다음을 수행합니다.

    1. 데이터베이스와 리스너가 실행 중인지 확인한 다음 테스트를 선택합니다.

    2. [로그인 변경]을 선택하여 접속 사용자 이름과 비밀번호를 변경한 다음 테스트를 선택합니다.

  12. 완료를 선택하여 구성을 저장하고 네트 서비스 이름 마법사를 종료합니다.

  13. 프로파일이 로컬(LOCAL)이나 디렉토리 이름 지정(LDAP) 방법을 반영하는지 확인합니다.
