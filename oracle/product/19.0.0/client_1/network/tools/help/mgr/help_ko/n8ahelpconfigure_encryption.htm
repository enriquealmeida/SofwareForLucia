암호화 구성

암호화는 지정된 수신자 이외의 사람이 읽을 수 없도록 메시지 표기를 바꾸는 프로세스입니다.

암호화 매개변수를 구성하려면 다음과 같이 하십시오.

  1. 네비게이터 창에서 Oracle Net 구성 > 로컬 > 프로파일을 확장합니다.

  2. 오른쪽 창에 있는 목록에서 Oracle Advanced Security를 선택합니다.

  3. 암호화 탭을 선택합니다.

  4. 암호화 목록에서 클라이언트 또는 서버를 선택합니다.

  5. 암호화 및 무결성을 조정할 때 클라이언트 또는 서버 동작을 지정하려면 암호화 유형 목록에서 값을 선택합니다. 다음 4개의 값을 사용할 수 있습니다.

    요소 설명
    승인됨 상대편 접속이 "필요함" 또는 "요청됨"을 지정하고 사용 가능한 호환 알고리즘이 있으면 서비스를 사용할 수 있습니다. 그 그렇지 않으면 서비스를 사용할 수 없습니다.
    거부됨 서비스를 사용할 수 없으며 상대편 접속이 "필요함"을 지정해도 접속에 실패합니다.
    요청됨 상대편 접속이 "승인됨", "필요함", "요청됨" 중 하나를 지정하고 사용 가능한 호환 알고리즘이 있으면 서비스를 사용할 수 있습니다.
    필요함 서비스를 사용할 수 있으며, 상대편 접속이 "거부됨"을 지정하거나 호환 알고리즘이 없으면 접속에 실패하게 됩니다.

  6. [암호화 시드] 필드에 최대 256자의 임의 문자열인 암호화 시드를 입력합니다. 이 암호화 시드는 다음 단계에서 선택하는 암호화 알고리즘과 함께 사용됩니다.

  7. [사용 가능한 방법] 목록에서 비밀번호화 알고리즘을 선택한 다음 오른쪽 화살표(<) 단추를 눌러 [선택된 방법] 목록으로 옮깁니다. [선택된 방법] 목록에 나열된 순서에 따라 조정 우선 순위가 결정됩니다. 다음은 Oracle Advanced Security가 지원하는 알고리즘입니다.

    요소 설명
    RC4_256 256비트 키 크기
    3DES168 3키(168비트)를 사용하는 Triple DES 옵션
    RC4_128 128비트 키 크기
    3DES112 2키(112비트)를 사용하는 Triple DES 옵션
    RC4_56 56비트 키 크기
    DES 표준 56비트 키 크기
    RC4_40 40비트 키 크기
    DES40 40비트 키 크기

  8. 파일 > 네트워크 구성 저장을 선택합니다.

주: 모든 Oracle Advanced Security 암호화 알고리즘 및 키 길이는 미국 내에서만 사용할 수 있습니다.
