Oracle Advanced Security : Authentification

L'onglet Authentication permet de sélectionner des méthodes d'authentification et de définir leur priorité. Voici la liste des méthodes d'authentification prises en charge par Oracle Advanced Security :

KERBEROS5

Fournit des fonctions de connexion unique sécurisées dans un environnement distribué grâce à la cryptographie à clé secrète. Les mots de passe sont stockés dans un référentiel central, rendant ainsi impossible toute violation depuis le réseau. Kerberos assure également l'authentification des liens de base de données ainsi qu'une meilleure sécurité PC.

CYBERSAFE

Serveur d'authentification Kerberos qui fournit une authentification sécurisée basée sur la gestion de clés et le partage de secrets. Le Challenger dispose d'un contrôle et d'une authentification du mot de passe à l'aide des cartes de sécurité à token.

NTS

Permet d'effectuer une authentification OS (système d'exploitation) entre un client et un serveur Oracle sur Windows NT. La méthode d'authentification native de Windows NT permet d'authentifier l'utilisateur de la base de données par l'intermédiaire de l'OS. Ceci permet aux ordinateurs client d'établir des connexions sécurisées avec une base de données Oracle sur un serveur Windows NT. Une connexion est dite sécurisée lorsque le nom utilisateur du client Windows NT est récupéré sur un serveur Windows NT par l'intermédiaire de la méthode d'authentification native de l'OS. Le serveur Windows NT permet ensuite à l'utilisateur concerné d'effectuer les actions de base de données désirées sur le serveur.

RADIUS

Prend en charge toute méthode d'authentification conforme à la norme RADIUS, y compris les cartes à token et les cartes intelligentes. Un serveur RADIUS transmet les informations du serveur Oracle aux serveurs d'authentification désignés. Du point de vue de l'utilisateur, l'intégralité de l'authentification s'effectue en toute transparence.

< bouton

Choisissez ce bouton pour faire passer une méthode d'authentification de la liste des méthodes disponibles dans la liste des méthodes sélectionnées.

Bouton >

Choisissez ce bouton pour faire passer la méthode d'authentification sélectionnée de la liste des méthodes sélectionnées dans la liste des méthodes disponibles.

Bouton Promouvoir

Choisissez ce bouton pour faire remonter une méthode d'authentification dans la liste des méthodes sélectionnées. Une négociation client/serveur tentera d'utiliser les méthodes d'authentification dans l'ordre de la liste. Oracle Net utilise d'abord l'authentification du haut de la liste. S'il ne peut pas l'utiliser, il passe à la deuxième, et ainsi de suite.

Bouton Rétrograder

Choisissez ce bouton pour faire descendre une méthode d'authentification dans la liste des méthodes sélectionnées.

Remarque : Si vous avez configuré le protocole TCP/IP avec SSL, la couche SSL (Secure Sockets Layer) est configurée par défaut. La méthode d'authentification choisie dans cet onglet remplace les fonctions d'authentification de SSL. Utilisez l'onglet SSL pour configurer d'autres options de SSL. Si vous utilisez le protocole TCP/IP avec SSL, n'employez pas une autre méthode d'authentification. La liste des méthodes sélectionnées doit être vide.

Pour plus d'informations, reportez-vous à la documentation Windows NT ou au Guide de l'administrateur Oracle Advanced Security.
