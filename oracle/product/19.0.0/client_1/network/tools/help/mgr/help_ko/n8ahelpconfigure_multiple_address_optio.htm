다중 주소 옵션 구성

다중 리스너 프로토콜 주소를 통해 데이터베이스 서비스에 액세스할 수 있을 경우 프로토콜 주소가 사용되는 순서를 지정해야 합니다. 예를 들어, 프로토콜 주소를 임의의 순서로 또는 순서대로 선택할 수 있습니다.

다중 프로토콜 주소가 네트 서비스 이름이나 데이터베이스 서비스에 대해 구성되어 있을 경우 다음 옵션을 구성할 수 있습니다.

  • 접속 시간 복구 - 첫번째 프로토콜 주소를 선택하고, 첫번째 프로토콜 주소가 실패하는 경우 다음 프로토콜 주소로 복구합니다.

  • 클라이언트 로드 밸런싱 - 임의로 프로토콜 주소를 선택합니다.

  • 소스 경로 지정 - 모든 프로토콜 주소를 순서대로 사용합니다.

다중 주소 옵션을 구성하려면 다음과 같이 하십시오.

  1. 네비게이터 창에서 Oracle Net 구성 > 디렉토리 > 로컬 또는 서비스 이름 지정을 확장합니다.

  2. 네트 서비스 이름 또는 데이터베이스 서비스를 선택합니다. 오른쪽 창에 현재 대상 서비스와 주소 목록이 표시됩니다.

  3. 주소 구성 상자에서 고급을 선택합니다. 주소 목록 옵션 대화상자가 나타납니다.

  4. Oracle9i 또는 Oracle8i 클라이언트에 대한 구성일 경우 Net8 8.0 클라이언트와 호환되는 옵션 사용의 선택을 해제합니다.

  5. 프로토콜 주소 목록에 대해 적합한 옵션을 선택한 다음 확인을 선택합니다.

  6. 오른쪽 창에서 적용을 선택합니다.

  7. [로컬] 폴더를 이렇게 변경하는 경우 파일 > 네트워크 구성 저장을 선택합니다.
