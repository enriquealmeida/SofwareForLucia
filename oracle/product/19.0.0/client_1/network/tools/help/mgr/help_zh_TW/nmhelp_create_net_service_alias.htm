建立網路服務別名對話方塊

您可以使用「建立網路服務別名」對話方塊在目錄伺服器建立網路服務別名。

網路服務別名是網路服務名稱或資料庫服務的替代名稱。網路服務別名項目沒有連線描述區資訊，而是僅參照其物件的位置。當從屬端要求進行網路服務別名的目錄查詢時，目錄會判斷項目是否為網路服務別名，將其視為參照的項目完成查詢。例如，您可以為 db1 的資料庫服務建立 db1alias 網路服務別名。當使用 db1alias 來連線資料庫服務時 (例如 CONNECT scott/tiger@db1alias)，實際上會解析成 db1 的連線描述區資訊並使用該資訊來連線。

網路服務別名有兩個主要用途：

  • 使用網路服務別名作為從屬端以其他名稱參照資料庫服務或網路服務名稱的方法。

  • 在不同 Oracle Context 的資料庫服務或網路服務名稱的一個 Oracle Context 中使用網路服務別名。這樣資料庫服務或網路服務名稱在目錄伺服器定義一次，就可以被使用其他 Oracle Context 的從屬端參照。

網路服務別名

輸入網路服務別名的名稱。

網路服務名稱或資料庫服務

使用「網路服務名稱」或「資料庫服務」方塊，即可輸入資料庫服務或網路服務名稱的相關資訊：

Oracle Context：從清單選取或在欄位中輸入，以指定資料庫服務的 Oracle Context 或網路服務名稱。

名稱：從清單選取或在欄位中輸入，以指定資料庫服務的辨別名稱 (DN) 或網路服務名稱。
