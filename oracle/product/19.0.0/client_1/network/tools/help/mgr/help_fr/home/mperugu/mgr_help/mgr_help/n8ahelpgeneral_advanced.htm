Général : Avancé

L'onglet Avancé vous permet de configurer des informations de profil avancées pour un client ou un serveur.

Temporisation des opérations d'émission

Utilisez cette option pour indiquer le délai, en secondes, dont dispose un serveur de base de données pour réaliser une opération d'émission vers les clients après établissement de la connexion.

Dans les environnements où il arrive que les clients soient arrêtés, normalement ou non, il est recommandé d'utiliser ce paramètre. Lorsque le serveur de base de données n'arrive pas à réaliser une opération d'émission dans le délai imparti, il consigne les erreurs ORA-12535 : TNS :le délai imparti à l'opération est écoulé et ORA-12608 : TNS : dépassement du délai imparti pour l'émission dans le fichier sqlnet.log .

En l'absence de ce paramètre, le serveur de base de données continue à envoyer des réponses à des clients qui ne peuvent pas les recevoir car leur système est arrêté ou à l'état occupé.

Vous pouvez également indiquer ce paramètre côté client, pour définir le délai, en secondes, dont dispose un client pour réaliser une opération d'émission vers le serveur de base de données après établissement de la connexion. En l'absence de ce paramètre, le client risque de continuer à envoyer des demandes à un serveur de base de données déjà saturé.

Temporisation des opérations de réception

Utilisez ce paramètre pour indiquer le délai, en secondes, pendant lequel un serveur de base de données attend des données client après établissement de la connexion. Un client doit envoyer des données dans le délai indiqué.

Dans les environnements où il arrive que les clients soient arrêtés, normalement ou non, il est recommandé d'utiliser ce paramètre. Lorsqu'un client n'envoie pas de données dans le délai imparti, le serveur de base de données consigne les erreurs ORA-12535 : TNS :le délai imparti à l'opération est écoulé et ORA-12609 : TNS : dépassement du délai imparti pour la réception dans le fichier sqlnet.log .

En l'absence de ce paramètre, le serveur de base de données continue à attendre des données de clients qui sont peut-être arrêtés ou ont peut-être des problèmes. Vous pouvez également utiliser ce paramètre côté client pour indiquer le délai, en secondes, pendant lequel un client attend des données de réponse du serveur de base de données après établissement de la connexion.

En l'absence de ce paramètre, le client risque d'attendre longtemps une réponse d'un serveur de base de données saturé.

Taille totale de tampon d'émission

Indiquez l'espace tampon maximum pour les opérations d'émission des sessions. Ce paramètre est pris en charge par les protocoles TCP/IP, TCP/IP avec SSL et SDP.

Taille totale de tampon de réception

Indiquez l'espace tampon maximum pour les opérations de réception des sessions. Ce paramètre est pris en charge par les protocoles TCP/IP, TCP/IP avec SSL et SDP.

Valeur de temporisation TNS

Indiquez l'intervalle, en minutes, auquel procéder à des essais afin de vérifier que les connexions client/serveur sont actives. Une valeur supérieure à 0 permet de s'assurer que des connexions ne restent pas ouvertes indéfiniment en raison d'un arrêt anormal du client. Lorsque l'essai trouve une connexion arrêtée, ou une connexion qui n'est plus utilisée, il renvoie une erreur, ce qui provoque la sortie du processus serveur. Ce paramètre est prévu pour le serveur de base de données, qui gère généralement plusieurs connexions simultanément.

Les limites d'utilisation de la fonction de détection des connexions arrêtées sont les suivantes :

  • Elle n'est pas autorisée pour les connexions transmises.

  • Même très petit, un groupe d'essai génère un trafic supplémentaire qui risque de diminuer les performances du réseau.

  • Selon le système d'exploitation utilisé, le serveur devra peut-être exécuter des traitements supplémentaires pour distinguer l'événement d'essai de connexion des autres événements qui se produisent. Cela peut également réduire les performances du réseau.

Remarque : Cette option ne peut être utilisée que sur le serveur.

ID d'inscription client

Entrez un identificateur unique pour ce client. Cet identificateur est transmis au processus d'écoute avec toute demande de connexion.

Le processus d'écoute ajoute cet identificateur à son fichier de journalisation. L'identificateur est une chaîne de 128 caractères au maximum.

Version du protocole d'authentification de connexion

Utilisez ce paramètre pour indiquer les protocoles d'authentification autorisés par le client ou la base de données. Si le client et le serveur de base de données n'ont pas au moins une version en commun, l'authentification échoue avec une erreur.

Les valeurs admises sont les suivantes :

  • 10 pour les protocoles d'authentification 10i

  • 9 pour les protocoles d'authentification 9i

  • 8 pour les protocoles d'authentification 8i

  • Valeur par défaut pour les potocoles d'authentification 10i, 9i et 8i

Toute valeur autre que 10 peut révéler des vulnérabilités existant dans les versions antérieures des protocoles d'authentification. Pour une compatibilité complète, choisissez la valeur Valeur par défaut afin d'inclure dans le système toutes les versions de base de données.

Désactiver le traitement des signaux UNIX

Choisissez cette option pour que le client transmette le nettoyage des processus enfants au processus d'initialisation UNIX en désactivant les gestionnaires de signaux.

L'application client régénérant un processus serveur, elle devient responsable du nettoyage du processus enfant lorsqu'il se termine. Un processus serveur une fois acquitté de ses fonctions de connexion devient un processus défunt. Les gestionnaires de signaux sont précisément chargés de nettoyer ces processus défunts.

Désactiver l'interruption hors bande

Choisissez cette option pour désactiver ou activer les interruptions hors bande.

Si cette option n'est pas sélectionnée, Oracle Net envoie et reçoit des messages d'interruption en utilisant les données urgentes fournies par le protocole sous-jacent.

Si cette option est sélectionnée, Oracle Net désactive la fonction d'envoi et de réception de messages d'interruption en utilisant les données urgentes fournies par le protocole sous-jacent. Une fois activée, cette fonction s'applique à tous les protocoles utilisées par ce client.
