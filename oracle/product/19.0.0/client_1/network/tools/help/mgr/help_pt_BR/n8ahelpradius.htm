RADIUS

O RADIUS (Remote Authentication Dial-In User Service) é um protocolo de segurança cliente/servidor conhecido principalmente por permitir autenticação e acesso remotos. Você pode permitir que sua rede utilize qualquer método de autenticação que suporte o padrão RADIUS, incluindo cartões de token e smartcard, simplesmente instalando e configurando o método RADIUS. O método de autenticação RADIUS requer que você defina os seguintes parâmetros:

Elemento Descrição
Nome do Host O nome do host do servidor RADIUS principal Default = localhost
Número da Porta Número da porta de listening do servidor RADIUS principal Padrão = 1645
Timeout (segundos) Número de segundos que o servidor Oracle esperará por uma resposta do servidor RADIUS principal. Default = 15 segundos
Número de Repetições Número de vezes que o servidor de banco de dados Oracle deve enviar as mensagens ao servidor RADIUS principal

Default = 3
Arquivo Secreto A localização do arquivo de chave secreta no servidor Oracle. Por motivos de segurança, a Oracle recomenda que você altere esse arquivo para acesso somente pela raiz.
Enviar Contabilidade A Contabilidade do RADIUS registra informações de acesso ao servidor de bancos de dados Oracle e armazena essas informações em um arquivo do servidor de contabilidade do RADIUS. Você só pode usar esse recurso quando os seus dois servidores de autenticação RADIUS o suportarem. Valores = ON ou OFF.
Resposta a Desafio O valor desse campo é ativo quando você quer apresentar ao usuário uma GUI de resposta a desafio
Palavra-Chave Default Palavra-chave que verifica a identidade de um usuário através de um mecanismo de resposta a desafio Default = challenge
Nome da Classe de Interface O nome da classe que trata a conversão de resposta a desafio entre o cliente Oracle e o servidor RADIUS. Default = DefaultRadiusInterface

Para obter mais informações, consulte o apêndice Integrating Authentication Devices Using RADIUS do Oracle Advanced Security Administrator's Guide.
