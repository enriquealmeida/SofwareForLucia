네트 서비스 별칭 생성 대화상자

네트 서비스 별칭 생성 대화상자를 사용하여 디렉토리 서버에 네트 서비스 별칭을 생성할 수 있습니다.

네트 서비스 별칭은 네트 서비스 이름 또는 데이터베이스 서비스에 대한 대체 이름입니다. 네트 서비스 별칭 항목에는 접속 기술자 정보가 없습니다. 대신 이 항목은 별칭을 가진 객체의 위치만 참조합니다. 클라이언트가 네트 서비스 별칭의 디렉토리 조회를 요청하면 디렉토리는 항목이 네트 서비스 별칭이고 별칭이 참조된 항목인 것처럼 조회를 완료하는지 확인합니다. 예를 들면, db1alias라는 네트 서비스 별칭이 데이터베이스 서비스 db1에 대해 생성될 수 있습니다. CONNECT scott/tiger@db1alias처럼 db1alias가 데이터베이스 서비스 접속에 사용되는 경우 실제로 db1에 대한 접속 기술자 정보로 분석되고 사용됩니다.

네트 서비스 별칭은 다음 두 가지 경우에 주로 사용됩니다.

  • 클라이언트가 데이터베이스 서비스나 네트 서비스 이름을 다른 이름으로 참조하는 방법으로 네트 서비스 별칭을 사용합니다.

  • 다른 Oracle Context에 있는 데이터베이스 서비스나 네트 서비스 이름에 대해 특정 Oracle Context에 있는 네트 서비스 별칭을 사용합니다. 이렇게 하면 데이터베이스 서비스나 네트 서비스 이름을 디렉토리 서버에서 한 번 정의하여 다른 Oracle Context를 사용하는 클라이언트가 참조하도록 할 수 있습니다.

네트 서비스 별칭

네트 서비스 별칭 이름을 입력합니다.

네트 서비스 이름 또는 데이터베이스 서비스

[네트 서비스 이름] 또는 [데이터베이스 서비스] 상자를 사용하여 데이터베이스 서비스나 네트 서비스 이름에 대한 정보를 입력합니다.

Oracle Context: 목록에서 하나를 선택하거나 필드에 직접 입력하여 데이터베이스 서비스 또는 네트 서비스 이름의 Oracle Context를 지정합니다.

이름: 목록에서 하나를 선택하거나 필드에 직접 입력하여 데이터베이스 서비스 또는 네트 서비스 이름의 DN(식별 이름)을 지정합니다.
