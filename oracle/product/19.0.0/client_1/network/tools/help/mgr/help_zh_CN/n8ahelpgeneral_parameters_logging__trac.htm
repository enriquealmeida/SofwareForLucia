一般参数：事件记录和跟踪

使用事件记录和跟踪选项卡可以启用监听程序的事件记录和跟踪。Oracle Net Manager 使您能够指定写入监听程序日志文件的名称和位置。

禁用事件记录

选择此选项以禁用监听程序上的事件记录。

启用事件记录

选择此选项以启用监听程序上的事件记录。一旦启用，您就可以指定写入日志信息的位置。

日志文件

输入监听程序日志信息将写入的目录路径名和文件名。请注意该文件的扩展名始终是 .log。默认名称是 listener.log。在 UNIX 上，默认目录是 $ORACLE_HOME/network/log；在 Windows 平台上，默认目录是 ORACLE_HOME\network\log。

禁用跟踪

选择此选项以禁用监听程序上的跟踪。

启用跟踪

选择此选项以启用监听程序上的跟踪。一旦启用跟踪，您就可以指定需要的跟踪级别，并通过更改默认跟踪文件来指定写入跟踪信息的位置。

跟踪级别

从列表中，选择跟踪级别：

禁用 - 不启用跟踪。OFF 是默认设置。

用户 - 将跟踪设置为与用户相应的级别。跟踪以标识用户导致的错误条件。

管理员 - 将跟踪设置为与数据库管理员相应的级别。跟踪以标识特定的安装问题。

支持 - 将跟踪设置为与客户支持相应的级别。

跟踪文件可能会变得很大。Oracle Corporation 建议在不进行网络问题诊断时关闭跟踪功能。

跟踪文件

输入监听程序跟踪信息将写入的目录路径名和文件名。请注意该文件的扩展名始终是 .trc。默认名称为 listener.trc。在 UNIX 上，默认目录为 $ORACLE_HOME/network/trace；在 Windows 平台上，默认目录为 ORACLE_HOME\network\trace。
