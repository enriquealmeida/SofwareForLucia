리스너 구성 개요

이 절에서는 다음 항목을 다룹니다.

설치 시 구성

Enterprise Edition 또는 Standard Edition을 데이터베이스 서버에 설치하는 동안 Oracle Net Configuration Assistant는 Oracle 데이터베이스에 대한 TCP/IP 수신 프로토콜 주소를 가지는 LISTENER 라는 리스너를 구성합니다. 사용자 정의 설치 중에 Oracle Net Configuration Assistant는 선택한 리스너 이름 및 프로토콜 주소를 구성하라는 메시지를 표시합니다.

또한 설치 유형에 관계없이 외부 프로시저 호출에 대한 수신 IPC 프로토콜 주소가 자동으로 구성됩니다.

Oracle Net Configuration Assistant는 listener.ora 파일에서 외부 프로시저에 대한 서비스 정보를 자동으로 구성하기도 합니다.

Oracle Net Manager로 구성

Oracle Net Manager를 사용하여 다음을 수행할 수 있습니다.

  • 리스너 생성

  • Oracle8 또는 이전 데이터베이스에 대한 서비스 정보 구성

  • 외부 프로시저와 이기종 서비스를 포함한 비Oracle 데이터베이스 서비스의 서비스 정보 구성

  • 추적 및 로깅 기능 사용

  • 리스너 제어 유틸리티에 대한 비밀번호 인증 사용

리스너 구성에 액세스하려면 다음과 같이 하십시오.

네비게이터 창에서 Oracle Net 구성 > 로컬 > 리스너를 확장합니다.

자세한 내용은 Oracle10i Net Administrator's Guide의 Configuring and Administering the Listener 장을 참조하십시오.
