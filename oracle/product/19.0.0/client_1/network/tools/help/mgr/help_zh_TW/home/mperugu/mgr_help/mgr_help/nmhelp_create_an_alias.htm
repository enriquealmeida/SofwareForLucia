建立網路服務別名

注意事項：

  • 只有身為 OracleNetAdmins 或 OracleContextAdmins 群組成員的使用者，才可以在目錄中建立網路服務別名項目。

  • 使用 Microsoft Active Directory 不支援網路服務別名。

  • 您可以從 Oracle Names 伺服器匯出現有的別名物件。如需詳細資訊，請參閱 Oracle10i Net Services Administrator's Guide 中的 Exporting Naming Data to a Directory Server 章節。

您可以為目錄伺服器中的目錄命名物件 (網路服務名稱或資料庫服務)，建立網路服務別名。網路服務別名中儲存其參照之物件的名稱。

當從屬端要求進行網路服務別名的目錄查詢時，目錄會判斷項目是否為網路服務別名，將其視為參照的項目完成查詢。例如，可針對 sales 網路服務名稱建立一個稱為 salesalias 的網路服務別名。當使用 salesalias 來連線資料庫服務時，例如 CONNECT scott/tiger@salesalias，實際上就會解析成 sales 的連線描述區資訊，並使用該資訊來連線。

網路服務名稱別名有兩個主要用途：

  • 使用網路服務名稱別名作為從屬端以其他名稱參照資料庫服務或網路服務名稱的方法。

  • 將網路服務名稱別名用於不同 Oracle Context 之資料庫服務或網路服務名稱的一個 Oracle Context。這樣只要在目錄伺服器定義一次資料庫服務或網路服務名稱，即可用於其他的 Oracle Context。

建立網路服務名稱別名：

  1. 在瀏覽窗格中，展開 Oracle Net 組態 >目錄 > 服務命名 > 別名。

  2. 從工具列選取加號 (+)，或者從功能表列選擇編輯 > 建立。建立網路服務別名對話方塊就會顯示。

  3. 在網路服務別名欄位中，輸入網路服務別名的名稱。

  4. 在「網路服務」或「資料庫服務」方塊中，輸入網路服務名稱或資料庫服務的下列相關資訊：

    • Oracle Context：從清單選取或在欄位中輸入，以指定資料庫服務的 Oracle Context 或網路服務名稱。

    • 名稱：從清單選取或在欄位中輸入，以指定資料庫服務的辨別名稱 (DN) 或網路服務名稱。

  5. 選擇建立。
