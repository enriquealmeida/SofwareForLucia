Datenbank-residentes Verbindungspooling

Das datenbank-residente Verbindungspooling ermöglicht das Pooling von "dedizierten" Verbindungen über Clientanwendungen oder Prozesse hinweg. Eine dedizierte Verbindung bezieht sich auf einen dedizierten Server zusammen mit dessen Datenbanksession. Verwenden Sie das datenbank-residente Verbindungspooling, wenn mehrere Clientanwendungen (wie Mid-Tiers) persistente Verbindungen mit der Datenbank verwalten müssen, oder wenn Server-Ressourcen (wie z.B. Speicher) optimiert werden müssen.

Bei dem datenbank-residenten Verbindungspooling sind Clients mit einem Hintergrundprozess verbunden, der als Connection Broker bezeichnet wird und die Pooling-Funktionalität bereitstellt.

Gegensatz zu dediziertem Server und Shared Server.
