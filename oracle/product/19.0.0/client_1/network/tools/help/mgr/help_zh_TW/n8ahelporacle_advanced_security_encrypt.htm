Oracle Advanced Security：加密

您可以使用加密頁籤，為選取的認證方法設定加密參數。

注意：您可以使用 SSL 頁籤來設定「安全通訊端層 (SSL)」的 Cipher Suite。

加密

從清單選取您目前要設定的電腦：從屬端或伺服器。

加密類型

選取一個值以指定從屬端或伺服器在交涉加密和完整性時的行為。下面是四個可能的值：

元素 描述
接受 如果連線的另一端為「必要」或「要求」，並且也有可用的相容演算法，則服務會在作用中；否則，服務將會是非作用中。
拒絕 服務必定不是作用中，如果另一端為「必要」，連線將會失敗。
要求 如果連線的另一端指定為「接受」、「必要」或「要求」，並且也有可用的相容演算法，則服務會在作用中；否則，服務將為非作用中。
必要 服務必須為作用中，如果另一端為「拒絕」，或者沒有可用的相容演算法，連線將會失敗。

加密原始內建值

在加密原始內建值欄位中輸入加密原始內建值 (最多 256 個字元的隨機字串)。加密原始內建值是用來產生加密的金鑰。這是啟用加密或完整性時所需。

注意：如果您選擇要使用特殊字元 (例如逗號 [,] 或右括弧 [)]) 作為「加密原始內建值」參數的一部分，Oracle 建議您用單引號將值括起來。

可用的方法/選取的方法清單

顯示可選取的認證方法以及選取的加密方法。清單中的方法順序會決定偏好的交涉順序。

使用向左箭號按鈕 (<) 和向右箭號按鈕 (>)，在清單之間移動選取的加密演算法。以下是 Oracle Advanced Security 支援的加密演算法：

元素 描述
AES256 AES 256 位元金鑰
RC4_256 RC4 256 位元金鑰
AES192 AES 192 位元金鑰
3DES168 三重 DES 與三金鑰 (168 位元) 選項
AES128 AES 128 位元金鑰
RC4_128 RC4 128 位元金鑰
3DES112 三重 DES 與雙金鑰 (112 位元) 選項
DES DES 56 位元金鑰
RC4_40 RC4 40 位元金鑰
DES40 DES 40 位元金鑰
