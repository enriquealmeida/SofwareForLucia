Oracle Advanced Security: 무결성

무결성 탭을 사용하여 인증 방법에 대한 데이터 무결성을 구성할 수 있습니다. 데이터 무결성은 접속을 통해 전송된 데이터를 클라이언트와 서버에서 비교하는 작업입니다.

주: SSL 탭을 사용하여 SSL(Secure Sockets Layer)에 대한 암호화 모음을 구성합니다.

무결성

서버 또는 [클라이언트] 체크 박스를 선택하여 현재 구성하는 시스템이 클라이언트인지 아니면 서버인지를 지정합니다. [서버] 옵션은 시스템이 접속하는 서버에는 적용되지 않습니다.

체크섬 레벨

체크섬 레벨을 선택합니다. 다음 네 가지 체크섬 레벨을 선택할 수 있습니다.

요소 설명
승인됨 상대편 접속이 "필요함" 또는 "요청됨"을 지정하고 사용 가능한 호환 알고리즘이 있으면 서비스를 사용할 수 있습니다. 그렇지 않으면 서비스를 사용할 수 없습니다.
거부됨 서비스를 사용할 수 없으며 상대편 접속이 "필요함"을 지정해도 접속에 실패합니다.
요청됨 상대편 접속이 "승인됨", "필요함", "요청됨" 중 하나를 지정하고 사용 가능한 호환 알고리즘이 있으면 서비스를 사용할 수 있습니다. 그렇지 않으면 서비스를 사용할 수 없습니다.
필요함 서비스를 사용할 수 있으며, 상대편 접속이 "거부됨"을 지정하거나 호환 알고리즘이 없으면 접속에 실패하게 됩니다.

사용 가능한 방법 목록

체크섬을 생성하는 데 사용되는 무결성 알고리즘을 표시합니다. 두 가지 방법이 있습니다.

알고리즘 이름 적합한 값
SHA-1(보안 해시 알고리즘) SHA1
MD5(Message Digest 5) MD5

선택된 방법 목록

선택된 알고리즘을 표시합니다. 목록에서의 알고리즘 순서에 따라 해당 알고리즘이 사용되는 순서가 결정됩니다.
