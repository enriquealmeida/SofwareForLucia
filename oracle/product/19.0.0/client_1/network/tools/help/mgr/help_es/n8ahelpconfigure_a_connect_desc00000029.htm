Configuración de un Descriptor de Conexión para Procedimientos Externos

Los procedimientos externos son procedimientos que se llaman desde otro programa, pero están escritos en un lenguaje diferente. Un ejemplo sería un programa PL/SQL que llama a una o más rutinas C que son necesarias para realizar un procesamiento para fines especiales.

Para que la base de datos Oracle se conecte a un procedimiento externo, el servidor de bases de datos debe estar configurado con un nombre de servicio de red para acceder a los procedimientos externos.

Nota: El Asistente de Configuración de Red de Oracle crea automáticamente durante la instalación un nombre de servicio de red para los procedimientos externos.

Para configurar un servidor de bases de datos para procedimientos externos:

  1. Compruebe que el listener está configurado para procedimientos externos, como se describe en Configuración de Procedimientos Externos para el Listener.

  2. Cree un nombre de servicio de red en el archivo tnsnames.ora de la base de datos. Asegúrese de que la información del descriptor de conexión coincide con la información configurada para el listener.

  3. Seleccione Archivo > Guardar Configuración de Red.
