Visão Geral da Nomeação de Diretório

A nomeação de diretório é um dos principais métodos de nomeação utilizados para resolver um identificador de conexão para um descritor de conexão utilizado pelo cliente para estabelecer conexão com um banco de dados. A nomeação de diretório armazena identificadores de conexão e seus descritores de conexão em um servidor de diretório compatível com LDAP. Os identificadores de conexão são armazenados sob uma subárvore do Oracle Context que contém entradas para uso com software Oracle.

Um usuário final informa uma string de conexão que inclui o identificador de conexão no seguinte formato:

CONNECT nome do usuário/senha@identificador_conexão

No exemplo a seguir, sales é o identificador de conexão pesquisado no diretório:

CONNECT scott/tiger@sales

O descritor de conexão retornado ao cliente inclui:

  • A localização do endereço de protocolo do listener

  • O nome de serviço do banco de dados

O cliente utiliza as informações de endereço do protocolo para estabelecer conexão com um listener. O listener, configurado com as mesmas informações de endereço, recebe a solicitação. Se o listener estiver atendendo a solicitações para o mesmo serviço de banco de dados que o especificado no descritor de conexão, uma conexão será concedida.

Para que a nomeação de diretório ocorra, configure um servidor de diretório para recursos do Oracle Net Services e o acesso de clientes ao diretório.

Para obter mais informações, consulte o capítulo Configuration Management Concepts no Oracle10i Net Administrator's Guide.
