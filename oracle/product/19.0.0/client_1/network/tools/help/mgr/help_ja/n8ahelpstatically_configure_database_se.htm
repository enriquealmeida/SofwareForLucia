データベース・サービス情報の静的構成

リスナーがOracle8またはOracle7のデータベースへのクライアント・リクエストを受け入れるようにするには、listener.oraファイル内にそのデータベースに関する情報を構成する必要があります。

データベースがOracle9iまたはOracle8iのリリースの場合、listener.oraファイル内に静的サービス情報は不要です。サービス情報は、サービス登録を通じてリスナーに自動的に登録されます。ただし、Oracle Enterprise Managerを使用する場合は、データベースがOracle8iの場合でも、静的なサービス情報が必要です。

リスナー用のデータベース・サービス情報を構成する手順は、次のとおりです。

  1. ナビゲータ・ペインで、「Oracle Netの構成」→「ローカル」→「リスナー」を展開します。

  2. リスナーを選択します。

  3. 右ペインのリストから「データベース・サービス」を選択します。

  4. 「データベースの追加」を選択します。新しい「データベース」タブが表示されます。

  5. 「グローバル・データベース名」フィールドにグローバル・データベース名、「Oracleホーム・ディレクトリ」フィールドにOracleホーム・ディレクトリの位置、「SID」フィールドにデータベースのOracleシステム識別子(SID)を入力します。

  6. 「ファイル」→「ネットワーク構成の保存」を選択します。
