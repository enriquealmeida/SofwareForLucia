Oracle Connection Manager 접속의 접속 기술자 구성

Oracle Connection Manager를 통해 클라이언트 경로를 데이터베이스 서버로 지정하려면 tnsnames.ora 파일을 구성하거나 Oracle Connection Manager의 프로토콜 주소를 지정하는 접속 기술자를 사용하여 디렉토리 서버를 구성합니다. 이 주소를 사용하여 클라이언트와 Oracle Connection Manager 컴퓨터를 연결할 수 있습니다.

Oracle Connection Manager에 대한 프로토콜 주소를 구성하려면 다음과 같이 하십시오.

  1. 네비게이터 창에서 Oracle Net 구성 > 디렉토리 또는 로컬 > 서비스 이름 지정을 확장합니다.

  2. 도구 모음에서 플러스 기호(+)를 선택하거나 메뉴 표시줄에서 편집 > 생성을 선택합니다. 네트 서비스 이름 마법사의 시작 페이지가 나타납니다.

  3. 네트 서비스 이름 필드에 이름을 입력합니다.

  4. 다음을 선택합니다. [프로토콜] 페이지가 나타납니다.

  5. Oracle Connection Manager가 수신하도록 구성된 프로토콜을 선택합니다. 기본적으로 이 프로토콜은 TCP/IP입니다.

  6. 다음을 선택합니다. [프로토콜 설정] 페이지가 나타납니다.

  7. 선택한 프로토콜에 해당하는 프로토콜 매개변수를 제공된 필드에 입력합니다. TCP/IP를 사용할 경우 기본 포트는 1630입니다.

  8. 다음을 선택합니다. [서비스] 페이지가 나타납니다.

  9. 대상 서비스가 Oracle8i 이상인 경우 Oracle8i 이상을 선택하고 서비스 이름 필드에 서비스 이름을 입력합니다. 대상 서비스가 Oracle8 또는 Oracle7인 경우 Oracle8 또는 이전을 선택하고 데이터베이스 SID 필드에 Oracle 시스템 식별자를 입력합니다.

  10. 다음을 선택합니다. [서비스] 페이지가 나타납니다.

  11. 완료를 눌러 구성을 저장하고 네트 서비스 이름 마법사를 종료합니다.

  12. 파일 > 네트워크 구성 저장을 선택합니다.

  13. 새 네트 서비스 이름이 [서비스 이름 지정] 폴더에 추가됩니다.
