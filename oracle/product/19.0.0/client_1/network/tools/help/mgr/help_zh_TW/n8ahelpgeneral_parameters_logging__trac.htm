一般參數：記錄日誌和追蹤

您可以使用記錄日誌與追蹤頁籤來啟動監聽器的記錄日誌和追蹤。您可以使用 Oracle Net Manager 來指定要寫入監聽器日誌檔的名稱與位置。

停用記錄日誌

選取停用監聽器記錄日誌。

啟用記錄日誌

選取啟用監聽器記錄日誌。啟用之後，您可以指定要寫入日誌資訊的位置。

日誌檔

輸入要寫入監聽器日誌資訊的目錄路徑名稱和檔案名稱。請注意，此檔案的副檔名一律是 .log。預設名稱是 listener.log。預設目錄是 $ORACLE_HOME/network/log (UNIX 平台) 以及 ORACLE_HOME\network\log (Windows 平台)。

停用追蹤

選取停用監聽器的追蹤。

啟用追蹤

選取啟用監聽器的追蹤。啟用之後，您可以變更預設追蹤檔，指定想要的追蹤層次以及追蹤資訊的寫入位置。

追蹤層次

從清單中選取追蹤層次：

Off--未啟用追蹤。OFF 是預設值。

User -- 將追蹤設定成適用於使用者的層次。透過追蹤來識別使用者導致的錯誤條件。

Administrator -- 將追蹤設定成適用於資料庫管理員的層次。追蹤識別與安裝有關的問題。

Support -- 將追蹤設定成適用於客戶支援的層次。

追蹤檔可能會變得非常大。Oracle Corporation 建議您在未進行網路問題疑難排解時，將追蹤關閉。

追蹤檔

輸入要寫入監聽器追蹤資訊的目錄路徑名稱和檔案名稱。請注意，此檔案的副檔名一律是 .trc。預設名稱是 listener.trc。預設目錄是 $ORACLE_HOME/network/trace (UNIX 平台) 以及 ORACLE_HOME\network\trace (Windows 平台)。
