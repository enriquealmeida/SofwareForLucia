일반 사항: 추적

추적 탭을 사용하여 이 컴퓨터에 대한 추적 매개변수를 설정할 수 있습니다. 이 컴퓨터가 클라이언트인 경우에는 클라이언트 정보 필드를 사용하고, 이 컴퓨터가 서버인 경우에는 서버 정보 필드를 사용하십시오.

클라이언트 추적은 클라이언트가 수행하는 작업을 추적합니다. 서버 추적은 네트워크 접속 기간 동안 서버가 수행한 작업을 추적합니다.

추적 레벨

목록에서 추적 레벨을 선택합니다.

OFF--추적이 설정되지 않습니다. [OFF]가 기본 설정입니다.

USER--추적이 사용자에게 적합한 레벨로 설정됩니다. 사용자가 생성한 오류 상태를 확인하기 위해 추적합니다.

ADMIN--추적이 데이터베이스 관리자에게 적합한 레벨로 설정됩니다. 설치 관련 문제를 확인하기 위해 추적합니다.

SUPPORT--추적이 고객 지원에 적합한 레벨로 설정됩니다.

추적 파일은 매우 커질 수 있습니다. 따라서 Oracle은 네트워크 문제를 해결하기 위한 경우가 아니면 추적 기능을 사용하지 않을 것을 권장합니다.

추적 디렉토리

추적 파일이 기록될 디렉토리를 입력합니다. 클라이언트와 서버 추적 파일은 클라이언트나 서버에 대한 정보를 포함하지만 두 파일 모두 로컬 컴퓨터에 기록됩니다. 기본값은 UNIX의 경우 $ORACLE_ HOME/network/trace 이며 Windows의 플랫폼의 경우 ORACLE_HOME\network\trace 입니다.

추적 파일

추적 파일 이름을 입력합니다. 기본 파일 이름은 클라이언트의 경우 sqlnet.trc이며 서버의 경우 svr_pid..trc입니다.

고유한 추적 파일 이름

각 추적 파일에 대해 고유한 이름을 생성합니다. 각 추적 파일 이름에는 프로세스 식별자가 추가되므로 여러 추적 파일이 공존할 수 있습니다. 고유한 추적 파일을 생성하지 않고 모든 추적 정보를 동일한 추적 파일에 저장하려면 이 옵션을 선택 해제합니다.
