서비스 등록

인스턴스와 디스패처가 자동으로 리스너에 정보를 등록하는 기능입니다. 이 정보는 리스너에 등록되기 때문에 listener.ora 파일을 이 정적 정보로 구성할 필요가 없습니다.

데이터베이스를 시작할 때 서비스 등록은 리스너에 다음과 같은 정보를 제공합니다

  • 데이터베이스의 실행 중인 각 인스턴스 서비스 이름

  • 데이터베이스 인스턴스 이름

  • 각 인스턴스에서 사용할 수 있는 서비스 처리기(디스패처와 전용 서버)

  • 작업 할당자, 인스턴스, 노드 로드 정보

이러한 정보를 사용하여 리스너는 클라이언트 요청을 적절하게 지정할 수 있습니다.

리스너는 이 로드 정보를 사용하여 클라이언트 접속 요청을 가장 잘 처리할 수 있는 디스패처를 결정할 수 있습니다. 모든 디스패처가 차단되면 리스너는 해당 접속을 위한 전용 서버를 생성할 수 있습니다.
