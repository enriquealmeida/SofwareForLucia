Identificación de Servicios

Esta hoja de propiedades permite ver o modificar la información de red de un nombre de servicio de red o servicio de base de datos seleccionados.

Identificación de Servicios

Utilice el campo Identificación de Servicios para visualizar o modificar la información de CONNECT_DATA.

Elemento Descripción
Nombre del Servicio Introduzca un nombre de servicio si está configurando este cliente para que se conecte a un servicio de base de datos de Oracle8 i o posterior.Nota: Para un servicio de base de datos almacenado en un servidor de directorios, este campo solo muestra el valor actual. No se permite realizar modificaciones.
SID Muestra el identificador del sistema Oracle (SID) de una base de datos Oracle8.
Tipo de Conexión De la lista, seleccione si desea que este nombre de servicio de red utilice un servidor dedicado, servidor compartido o el servidor con el que esté configurada la base de datos. Si el servidor compartido está configurado en el archivo de parámetros de inicialización, seleccione Servidor Dedicado para hacer que el listener inicie un servidor dedicado para ignorar la configuración de servidor compartido. Si el servidor compartido está configurado en el archivo de parámetros de inicialización y desea asegurarse de que esta conexión siempre utiliza un servidor compartido, seleccione Servidor Compartido. Si la opción Pool de Conexiones Residentes de la Base de Datos está activada en el servidor, puede seleccionar Servidor de Pool para obtener una conexión desde el pool.
Usar Identificación Compatible con Oracle8 Versión 8.0 Seleccione esta opción si va a configurar un cliente para conectarse a un servicio de base de datos anterior a la versión Oracle8i. Elimine esta opción si va a configurarlo para un servicio de base de datos Oracle9i u Oracle8i.
Avanzado Seleccione esta opción para configurar parámetros avanzados de CONNECT_DATA.

Configuración de Direcciones

Utilice el campo Dirección para modificar, agregar o suprimir una dirección de protocolo de un listener.

Si la dirección ya está configurada, se muestra la información del protocolo en los campos.

Elemento Descripción
Protocolo En la lista desplegable, seleccione el protocolo para la dirección del listener. Dependiendo del protocolo seleccionado, introduzca la información en los campos según corresponda.
Botón Mostrar Avanzadas/Ocultar Avanzadas Utilice este botón para mostrar u ocultar los valores para especificar el límite de espacio de E/S para la dirección.

Estos valores son compatibles con los protocolos TCP/IP, TCP/IP con SSL y SDP.

  • Tamaño Total de Buffer de Envío: Especifique, en bytes, el espacio de buffer para las operaciones de envío de las sesiones.

  • Tamaño Total de Buffer de Recepción : Especifique, en bytes, el espacio de buffer para las operaciones de recepción de las sesiones.

Nota: La especificación de estos valores los anula en el archivo sqlnet.ora del cliente.
Botón + Haga clic para crear una nueva dirección de protocolo. Aparece un nuevo separador Dirección donde puede seleccionar el protocolo e introducir la dirección de protocolo en los campos. El protocolo seleccionado se debe instalar en el cliente y en el servidor (donde está el listener).
Botón X Haga clic en este botón para eliminar el separador Dirección seleccionado y su información de dirección.
Botón < Haga clic en este botón para subir la dirección del protocolo en el orden de lista.
Botón > Haga clic en este botón para bajar la dirección del protocolo en el orden de lista.
Botón Avanzado Haga clic en este botón para configurar opciones de varias direcciones, como redirección automática en caso de fallo del tiempo de conexión y equilibrio de carga del cliente.

Botón Aplicar

Haga clic para aplicar los valores.

Botón Revertir

Haga clic para restablecer los valores originales.

Para obtener más información sobre la configuración del espacio del buffer de E/S, consulte el capítulo sobre optimización del rendimiento, en Oracle10i Net Administrator's Guide (Guía del Administrador de Red de Oracle10i).
