Geral: Avançado

A guia Avançado permite que você configure informações avançadas sobre perfil para um computador servidor ou cliente.

Timeout de operação de Envio

Use para especificar o tempo, em segundos, para que um servidor de banco de dados conclua uma operação de envio para clientes após o estabelecimento da conexão.

Para ambientes em que os clientes são desligados ocasionalmente ou de forma anormal, é recomendável a definição deste parâmetro. Se o servidor de bancos de dados não puder concluir uma operação de envio no tempo especificado, ele registrará ORA-12535:TNS:a operação sofreu timeout e ORA-12608: TNS: Ocorreu timeout de envio para o arquivo sqlnet.log.

Sem este parâmetro, o servidor de banco de dados continuará a enviar respostas a clientes que não podem receber dados devido a uma situação de computador desligado ou a um estado de ocupado.

Você também pode definir este parâmetro no lado cliente para especificar o tempo, em segundos, para que um cliente conclua operações de envio para o servidor de banco de dados após o estabelecimento da conexão. Sem este parâmetro, o cliente poderá continuar a enviar solicitações a um servidor de banco de dados já saturado de solicitações.

Timeout de operação de Recepção

Use para especificar o tempo, em segundos, para que um servidor de banco de dados aguarde dados do cliente após o estabelecimento da conexão. Um cliente deverá enviar dados nesse intervalo.

Para ambientes em que os clientes sejam desligados ocasionalmente ou em caso de anormalidade, é recomendável a definição deste parâmetro. Se um cliente não enviar dados no tempo especificado, o servidor de banco de dados registra um ORA-12535:TNS:timeout de operação e ORA-12609: TNS: Ocorreu timeout de recepção no arquivo sqlnet.log.

Sem este parâmetro, o servidor de banco de dados continuará a aguardar dados dos clientes que podem estar desligados ou em dificuldades. Você também pode usar esta definição no lado do cliente para especificar o tempo, em segundos, para que um cliente aguarde dados de resposta do servidor de banco de dados após o estabelecimento da conexão.

Sem este parâmetro, o cliente poderá aguardar por muito tempo a resposta de um servidor de banco de dados saturado de solicitações.

Tamanho Total do Buffer de Envio

Especifique o limite de espaço do buffer para operações de envio de sessões. Este parâmetro é suportado pelos protocolos TCP/IP, TCP/IP com SSL e SDP.

Tamanho Total do Buffer de Recepção

Especifique o limite de espaço do buffer para operações de recebimento de sessões. Este parâmetro é suportado pelos protocolos TCP/IP, TCP/IP com SSL e SDP.

Valor de Timeout de TNS

Especifique o intervalo, em minutos, para confirmar se a conexão cliente/servidor ainda está ativa. A definição de um valor diferente de zero assegura que as conexões não fiquem abertas indefinidamente, devido a um encerramento anormal do cliente. Se a verificação encontrar uma conexão desativada ou que não seja mais utilizada, retornará um erro, provocando a finalização do processo do servidor. Esta definição destina-se ao servidor de banco de dados, que normalmente lida com várias conexões simultaneamente.

As restrições de uso deste recurso de detecção de conexões desativadas são:

  • Ela não é permitida em conexões transmitidas.

  • Embora seja muito pequeno, um pacote de verificação gera um tráfego adicional que pode afetar o desempenho da rede.

  • Dependendo do sistema operacional utilizado, o servidor talvez precise executar processamento adicional para distinguir o evento de verificação de conexão de outros eventos. Isso pode também resultar em degradação no desempenho da rede.

Observação: Essa opção só pode ser usada no servidor.

ID de Registro de Cliente

Informe um identificador exclusivo para este cliente. Esse identificador é passado para o listener com qualquer solicitação de conexão.

O listener inclui esse identificador em seu arquivo de log. O identificador pode ser qualquer string de até 128 caracteres.

Versão do Protocolo de Autenticação de Log-on

Use para especificar que protocolos de autenticação são permitidos pelo banco de dados do cliente. Se o cliente e o servidor de banco de dados não tiverem pelo menos uma versão correspondente, haverá falha de autenticação com erro.

Os valores suportados incluem:

  • protocolos de autenticação 10 para 10i

  • protocolos de autenticação 9 para 9i

  • protocolos de autenticação 8 para 8i

  • Padrão para protocolos de autenticação 10i, 9i e 8i

Qualquer valor diferente de 10 poderá expor vulnerabilidades que possam ter existido na versão anterior dos protocolos de autenticação. Para obter compatibilidade completa, defina a lista de versões permitidas para log-on em Padrão para que incluam todas as versões de bancos de dados no sistema.

Desativar Tratamento de Sinal UNIX

Escolha para configurar o cliente a passar a limpeza de processos filhos para o processo de inicialização do UNIX desativando handlers de sinais.

Como a aplicação cliente gera um processo de servidor internamente, a aplicação cliente passará a ser responsável pela remoção do processo de cliente quando ele for concluído. Quando o processo de servidor concluir a sua responsabilidade na conexão, ele passará a ser um processo desativado. Os handlers de sinais são responsáveis por remover esses processos desativados.

Desativar Interrupção Fora da Faixa

Escolha para desativar ou ativar interrupções fora da faixa.

Se for desmarcada, essa opção permitirá ao Oracle Net enviar e receber mensagens de "interrupção" usando dados urgentes fornecidos pelo protocolo subjacente.

Se for selecionada, desativará a capacidade de envio e recebimento de mensagens de "interrupção" com o uso de dados urgentes fornecidos pelo protocolo subjacente. Uma vez ativado, esse recurso se aplica a todos os protocolos usados por este cliente.
