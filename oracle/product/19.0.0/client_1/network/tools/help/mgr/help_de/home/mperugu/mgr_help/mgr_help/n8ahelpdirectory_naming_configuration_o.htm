Überblick über die Konfiguration der Verzeichnisbenennung

In diesem Thema werden die Details der Konfiguration der Verzeichnisbenennung beschrieben.

Server-Konfiguration während der Installation

Während einer Benutzerdefinierten Server-Installation fordert der Oracle Net-Konfigurationsassistent Sie zur Ausführung der folgenden Schritte auf:

  • Wählen Sie den Typ des Directory Servers

  • Geben Sie den Speicherort des Verzeichnisses an.

  • Wählen oder erstellen Sie einen Oracle Context, aus dem der Server Connect Identifier suchen, erstellen und ändern kann.

  • Konfigurieren Sie die Verzeichnisbenennung als bevorzugte Benennungsmethode.

Wenn die Directory-Verwendung richtig konfiguriert wurde, versucht der Datenbank-Konfigurationsassistent, sich bei dem Verzeichnis anzumelden und einen Eintrag für einen Datenbankdienst im Directory zu registrieren. Der Eintrag für den Datenbankdienst wird mit einem Verbindungsdeskriptor definiert, der folgende Netzwerkinformationen enthält:

  • Adresse des Listeners

  • Name des Datenbankdienstes

Clientkonfiguration während der Installation

Während allen Client-Installationen fordert der Oracle Net-Konfigurationsassistent Sie auf, die folgenden Einstellungen für die Directory-Verwendung zu konfigurieren:

  • Geben Sie den Typ des Verzeichnisses an.

  • Geben Sie den Speicherort des Verzeichnisses an.

  • Wählen Sie einen standardmäßigen Oracle Context, aus dem der Client Connect Identifier suchen kann.

  • Verzeichnisbenennung als bevorzugte Benennungsmethode

Konfiguration mit dem Oracle Net-Konfigurationsassistenten nach der Installation

Der Oracle Net-Konfigurationsassistent kann auch im Standalone-Modus ausgeführt werden, um:

  • den Client oder Server (wie oben beschrieben) zu konfigurieren.

  • das Oracle-Schema zu erstellen.

  • einen Oracle Context zu erstellen.

So führen Sie den Oracle Net-Konfigurationsassistenten im Standalone-Modus aus:

  • Führen Sie unter UNIX netca in $ORACLE_HOME/bin auf.

  • Bei Windows NT wählen Sie Start > Programme > Oracle - HOME_NAME> Netzwerk-Administration > Oracle Net-Konfigurationsassistent.

Konfiguration mit Oracle Net Manager

Mit Oracle Net Manager können Sie:

  • Oracle Context ändern

  • die ID-Daten für die Berechtigungsprüfung ändern.

  • Net Service-Namen erstellen

  • Net Service Alias erstellen

  • mehrere Routes zu einem Listener über mehrere Protokolladressen konfigurieren.

  • erweiterte CONNECT_DATA-Informationen für einen Datenbankdienst oder einen Net Service-Namen konfigurieren.

  • Net Service-Name-Einträge aus einer Datei tnsnames.ora zu einem Directory-Server migrieren.

Um die Konfiguration der Verzeichnisbenennung aufzurufen, wählen Sie im Navigator-Fensterbereich Directory-Dienstebenennung.

Weitere Informationen finden Sie im Hilfesystem des Oracle Net-Konfigurationsassistenten.
