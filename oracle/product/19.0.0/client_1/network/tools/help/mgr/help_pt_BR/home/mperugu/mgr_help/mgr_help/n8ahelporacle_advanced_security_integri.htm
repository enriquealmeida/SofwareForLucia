Oracle Advanced Security: Integridade

A guia Integridade permite que você configure a integridade de dados para os métodos de autenticação. A integridade de dados é a comparação dos dados enviados pela conexão, conforme eles são vistos pelo cliente e pelo servidor.

Observação: Use a guia SSL para configurar conjuntos de cifras para a SSL (Secure Sockets Layer).

Integridade

Selecione a caixa Servidor ou Cliente para indicar se o computador que você está configurando no momento é um Cliente ou um Servidor. A opção Servidor não se aplica aos servidores aos quais este computador se conecta.

Nível de Checksum

Selecione um nível de checksum. Os quatro níveis possíveis são:

Elemento Descrição
aceito O serviço estará ativo se o outro lado da conexão especificar "necessário" ou "solicitado" e houver um algoritmo compatível disponível do outro lado; caso contrário, ele estará inativo.
rejeitado O serviço não deve estar ativo e a conexão falhará se o outro lado especificar "necessário".
solicitado O serviço estará ativo se o outro lado da conexão especificar "aceito", "necessário" ou "solicitado" e houver um algoritmo compatível disponível do outro lado; caso contrário, ele estará inativo.
necessário O serviço deverá estar ativo e a conexão falhará se o outro lado especificar "rejeitado" ou se não houver um algoritmo compatível do outro lado.
