Protocolo

Juego de reglas que define cómo se transportan los datos por las redes. El Gestor de Red de Oracle soporta los siguientes protocolos:

IPC

La comunicación entre procesos (IPC) se utiliza para aplicaciones cliente en las misma computadora que el listener. IPC puede proporcionar una conexión local más rápida que TCP/IP.

Named Pipes

Named Pipes se utiliza en computadoras personales que no usan TCP/IP. Named Pipes sólo está disponible en Windows NT.

TCP/IP

Transmission Control Protocol/Internet Protocol (TCP/IP) es el protocolo Ethernet estándar de hecho utilizado para la conversación del servidor de base de datos/cliente en la red. TCP/IP permite que una aplicación Oracle de un cliente se comunique con bases de datos Oracle remotas mediante TCP/IP (si la base de datos Oracle se ejecuta en un sistema host que soporte la comunicación de red con TCP/IP).

Se trata del protocolo de redes estándar. Se utiliza de forma generalizada tanto en las redes para UNIX como para PC. Es también el protocolo con el que se realizan las conexiones a Internet.

TCP/IP con SSL

El protocolo TCP/IP con Secure Sockets Layer (SSL) permite que una aplicación Oracle de un cliente se comunique con bases de datos Oracle remotas con mediante TCP/IP y SSL (si la base de datos Oracle se ejecuta en un sistema host que soporte la comunicación de red con TCP/IP y SSL). TCP/IP con SSL sólo se utiliza con el método de autenticación SSL.

Nota: Puede que el sistema operativo soporte otros protocolos.
