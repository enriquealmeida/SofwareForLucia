Configuration d'un descripteur de connexion pour les connexions à Oracle Connection Manager

Pour acheminer les clients vers le serveur de base de données via Oracle Connection Manager, configurez soit le fichier tnsnames.ora soit le serveur d'annuaire avec un descripteur de connexion indiquant l'adresse de protocole d'Oracle Connection Manager. Cette adresse permet au client de se connecter à l'ordinateur d'Oracle Connection Manager.

Pour configurer une adresse de protocole pour Oracle Connection Manager :

  1. Dans le panneau du navigateur, développez Oracle Net Configuration > Annuaire ou Local > Résolution de noms de service.

  2. Sélectionnez plus (+) dans la barre d'outils ou choisissez Modifier > Créer dans la barre de menus. La page Bienvenue de l'assistant Nom de service réseau apparaît.

  3. Entrez un nom dans le champ Nom de service réseau.

  4. Cliquez sur Suivant. La page Protocole apparaît.

  5. Sélectionnez le protocole d'écoute qu'Oracle Connection Manager utilise. Par défaut, il s'agit du protocole TCP/IP.

  6. Cliquez sur Suivant. La page Paramètres de protocole apparaît.

  7. Entrez dans les champs prévus à cet effet les paramètres de protocole adéquats pour le protocole choisi. Pour TCP/IP, le port par défaut est 1630.

  8. Cliquez sur Suivant. La page Service apparaît.

  9. Si le service de destination est Oracle8i ou supérieur, sélectionnez Oracle8i ou supérieur, et saisissez un nom de service dans le champ Nom de service. Si le service de destination est Oracle8 ou Oracle7, sélectionnez Oracle8 ou version antérieure, et entrez un identificateur système Oracle dans le champ SID de base de données.

  10. Cliquez sur Suivant. La page Service apparaît.

  11. Cliquez sur Fin pour enregistrer votre configuration et quitter l'assistant Nom de service réseau.

  12. Choisissez Fichier > Enregistrer la configuration réseau.

  13. Le nouveau nom de service réseau est ajouté au dossier Résolution de noms de service.

Pour plus d'informations sur l'activation complète des fonctionnalités Oracle Connection Manager, reportez-vous au chapitre Configuring and Administering Oracle Connection Manager dans le Guide de l'administrateur Oracle10i Net.
