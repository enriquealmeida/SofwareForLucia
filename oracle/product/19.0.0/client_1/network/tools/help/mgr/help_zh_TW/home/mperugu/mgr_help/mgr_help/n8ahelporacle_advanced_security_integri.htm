Oracle Advanced Security：完整性

您可以使用完整性頁籤來設定認證方法的資料完整性。資料完整性是指由從屬端與伺服器兩者的角度來看，對透過連線傳送的資料所做的比較。

注意：您可以使用 SSL 頁籤來設定「安全通訊端層 (SSL)」的 Cipher Suite。

完整性

請選取「伺服器」或「從屬端」核取方塊，以指示您目前要設定電腦是「從屬端」或「伺服器」。「伺服器」選項不會套用到此電腦連線的伺服器。

總和檢查層次

請選取總和檢查層次。可用的總和檢查層次有四種：

元素 描述
接受 如果連線的另一端為「必要」或「要求」，並且也有可用的相容演算法，則服務會在作用中；否則，服務將會是非作用中。
拒絕 服務必定不是作用中，如果另一端為「必要」，連線將會失敗。
要求 如果連線的另一端指定為「接受」、「必要」或「要求」，並且也有可用的相容演算法，則服務會在作用中；否則，服務將為非作用中。
必要 服務必須為作用中，如果另一端為「拒絕」，或者沒有可用的相容演算法，連線將會失敗。

可用的方法清單

顯示用於建立總和檢查的完整性演算法。類似的方法有二種：

演算法名稱 有效值
安全雜湊演算法 (Secure Hash Algorithm, SHA-1) SHA1
訊息摘要 5 (Message Digest 5, MD5) MD5

顯示選取的演算法。清單中的演算法順序會決定使用的順序。
