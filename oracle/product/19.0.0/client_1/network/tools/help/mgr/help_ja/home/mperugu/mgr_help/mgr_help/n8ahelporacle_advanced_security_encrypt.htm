Oracle Advanced Security: 暗号化

「暗号化」タブを使用すると、選択した認証方式の暗号化パラメータを構成できます。

注意: Secure Sockets Layer (SSL)の暗号スイートを構成する場合は、「SSL」タブを使用してください。

暗号化

リストから、現在構成しているコンピュータの種類(「クライアント」または「サーバー」)を選択します。

暗号化タイプ

値を選択し、暗号化および整合性のネゴシエーション時のクライアントまたはサーバーの動作を指定します。次の4つの暗号化タイプから選択できます。

要素 説明
適用 一方の接続側で「必要」または「要求」が指定されており、もう一方の接続側で矛盾のないアルゴリズムが使用可能になっている場合、サービスはアクティブになります。それ以外の場合、サービスはアクティブにはなりません。
拒否 サービスはアクティブにはなりません。一方の接続側で「必要」が指定されている場合、接続は失敗します。
要求 一方の接続側で「適用」、「必要」または「要求」が指定されており、矛盾のないアルゴリズムが使用可能になっている場合、サービスはアクティブになります。それ以外の場合、サービスはアクティブにはなりません。
必要 サービスはアクティブになります。一方の接続側で「拒否」が指定されている場合、または両立するアルゴリズムがない場合、接続は失敗します。

暗号化シード

「暗号化シード」フィールドに、暗号化シード、つまり256文字以内の任意の文字列を入力します。暗号化シードは、暗号キーの生成に使用されます。これは、暗号化または整合性が使用可能になっている場合に必要です。

注意: 暗号化シード・パラメータの一部にカンマ「,」、右括弧「)」などの特殊文字を使用する場合は、一重引用符で値を囲むことをお薦めします。

「使用可能なメソッド/選択メソッド」リスト

選択可能な認証方式と、選択された暗号化メソッドが表示されます。リスト内のメソッドの順序によって、ネゴシエーション時の優先順序が決まります。

左矢印ボタン(「<」)と右矢印ボタン(「>」)を使用して、選択した暗号化アルゴリズムをリスト間で移動します。Oracle Advanced Securityでは、次の暗号化アルゴリズムがサポートされます。

要素 説明
AES256 AES 256ビット・キー
RC4_256 RC4 256ビット・キー
AES192 AES 192ビット・キー
3DES168 3キー(168ビット)オプションによるトリプルDES
AES128 AES 128ビット・キー
RC4_128 RC4 128ビット・キー
3DES112 2キー(112ビット)オプションによるトリプルDES
DES DES 56ビット・キー
RC4_40 RC4 40ビット・キー
DES40 DES 40ビット・キー
