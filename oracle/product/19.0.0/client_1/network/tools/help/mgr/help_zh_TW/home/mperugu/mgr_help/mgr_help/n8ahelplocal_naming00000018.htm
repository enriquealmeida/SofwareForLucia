區域命名簡介

區域命名是主要的命名方法之一，用來將簡單名稱 (一種網路服務名稱)，解析成從屬端連線資料庫或服務時所使用的連線描述區。區域命名會將網路服務名稱及其連線描述區，儲存在區域 tnsnames.ora 檔案中。

一般使用者輸入的連線字串包含以下格式的網路服務名稱：

CONNECT username/password@net_service_name

在以下的範例中，sales 是在 tnsnames.ora 檔案中查詢的網路服務名稱：

CONNECT scott/tiger@sales

傳回從屬端的連線描述區包括：

  • 監聽器的協定位址位置

  • 資料庫的服務名稱

從屬端使用此資訊來連線監聽器。設定相同位址資訊的監聽器會接收連線要求。如果監聽器監聽到在從屬端連線描述區中指定的相同服務，就會授予連線。

如需詳細資訊，請參閱 Oracle10i Net Administrator's Guide 中的 Connectivity Concepts 章節。
