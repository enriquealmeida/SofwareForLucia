Oracle Net Services構成の概要

Oracle Net Managerを使用すると、ネーミング・メソッド、クライアント/サーバー・プロファイル、リスナーなど、多数のネットワーク要素を構成できます。

ネーミング・メソッド

サービス(たとえば、データベース)の位置を識別する単純名を定義します。これらの単純名は接続識別子と呼ばれ、サービスのネットワーク位置と識別を含む接続記述子にマップされます。

Oracle Net Managerを使用すると、ネーミング・メソッドによる接続識別子から接続記述子への様々な変換方法を構成できます。

クライアント/サーバー・プロファイル

プロファイルを使用して、クライアント/サーバーでOracle Netの機能を使用可能にし、構成するための作業環境を設定します。プロファイルを使用すると、ネーミング・メソッドの優先順位を付ける作業環境、Oracle Advanced Security機能を使用可能にする作業環境、ロギングおよびトレース、またはルート接続を可能にする作業環境を設定できます。

サーバー上のリスナー構成

クライアント接続を受信するように、リスナーを構成します。リスナーの構成時には、次のものを指定する必要があります。

  • リスニングする1つ以上のプロトコル・アドレス

  • 接続先サービスに関するサービス情報

プロトコル・アドレスは、リスナー・コンフィギュレーション・ファイルlistener.oraで構成されます。サービス情報は、listener.oraファイルに構成される場合とされない場合があります。

  • Oracle8i以上のデータベースでは、サービス名、インスタンス名、サービス・ハンドラ、およびロード情報などの特定の情報は、自動的にリスナーに登録されます。この機能はサービス登録と呼ばれ、listener.oraファイルでの構成を必要としません。

  • Oracle7およびOracle8データベースなど、その他のサービスは、listener.oraファイル内にサービスを構成する必要があります。

Oracle Namesの詳細は、「Oracle Namesの概要」を参照してください
