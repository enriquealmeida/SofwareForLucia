ネット・サービス別名の作成

注意:

  • ディレクトリにネット・サービス別名エントリを作成できるのは、OracleNetAdminsまたはOracleContextAdminsグループのメンバーであるユーザーのみです。

  • Microsoft Active Directoryを使用している場合、ネット・サービス別名はサポートされません。

  • Oracle Names Serverから既存の別名オブジェクトをインポートできます。詳細は、Oracle10i Net Services管理者ガイドの「ディレクトリ・サーバーへのネーミング・データのエクスポート」の章を参照してください。

ディレクトリ・サーバーのディレクトリ・ネーミング・オブジェクト(ネット・サービス名またはデータベース・サービス)にネット・サービス別名を作成できます。ネット・サービス別名は、参照先のオブジェクトの名前を格納します。

クライアントがネット・サービス別名のディレクトリ検索を要求すると、ディレクトリは、エントリがネット・サービス別名であることを判断し、参照エントリであるかのように検索を完了します。たとえば、salesというネット・サービス名に対してsalesaliasというネット・サービス別名を作成できます。データベース・サービスに接続する場合に、CONNECT scott/tiger@salesaliasのようにsalesaliasを使用すると、salesの接続記述子情報が解決および使用されます。

ネット・サービス名の別名には次の2つの主要な使用方法があります。

  • クライアントが別の名前でデータベース・サービスまたはネット・サービス名を参照する場合に、ネット・サービス名の別名を使用します。

  • 異なるOracleコンテキストのデータベース・サービスまたはネット・サービス名に対して、1つのOracleコンテキストのネット・サービス別名を付けます。このようにした場合、データベース・サービスまたはネット・サービス名をディレクトリ・サーバーに定義すると、他のOracleコンテキストからも使用できます。

ネット・サービス別名を作成する手順は、次のとおりです。

  1. ナビゲータ・ペインで、「Oracle Netの構成」→「ディレクトリ」→「サービス・ネーミング」→「別名」を展開します。

  2. ツールバーのプラス(+)ボタンを選択するか、メニュー・バーから「編集」→「作成」を選択します。「ネット・サービス別名」ダイアログ・ボックスが表示されます。

  3. 「ネット・サービス別名」フィールドに、ネット・サービスの別名を入力します。

  4. 「ネット・サービスまたはデータベース・サービス」ボックスで、ネット・サービス名またはデータベース・サービスについての次の情報を入力します。

    • Oracleコンテキスト: リストから選択するか、フィールドに入力して、データベース・サービスまたはネット・サービス名のOracleコンテキストを指定します。

    • 名前: リストから選択するか、フィールドに入力して、データベース・サービスまたはネット・サービス名の識別名(DN)を指定します。

  5. 「作成」を選択します。
