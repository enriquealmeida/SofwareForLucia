Connection pooling residente del database

Il connection pooling residente del database fornisce la funzionalità di pooling delle connessioni "dedicate" per le applicazioni o i processi client. Oltre che alla propria sessione del database, una connessione dedicata fa riferimento a un server dedicato. Utilizzare il connection pooling residente del database quando per più applicazioni client, quali ad esempio i livelli intermedi, è necessario gestire connessioni al database persistenti oppure quando è necessario ottimizzare risorse del server quali la memoria.

Nel connection pooling residente del database i client sono connessi a un processo di background, denominato Connection Broker, che fornisce la funzionalità di pooling.
