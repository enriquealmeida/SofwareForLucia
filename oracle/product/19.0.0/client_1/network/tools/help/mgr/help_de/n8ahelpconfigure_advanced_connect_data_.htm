Optionen für Erweiterte Verbindungsdaten konfigurieren

Neben dem Service Namen oder Oracle System Identifier (SID) der Datenbank können Sie optional den Abschnitt CONNECT_DATA eines Verbindungsdeskriptors mit den folgenden Optionen konfigurieren:

  • Instanzname zur Anmeldung bei einer bestimmten Datenbankinstanz

  • Oracle-Standardverzeichnis der Datenbank, aus dem die Datenbank netzwerkübergreifend hochgefahren werden kann

  • Größe der Dateneinheit der Session zur Angabe der Puffergröße

  • Dedizierter Server zur Anmeldung bei einem dedizierten Server-Prozess

  • Heterogene Dienste für den Zugriff auf ein fremdes System

  • Oracle Rdb-Einstellungen für den Zugriff auf eine Oracle Rdb-Datenbank

So konfigurieren Sie erweiterte Verbindungsoptionen für einen Net Service-Namen:

  1. Blenden Sie im Navigator-Fensterbereich Oracle Net-Konfiguration> Verzeichnis> Lokal oder Dienstebenennung ein.

  2. Wählen Sie einen Net Service-Namen oder einen Datenbankdienst. Im rechten Fensterbereich werden der aktuelle Zieldienst und die Adressliste angezeigt.

  3. Klicken Sie im Feld Dienstidentifikation auf Erweitert. Daraufhin wird das Dialogfeld Erweiterte Dienstoptionen angezeigt.

  4. Geben Sie Informationen in die Felder ein, oder wählen Sie die entsprechenden Optionen, und klicken Sie auf OK.

  5. Wählen Sie Anwenden im rechten Fensterbereich.

  6. Wenn Sie diese Änderungen an dem Ordner Lokal vornehmen, wählen Sie Datei > Netzwerkkonfiguration speichern.
