Panoramica sulla denominazione delle directory

La denominazione delle directory è uno dei principali metodi di denominazione utilizzati per risolvere un identificativo di connessione nel descrittore di connessione che serve al client per connettersi a un database. Con la denominazione delle directory, gli identificativi di connessione e i relativi descrittori di connessione vengono memorizzati in un server delle directory conforme al protocollo LDAP. Gli identificativi di connessione vengono memorizzati in una struttura secondaria del contesto Oracle che include le voci da utilizzare con il software Oracle.

Un utente finale immette una stringa di connessione che include l'identificativo di connessione nel seguente formato:

CONNECT username/password@connect_identifier

Nell'esempio che segue, sales è l'identificativo di connessione cercato nella directory:

CONNECT scott/tiger@sales

Il descrittore di connessione restituito al client include:

  • la posizione dell'indirizzo di protocollo del listener;

  • il nome di servizio del database.

Il client utilizza le informazioni sull'indirizzo di protocollo per connettersi a un listener. Il client listener, configurato con lo stesso indirizzo, riceve la richiesta. Se il listener sta ascoltando le richieste sullo stesso servizio di database, secondo quanto specificato nel descrittore di connessione, la connessione potrà essere stabilita.

Per utilizzare la denominazione delle directory è necessario configurare un server delle directory per le funzioni di Oracle Net e configurare l'accesso alle directory da parte dei client.

Per ulteriori informazioni, vedere il capitolo Configuration Management Concepts del manuale Oracle10i Net Administrator's Guide.
