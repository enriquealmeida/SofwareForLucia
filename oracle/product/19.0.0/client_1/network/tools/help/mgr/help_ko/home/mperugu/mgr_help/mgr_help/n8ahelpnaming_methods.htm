이름 지정: 방법

방법 탭을 사용하여 해당 클라이언트 컴퓨터가 사용할 이름 지정 방법 및 해당 우선 순위를 지정할 수 있습니다. 목록에 제일 위에 있는 이름 지정 방법이 가장 먼저 접속 식별자, 네트 서비스 이름 또는 데이터베이스 서비스의 분석에 사용됩니다. 첫번째 이름 지정 방법이 이름을 분석하지 못하면 목록에 지정된 다음 방법이 사용됩니다.

클라이언트는 접속 요청을 할 때 접속 기술자에 대해 접속 식별자를 분석하기 위한 첫번째 이름 지정 방법에 접속을 시도합니다. 그런 다음 클라이언트 요청은 접속 기술자에 지정된 리스너 주소로 해당 요청을 전송합니다.

Oracle Net은 다음과 같은 이름 지정 방법을 지원합니다.

  • 로컬 이름 지정(TNSNAMES)

  • 디렉토리 이름 지정(LDAP)

  • 호스트 이름 지정(HOSTNAME)

  • 쉬운 접속 이름 지정(EZCONNECT)

  • 네트워크 정보 서비스(NIS)

기본 도메인

대부분의 클라이언트 요청이 발생하는 네트워크 도메인의 이름을 입력합니다. 이 도메인 이름은 클라이언트가 상주하는 도메인이나 클라이언트가 네트워크 서비스를 자주 요청하는 도메인이 될 수 있습니다.

도메인이 설정되면 도메인 이름은 비검증 이름에 자동으로 추가됩니다.
