Locais de Atendimento

Os listeners são configurados para atender as solicitações de conexão recebidas referentes a um banco de dados em um endereço de protocolo de rede específico. A guia Locais de Atendimento permite que você configure um ou mais endereços de locais de atendimento nos quais um listener aguardará conexões de clientes.

Protocolo

Na lista, selecione o protocolo para o endereço. Dependendo do protocolo escolhido, especifique informações sobre parâmetros de protocolo nos campos apropriados.

Botão Mostrar Avançado/Ocultar Avançado

Use a alternância Avançado/Ocultar Avançado para exibir ou ocultar configurações para especificar o limite de espaço no buffer de E/S para o listener.

Essas definições são suportadas pelos protocolos TCP/IP, TCP/IP com SSL e SDP.

  • Tamanho Total do Buffer de Envio

Especifique, em bytes, o espaço do buffer para operações de envio das sessões.

Como o servidor de banco de dados grava dados nos clientes, esta definição no lado do servidor é normalmente adequada. Se o servidor de banco de dados estiver recebendo um grande volume de solicitações, defina também o Tamanho Total do Buffer de Recepção.

  • Tamanho Total do Buffer de Recepção

Especifique, em bytes, o espaço do buffer para operações de recebimento das sessões.

Dedique esse endereço estaticamente para conexões JServer

Opte por configurar o arquivo listener.ora com pontos finais General Inter-Orb Protocol (GIOP) para conexões Oracle JServer release 8.1.

Adicionar Endereço

Escolha para criar um novo endereço de protocolo . É exibida uma nova guia Endereço , na qual você pode selecionar o protocolo e especificar informações sobre o protocolo nos campos.

Remover Endereço

Escolha para deletar a guia Endereço selecionada e as respectivas informações de endereço.

Para obter mais informações sobre como configurar o espaço de buffer de Entrada/Saída, consulte o capítulo Optimizing Performance no Oracle10i Net Administrator's Guide.
