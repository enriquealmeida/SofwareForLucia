일반 매개변수: 로깅 및 추적

로깅 및 추적 탭을 사용하여 리스너에 대한 로깅 및 추적을 설정할 수 있습니다. Oracle Net Manager를 사용하여 로그 파일을 작성할 위치와 이름을 지정할 수 있습니다.

로깅이 사용 안함으로 설정됨

리스너의 로깅을 사용 안함으로 설정합니다.

로깅이 사용으로 설정됨

리스너에 대한 로깅을 사용으로 설정합니다. 로깅을 활성화하면 로그 정보를 기록할 위치를 지정할 수 있습니다.

로그 파일

리스너의 로그 정보를 작성할 디렉토리 경로명과 파일 이름을 입력합니다. 이 파일에 대한 파일 이름 확장자는 항상 .log입니다. 기본 이름은 listener.log입니다. 기본 디렉토리는 UNIX의 경우 $ORACLE_HOME/network/log이며 Windows 플랫폼의 경우 ORACLE_HOME\network\log입니다.

추적 해제

리스너의 추적을 해제합니다.

추적 설정

리스너에 대한 추적을 활성화합니다. 추적을 활성화하면 원하는 추적 레벨과 기본 추적 파일을 변경하여 추적 정보 기록 위치를 지정할 수 있습니다.

추적 레벨

목록에서 추적 레벨을 선택합니다.

Off--추적이 사용 안함으로 설정됩니다. [OFF]가 기본 설정입니다.

User--추적이 사용자에게 적합한 레벨로 설정됩니다. 사용자가 생성한 오류 상태를 확인하기 위해 추적합니다.

Administrator--추적이 데이터베이스 관리자에게 적합한 레벨로 설정됩니다. 설치 관련 문제를 확인하기 위해 추적합니다.

Support--추적이 고객 지원에 적합한 레벨로 설정됩니다.

추적 파일은 매우 커질 수 있습니다. 따라서 Oracle은 네트워크 문제를 해결하기 위한 경우가 아니면 추적 기능을 사용하지 않을 것을 권장합니다.

추적 파일

리스너의 추적 정보를 작성할 디렉토리 경로명과 파일 이름을 입력합니다. 이 파일에 대한 파일 이름 확장자는 항상 .trc입니다. 기본 이름은 listener.trc입니다. 기본 디렉토리는 UNIX의 경우 $ORACLE_HOME/network/trace이고, Windows 플랫폼의 경우 ORACLE_HOME\network\trace입니다.
