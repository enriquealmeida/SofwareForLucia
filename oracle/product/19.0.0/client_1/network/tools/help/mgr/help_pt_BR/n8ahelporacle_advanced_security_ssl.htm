Oracle Advanced Security: SSL

A guia SSL permite que você modifique definições da SSL (Secure Sockets Layer). A SSL é um protocolo padrão do mercado que proporciona comunicações de rede seguras. A SSL oferece autenticação, criptografia e integridade de dados. Use a SSL para obter comunicações seguras entre quaisquer clientes e servidores. Especificamente, você pode usar a SSL para autenticar qualquer cliente ou servidor para um ou mais servidores Oracle ou para autenticar um servidor Oracle para qualquer cliente.

Verificação de Revogação (Servidor somente)

Especifique uma verificação de revogação para um certificado. Selecione um dos seguintes valores:

  • Nenhum: selecione para desativar a verificação de revogação de certificados.

  • Necessário: Selecione para executar a revogação de certificados quando um certificado estiver disponível. Se um certificado for revogado e não for encontrada a CRL ("Certificate Revocation List", Lista de Revogação de Certificados), rejeite a conexão SSL. Se não for encontrada a CRL para verificar o status de revogação do certificado e ele não for revogado, aceite a conexão SSL.

  • Solicitado: Selecione para executar a revogação de certificados caso haja uma CRL disponível. Rejeite a conexão SSL se o certificado for revogado. Se não for encontrada a CRL apropriada para determinar o status de revogação do certificado e ele não for revogado, aceite a conexão SSL

Exigir a Versão da SSL (opcional)

Na lista, selecione a versão da SSL. O cliente e o servidor devem usar versões compatíveis de SSL. Você poderá selecionar SSL v3.0 ou optar por permitir que qualquer versão existente ou futura de SSL seja utilizada.

Requer Autenticação de Cliente (Apenas servidor)

Esta caixa de seleção é selecionada por default. Desmarque-a se não for necessária a autenticação por parte do cliente.

Fazer a correspondência com o nome do servidor X.509 (Apenas cliente)

Na lista, indique se deve ser imposto que o DN (Distinguished Name – Nome Distinto) do servidor coincida com o nome de serviço correspondente. Se você impor a verificação de correspondência, o SSL garantirá que o certificado é proveniente do servidor. Caso a verificação de correspondência não seja imposta, o SSL fará a verificação, mas permitirá a conexão, independentemente de haver uma correspondência. A não-imposição de uma correspondência permite que o servidor falsifique a identidade. Selecione um dos seguintes valores:

  • Sim: Selecione para verificar o DN do servidor. Se o DN corresponder ao nome do serviço, a conexão será bem-sucedida. Se o DN não corresponder ao nome do serviço, a conexão será bem-sucedida, mas um erro será registrado no arquivo sqlnet.log.

  • Não: Selecione para não verificar o DN do servidor. Ignorar esta verificação pode permitir que o servidor falsifique sua identidade.
