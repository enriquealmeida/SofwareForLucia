Panoramica sulla denominazione locale

La denominazione locale è uno dei metodi di denominazione principali utilizzati per risolvere un nome semplice, un nome di servizio di rete, in un descrittore di connessione utilizzato dal client per connettersi a un database o a un servizio. La denominazione locale memorizza i nomi dei servizi di rete e i relativi descrittori di connessione in un file tnsnames.oralocale.

Un utente finale immette una stringa di connessione che include il nome di servizio di rete nel seguente formato:

CONNECT username/password@net_service_name

Nell'esempio che segue, sales è il nome di servizio di rete cercato nel file tnsnames.ora:

CONNECT scott/tiger@sales

Il descrittore di connessione restituito al client include:

  • la posizione dell'indirizzo di protocollo del listener;

  • il nome di servizio del database

Il client utilizza queste informazioni per connettersi a un listener. Quest'ultimo, configurato con le stesse informazioni sull'indirizzo, riceve la richiesta. Se il listener sta effettuando le operazioni di ascolto per le richieste sullo stesso servizio di database specificato nel descrittore di connessione del client, la connessione viene assegnata.

Per ulteriori informazioni, vedere il capitolo Connectivity Concepts del manuale Oracle10i Net Administrator's Guide.
