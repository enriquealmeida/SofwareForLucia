リスナーの概要

サービスは、リスナーの助けを借りてサービス・セッションを調整します。これは、サーバー上で行われるプロセスで、クライアント・アプリケーションにかわって接続リクエストを受信します。リスナーは、データベース・サービスまたはデータベース以外のサービスに対するプロトコル・アドレスをリスニングするように構成されています。

プロトコル・アドレスは、データベース・サービスまたはデータベース以外のサービスに対して、リスナー・コンフィギュレーション・ファイルlistener.ora内で構成されます。Oracle9iおよびOracle8iのデータベース・サービスでは、サービス名、インスタンス名、ロード情報などが、リスナーに自動的に登録されます。この機能はサービス登録と呼ばれるもので、listener.oraファイルでサービスを構成する必要がありません。Oracle8およびOracle7のデータベース・サービス、およびOracle Enterprise Manager管理ツールの場合は、listener.oraファイルでサービスを構成する必要があります。

リスナーを構成した後は、リスナー制御ユーティリティで管理できます。
