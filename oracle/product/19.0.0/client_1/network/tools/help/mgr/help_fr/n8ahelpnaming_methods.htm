Résolution de noms : Méthodes

L'onglet Méthodes vous permet d'indiquer les méthodes de résolution de noms que ce client doit utiliser, ainsi que l'ordre de priorité. La méthode du haut de la liste est utilisée la première pour résoudre un identificateur de connexion, un nom de service réseau ou un service de base de données. Si elle ne peut pas résoudre le nom, la prochaine méthode de la liste est alors utilisée.

Lors d'une demande de connexion, le client tente d'appeler la première méthode de résolution de noms répertoriée pour convertir l'identificateur de connexion en descripteur de connexion. Le client achemine ensuite sa demande à l'adresse du processus d'écoute indiquée dans le descripteur de connexion.

Oracle Net assure la prise en charge des méthodes de résolution de noms suivantes :

  • Résolution de noms locaux (TNSNAMES)

  • Résolution de noms de répertoire (LDAP)

  • Résolution de noms d'hôte (HOSTNAME)

  • Résolution de noms Easy Connect (EZCONNECT)

  • Network Information Service (NIS)

Liste des méthodes sélectionnées

Affiche les méthodes sélectionnées. Leur ordre dans la liste détermine l'ordre dans lequel Oracle Net les essaiera.

< bouton

Cliquez sur ce bouton pour déplacer la méthode de résolution de noms sélectionnée de la liste Méthodes disponibles vers la liste Méthodes sélectionnées.

Bouton >

Cliquez sur ce bouton pour déplacer la méthode de résolution de noms sélectionnée de la liste Méthodes sélectionnées vers la liste Méthodes disponibles.

Bouton Promouvoir

Cliquez sur ce bouton pour déplacer vers le haut la méthode de résolution de noms sélectionnée dans la liste Méthodes sélectionnées. Oracle Net tente d'utiliser la première méthode pour résoudre un nom de service réseau ou de service de base de données. S'il n'y parvient pas, il utilise la deuxième méthode, etc.

Bouton Rétrograder

Cliquez sur ce bouton pour déplacer vers le bas la méthode de résolution de noms sélectionnée dans la liste Méthodes sélectionnées.

Domaine par défaut

Indiquez le nom du domaine réseau à partir duquel sont effectuées la plupart des demandes client. Ce peut être le nom du domaine où le client réside ou de celui à partir duquel il émet fréquemment des demandes de service réseau.

Une fois défini, le nom de domaine est automatiquement ajouté à un nom non qualifié.
