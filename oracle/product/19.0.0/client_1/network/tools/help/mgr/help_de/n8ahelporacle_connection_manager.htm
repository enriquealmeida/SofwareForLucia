Oracle Connection Manager

Ein Router, über den die Anforderung einer Clientverbindung entweder an den nächsten Hop oder direkt an den Datenbank-Server gesendet werden kann. Clients, die ihre Verbindungsanforderungen über einen Oracle Connection Manager weiterleiten, können dann die Funktionen wie Zugriffskontrolle, Multiplexing und Protokollkonvertierung nutzen, die auf diesem Oracle Connection Manager konfiguriert sind.

Oracle Connection stellt Informationen für das Verbindungs-Routing bereit, wenn:

  • eine große Anzahl von Benutzern auf einen einzelnen Dienst über ein einzelnes Protokoll zugreifen müssen.

  • verschiedene Protokolle auf dem Client und Server installiert sind, sodass eine typische Verbindung nicht erfolgreich verläuft.

  • typische Oracle Net-Verbindungen fordern, dass für den Client und Server dasselbe Protokoll installiert ist.

  • die Kontrolle des Client-Zugriffs zu angegebenen Servern in einer TCP/IP-Umgebung erforderlich ist.

Informationen zu Oracle Connection Manager finden Sie im Kapitel Networking Challenges in the Internet Age im Oracle9i Net Services Administrator's Guide.
