Oracle Connection Managerへの接続に使用する接続記述子の構成

Oracle Connection Managerでクライアントをデータベース・サーバーに転送するには、Oracle Connection Managerのプロトコル・アドレスを指定する接続記述子を使用して、tnsnames.oraファイルまたはディレクトリ・サーバーを構成します。このアドレスを使用して、クライアントからOracle Connection Managerコンピュータに接続できます。

Oracle Connection Manager用にプロトコル・アドレスを構成する手順は、次のとおりです。

  1. ナビゲータ・ペインで、「Oracle Netの構成」→「ディレクトリ」または「ローカル」→「サービス・ネーミング」を展開します。

  2. ツールバーのプラス(+)ボタンを選択するか、メニュー・バーから「編集」→「作成」を選択します。Netサービス名ウィザードのようこそページが表示されます。

  3. 「ネット・サービス名」フィールドに名前を入力します。

  4. 「次へ」を選択します。プロトコル・ページが表示されます。

  5. Oracle Connection Managerがリスニングするように構成されているプロトコルを選択します。デフォルトのプロトコルは、TCP/IPです。

  6. 「次へ」を選択します。プロトコル設定ページが表示されます。

  7. 表示されるフィールドに、選択したプロトコルに適切なプロトコル・パラメータを入力します。TCP/IPを使用する場合、デフォルトのポートは1630です。

  8. 「次へ」を選択します。サービス・ページが表示されます。

  9. 接続先サービスがOracle8i以上である場合は、「Oracle8i以上」を選択して「サービス名」フィールドにサービス名を入力します。接続先サービスがOracle8またはOracle7である場合は、「Oracle8以下」を選択して「データベースSID」フィールドにOracleシステム識別子を入力します。

  10. 「次へ」を選択します。サービス・ページが表示されます。

  11. 「終了」をクリックして構成を保存し、「Netサービス名ウィザード」を終了します。

  12. 「ファイル」→「ネットワーク構成の保存」を選択します。

  13. 新しいネット・サービス名が「サービス・ネーミング」フォルダに追加されます。

Oracle Connection Manager機能の完全な有効化の詳細は、Oracle10i Net管理者ガイドの「Oracle Connection Managerの構成と管理」の章を参照してください
