リスナー・アドレスの構成

リスナーは、1つ以上のプロトコル・アドレスを指定して構成する必要があります。リスナーは、同じアドレスで構成されたクライアントからの接続要求を受信します。

リスナーのプロトコル・アドレスを構成する手順は、次のとおりです。

  1. ナビゲータ・ペインで、「Oracle Netの構成」→「ローカル」→「リスナー」を展開します。

  2. リスナーを選択します。

  3. 右ペインのリストから「リスニング位置」を選択します。

  4. 「アドレスの追加」を選択します。新しい「アドレス」タブが追加されます。

  5. プロトコルを選択し、そのプロトコルでの適切なプロトコル・パラメータ情報を入力します。

    リスナーをTCP/IP上で、Oracle Netクライアントをリスニングするように構成する場合は、デフォルト・ポート、つまり1521を選択する必要があります。それ以外の場合は、初期化パラメータ・ファイルでLOCAL_LISTENERパラメータを構成し、ネーミング・メソッドを使用してリスナー名を解決する必要があります。

  6. コンピュータに複数のIPアドレスがあり、使用可能なすべてのIPアドレスをリスナーでリスニングする場合は、TCP/IPまたはSSL付きTCP/IPを構成し、「ホスト」フィールドにコンピュータのホスト名を入力します。

  7. 必要な場合は、「詳細の表示」をクリックし、セッションの送受信操作に使用するI/Oバッファ容量を設定します。

  8. 「ファイル」→「ネットワーク構成の保存」を選択します。

デフォルト以外のポートの構成の詳細は、Oracle10i Net管理者ガイドの「リスナーの構成および管理」の章を参照してください。

I/Oバッファ容量の構成の詳細は、Oracle10i Net管理者ガイドの「パフォーマンスの最適化」の章を参照してください。
