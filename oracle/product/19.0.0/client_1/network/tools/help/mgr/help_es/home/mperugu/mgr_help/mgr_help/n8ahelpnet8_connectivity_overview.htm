Visión General de la Conectividad de Red de Oracle

La Red de Oracle permite conexiones a distintos servicios, como bases de datos Oracle u otras bases de datos, gateways y procedimientos externos.

Los usuarios inician una solicitud de conexión mediante el envío de un nombre de usuario y una contraseña junto con un identificador del servicio al que se desean conectar. Dicho identificador, denominado identificador de conexión, identifica:

  • El servicio de destino.

  • La ruta de acceso o ruta de red, en la red para obtener dicho servicio.

Los identificadores de conexión se especifican de varias formas. Una de las más comunes es mediante el uso de un nombre del servicio de red, otro nombre para el servicio, que se corresponde con un descriptor de conexión. Estos descriptores pueden contener información de la ruta de red y del servicio de destino. El servicio deseado se indica mediante el nombre de servicio de las bases de datos Oracle9i u Oracle8i o el identificador del sistema Oracle (SID) de las bases de datos Oracle8 u Oracle7. La ruta de red proporciona, como mínimo, la ubicación del listener mediante una dirección de red.

A continuación, se muestra un servicio de red denominado sales asignado a un descriptor de conexión:

sales=

(DESCRIPTION=

(ADDRESS=(PROTOCOL=tcp)(HOST=sales-server1)(PORT=1521))

(CONNECT_DATA=(SERVICE_NAME=sales.us.example.com))

El listener, ubicado en sales1-server, recibe con el protocolo TCP/IP en el puerto 1521 las solicitudes de conexión de un servicio de base de datos denominado sales.us.example.com.

El identificador de conexión y el descriptor de conexión correspondiente están almacenados en al menos un método de nomenclatura. Durante la solicitud de conexión, el cliente resuelve el identificador de conexión (sales en el ejemplo anterior) en un descriptor de conexión mediante un método de nomenclatura. A continuación, el cliente entra en contacto con el listener especificado en el descriptor de conexión.

El listener, mediante un protocolo, acepta la conexión del cliente. Compara la información del cliente con la recibida del servicio de base de datos mediante el registro del servicio, así como la información que ha almacenado en su propio archivo de configuración, listener.ora. Si la información coincide, se otorga una conexión.
