命名：方法

您可以使用方法頁籤來指定從屬端電腦要使用的命名方法以及優先順序。清單最上方的命名方法會先用來解析指定的連線 ID、網路服務名稱或資料庫服務。如果無法解析名稱，則會使用在清單中指定的下一個方法。

連線要求期間，從屬端會嘗試連繫列出的第一個命名方法，將連線 ID 解析成連線描述區。接下來從屬端會將要求轉送至連線描述區中指定的監聽器位址。

Oracle Net 支援以下命名方法：

  • 區域命名(TNSNAMES)

  • 目錄命名(LDAP)

  • 主機命名(HOSTNAME)

  • 簡易連線命名(EZCONNECT)

  • 網路資訊服務(NIS)

顯示選取的命名方法。清單中的命名方法順序會決定 Oracle Net 嘗試命名方法的順序。

< 按鈕

選擇此按鈕可從「可用的方法」清單中，將選取的命名方法移至「選取的方法」清單。

> 按鈕

選擇此按鈕可從「選取的方法」清單中，將選取的命名方法移至「可用的方法」清單。

「升級」按鈕

選擇此按鈕可在「選取的方法」清單中，將選取的命名方法往上移。Oracle Net 會嘗試使用第一個方法來解析網路服務名稱或資料庫服務名稱。如果無法使用第一個方法，就會使用二個方法，依此類推。

「降級」按鈕

選擇此按鈕可在「選取的方法」清單中，將選取的命名方法往下移。

預設網域

輸入發生大部分從屬端要求的網路網域名稱。網域名稱可以是從屬端所在的網域，或者是從屬端經常要求網路服務的網域。

如果設定，網域名稱會自動附加至任何不完整的名稱。
