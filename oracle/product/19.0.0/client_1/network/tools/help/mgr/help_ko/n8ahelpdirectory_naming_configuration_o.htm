디렉토리 이름 지정 구성 개요

이 항목에는 디렉토리 이름 지정 구성에 대한 세부 정보가 나와 있습니다.

설치 시 서버 구성

사용자 정의 서버 설치 중 Oracle Net Configuration Assistant는 다음을 수행하라는 메시지를 표시합니다.

  • 디렉토리 서버 유형 선택

  • 디렉토리 위치 식별

  • 서버가 접속 기술자를 검색, 생성 및 수정할 수 있는 Oracle Context 선택 또는 생성

  • 사용할 선호 이름 지정 방법으로 디렉토리 이름 지정 구성

디렉토리 사용이 제대로 구성된 경우 Database Configuration Assistant는 디렉토리에 접속하여 디렉토리에 데이터베이스 서비스 항목을 등록하려고 합니다. 데이터베이스 서비스 항목은 다음에 대한 네트워크 정보를 포함하는 접속 기술자로 구성됩니다.

  • 리스너 주소

  • 데이터베이스 서비스 이름

설치 시 클라이언트 구성

모든 클라이언트 설치 중 Oracle Net Configuration Assistant는 다음과 같은 디렉토리 사용 설정을 구성하라는 메시지를 표시합니다.

  • 디렉토리 유형 지정

  • 디렉토리 위치 식별

  • 클라이언트가 접속 식별자를 검색할 수 있는 기본 Oracle Context 선택

  • 사용할 선호 이름 지정 방법으로 디렉토리 이름 지정

설치 후 Oracle Net Configuration Assistant를 사용하여 구성

또한 OracleNet Configuration Assistant는 다음 작업을 위해 독립형 모드로 실행될 수 있습니다.

  • 클라이언트 또는 서버 구성(위에서 설명)

  • Oracle Schema 생성

  • Oracle Context 생성

Oracle Net Configuration Assistant를 독립형 모드로 실행하려면 다음과 같이 하십시오.

  • UNIX의 경우 $ORACLE_HOME/bin에서 netca를 실행합니다.

  • Windows NT의 경우 시작 > 프로그램 > Oracle - HOME_NAME > 네트워크 관리 > Oracle Net Configuration Assistant를 선택합니다.

Oracle Net Manager로 구성

Oracle Net Manager를 사용하여 다음을 수행할 수 있습니다.

  • Oracle Context 변경

  • 인증서 변경

  • 네트 서비스 이름 생성

  • 네트 서비스 별칭 생성

  • 여러 프로토콜 주소를 통해 리스너에 대한 다중 경로 구성

  • 데이터베이스 서비스 또는 네트 서비스 이름에 대한 고급 CONNECT_DATA 정보 구성

  • 네트 서비스 이름 항목을 tnsnames.ora 파일에서 디렉토리 서버로 이전

디렉토리 이름 지정 구성에 액세스하려면 네비게이터 창에서 디렉토리 서비스 이름 지정을 선택합니다.

자세한 내용은 Oracle Net Configuration Assistant 도움말 시스템을 참조하십시오.
