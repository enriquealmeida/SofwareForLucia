Oracle Advanced Securityの認証方式の構成

中央で管理されるセキュリティ認証方式によって、分散環境でユーザー、クライアントおよびサーバーのアイデンティティの機密を保護できます。ネットワーク認証方式には、ユーザーがシングル・サインオンできるというメリットもあります。

Oracle Net Managerを使用して次の認証アダプタを構成します。

  • Kerberos

  • CyberSafe

  • RADIUS

  • SSL

  • Windows NTシステム固有の認証方式

注意: SSL付きTCP/IPプロトコルを構成している場合、デフォルトでSecure Sockets Layer (SSL)が構成されます。このタブで選択した認証方式は、SSLの認証方式より優先されます。SSLの追加オプションを構成するには、「SSL」タブを使用します。SSL付きTCP/IPプロトコルを使用する場合は、他の認証方式を使用しないでください。

認証方式を構成する手順は、次のとおりです。

  1. ナビゲータ・ペインで、「Oracle Netの構成」→「ローカル」→「プロファイル」を展開します。

  2. 右ペインのリストから「Oracle Advanced Security」を選択します。

  3. 「認証」タブを選択します。

  4. 使用する認証方式を「使用可能なメソッド」リストから選択し、右矢印ボタン(「>」)を選択します。

  5. 使用するメソッドに従って認証方式を順序付けします。「選択メソッド」リストから認証方式を選択し、「上へ」または「下へ」ボタンを使用してリスト内で選択項目を上下に移動します。

  6. 次の認証方式の追加パラメータを設定する場合は、「その他のパラメータ」タブを選択します。

    • Kerberos

    • CyberSAFE

    • RADIUS

  7. 「ファイル」→「ネットワーク構成の保存」を選択します。
