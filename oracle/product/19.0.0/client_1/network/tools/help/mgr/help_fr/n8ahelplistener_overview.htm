Présentation du processus d'écoute

Les services coordonnent leurs sessions à l'aide d'un processus d'écoute, ou processus sur le serveur qui reçoit les demandes de connexion de la part d'une application client. Les processus d'écoute sont configurés pour "écouter" les adresses de protocole pour un service de base de données ou un service autre qu'un service de base de données.

Les adresses de protocole sont configurées dans le fichier de configuration du processus d'écoute, listener.ora, pour un service de base de données ou autre. Un service de base de données Oracle9i ou Oracle8i inscrit automatiquement auprès du processus d'écoute ses informations, comme les noms de service et d'instance, et les données de chargement. Cette fonctionnalité, appelée inscription de service, n'exige aucune configuration de service dans le fichier listener.ora. D'autres services, notamment les services de base de données Oracle8 et Oracle7, ainsi que les outils de gestion Oracle Enterprise Manager exigent une configuration de serveur dans le fichier listener.ora .

Une fois configuré, le processus d'écoute peut être géré avec l'utilitaire LSNRCTL (Listener Control).
