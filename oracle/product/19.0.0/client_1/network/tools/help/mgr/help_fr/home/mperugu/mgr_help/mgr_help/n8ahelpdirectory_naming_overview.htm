Présentation de la résolution de noms d'annuaire

La résolution de noms d'annuaire est l'une des principales méthodes de résolution de noms utilisées pour convertir un identificateur de connexion en descripteur de connexion que le client utilise pour se connecter à une base de données. La résolution de noms d'annuaire stocke les identificateurs de connexion et les descripteurs correspondants dans un serveur d'annuaire compatible LDAP. Les identificateurs de connexion sont stockés sous un sous arbre de contexte Oracle qui contient les entrées à utiliser pour le logiciel Oracle.

Un utilisateur final entre une chaîne de connexion comportant un identificateur de connexion :

CONNERCT NomUtilisateur/MotDePasse@identificateur_connexion

Dans l'exemple suivant, sales est l'identificateur de connexion recherché dans l'annuaire :

CONNECT scott/tiger@sales

Le descripteur de connexion renvoyé au client comprend les éléments suivants :

  • Emplacement de l'adresse de protocole du processus d'écoute

  • Nom de service de la base de données

Le client utilise ces informations pour se connecter à un processus d'écoute. Le processus d'écoute, configuré avec les mêmes informations d'adresse, reçoit la demande de connexion. Si le processus est à l'écoute des demandes concernant le même service de base de données que celui qui est indiqué dans le descripteur de connexion du client, une connexion est autorisée.

Pour que les noms d'annuaire soient résolus, configurez un serveur d'annuaire pour les fonctions Oracle Net Services et configurez les clients pour accéder à l'annuaire.

Pour plus d'informations, reportez-vous au chapitre Configuration Management Concepts dans le Guide de l'administrateur Oracle10i Net.
