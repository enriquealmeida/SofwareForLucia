Registration Handler property

Allows developers to specify which procedure will be executed to register the device information needed to send messages (Push Notifications) to the device.

Description

The Procedure object set on this property must be generated as a web service. It will be executed from the Smart Device at application startup (which has Enable Notification property enabled).

GeneXus, by default, creates a procedure called NotificationsRegistrationHandler in GeneXus/SD/Notifications directory under the Root Module in your Knowledge Base that has:

  • Main program property = True
  • Connectivity Support property = Online

You can create another with your own business logic for registering the devices in your system. 

Run-time/Design-time

This property applies only at design-time.

How to apply changes

To apply changes made by this property, do a Build All.

Scope

Objects: Main Smart Devices
Platforms: Smart Devices(Android, IOS)
