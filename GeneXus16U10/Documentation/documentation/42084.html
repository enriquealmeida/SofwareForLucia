Isolation level property (GeneXus 16 Upgrade 1 or prior)

Sets the isolation level.

Values

Read Committed In this isolation level the programs do NOT see the changes made by other users until they perform a COMMIT. This is the default value.
Read Uncommitted In this isolation level the programs see the changes made by other users even if they haven't performed a COMMIT yet.

Description

Setting Isolation level affects read phenomena and concurrency control. Using Committed Read provides higher consistency levels but generates more locks on the database. There is more information about Isolation Level at https://en.wikipedia.org/wiki/Isolation_(database_systems) and the documentation of different DBMSes. 

Important:

  • The property is NOT valid for Oracle (Oracle Database doesn't use dirty reads, nor does it even allow them.). Even if the property is set in GeneXus, the generated programs use Committed Read.
  • The value set on this property is not taken into account for SQL Server. Even if the property is set to Read Committed, the generated programs use Uncommitted Read (SELECTs with (nolock) are generated).
  • The isolation level used for SQLite (Android and iOS programs) is Committed Read.

The property applies to the following DBMSes: Iseries, DB2 UDB, INFORMIX, MYSQL.

 

Scope

Platforms: Web(.Net, Java)
