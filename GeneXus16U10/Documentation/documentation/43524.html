GeneXus Standard Classes (GeneXus 16 Upgrade 3 and prior)

The GeneXus standard classes are a set of routines, written in native code, that are referenced by the generated code. As an example, the code to send an e-mail is not generated in every object that sends e-mails. It is "packed" as part of the Standard Classes and referenced by the generated code.

Every GeneXus Generator has its set of Standard Classes.

Distribution

Standard Classes in an executable format are distributed on every GeneXus version and are automatically installed.

Source code

Standard Classes source code is _not_ distributed with GeneXus for many generators (see below). If you want to take a look at the source code, you can download it from the GeneXus Assembla space. Both Java and .NET standard classes can be reached from http://svn2.assembla.com/svn/genexus_standard_classes/

License

You can download (if required), change and redistribute the Standard Classes source code. For details see the license information.

Packaging

Depending on the generator, those routines are packaged differently.

Java

In the Java space the GeneXus Standard routines are packaged as standard jar files

  • gxclassD.zip: Debug version, used when enabling the Log JDBC property.
  • gxclassR.zip: Release version (default version used).
  • GxUtils.jar: Developer Menu utilities.

To check the version used in an application call the gxver from the package com.genexus.webpanels for example 

http://serverhost:port/application/servlet/com.genexus.webpanels.gxver

The result should be similar to

Running GeneXus Runtime Classes Version 16_0_3-132732

C#

The C# generator provides the GeneXus standard routines in the GxClasses.dll assembly located on the "GeneXusInstallationFolder"\gxnet\bin folder.

Ruby

The Ruby generator provides the standard routines as standard ruby files (.rb extension) and are located on the "GeneXusInstallationFolder"\gxruby folder which are copied to the target generation folder during the generation phase.

iSeries Native

In order to execute any iSeries (Cobol|RPG) program you need to install the GX library; the standard GeneXus routines used by the GeneXus Cobol and RPG programs.

Android

The GeneXus Standard routines for Android are packaged on the FlexibleClient Eclipse project located on the "GeneXusInstallationFolder"\Android\FlexibleClient\ folder (java source files) which is copied to the target generation folder KnowledgeBaseName\ModelName\mobile\Android\FlexibleClient during the generation phase. This project depends on other projects located on the UserControlLibraries folder.

iOS

The GXFlexibleClient framework is the iOS GeneXus standard classes deployed as a framework which is referenced in your xCode project. You will notice it is copied and installed in the MAC machine during the compilation process. 

Blackberry

The Blackberry generator packages the GeneXus Standard routines in the KnowledgeBaseNavigator*.jar file located in the "GeneXusInstallationFolder"\BlackBerry\Templates\Prototyper\Prototyper folder.
