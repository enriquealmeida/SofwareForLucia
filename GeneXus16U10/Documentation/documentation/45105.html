Enable national language support property (GeneXus 16 Upgrade 4 or prior)

Allows you to insert multi-byte characters in character fields created in the database, in order to represent a greater range of characters. For example, it allows you to insert data in Chinese.

Values

No Does not allow you to insert multi-byte characters in the database. This is the default value.
Yes Allows you to insert multi-byte characters in character type fields.

Description

The property must be set at Version preferences level and it's granularity can be changed at Attribute definition or GeneXus Domains level.

Notes:

1- In order to apply changes: execute a Create database and Rebuild all objects operation when the version level property is set; The Impact Analysis does not take into account the change of this property. It is always necessary to recreate tables and objects.

When changing this property at attribute level there is an Impact Analysis, this feature is availlable since Genexus Evolution 2 Upgrade 3

2- GeneXus X and prior versions: The property must be configured in the Data Store's properties..

Run-time/Design-time

This property applies only at design-time.

How to apply changes

To apply changes made by this property, do a Re-Build All.

Scope

Available at the Data Store.
Data Store: Iseries, DB2 UDB, MYSQL, ORACLE, SQLSERVER
Platforms: Web(.Net, Java), Web(.Net Core)
