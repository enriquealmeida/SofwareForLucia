Newest Version

GeneXus 16 upgrade 1

This article is an overview of GeneXus 16 Upgrade 1 features (compared to GeneXus 16) and what needs to be taken into account to adopt it. 

It has been released on 30th November 2018.

Download

  • https://www.genexus.com/developers/DownloadCenter?,,,5667

Overview

This is an important upgrade that ships new features for integration with AI and Chatbots and allows to authenticate with SAML 2.0 identity providers. It includes improvements for Web and Mobile apps generation adding more flexibility to UX design. Furthermore, it adds stability fixes to the GeneXus IDE; all these are part of more than 260 improvements and bug fixes.

GeneXus Access Manager (GAM)

  • Integration with SAML 2.0 Identity Providers

Artificial Intelligence

  • Now you can process and analyze Videos
  • Google has been added to the list of supported providers
  • More at GeneXusAI: Release notes

Conversational Flows & Chatbot Generator

  • Dialog Flow's Follow up intents and Training phrases support added
  • This update is required given the new authentication mechanism of IBM Cloud / Watson

Team Development / GeneXus Server

  • On an already connected KB, you can now bring newly created versions from the server to the client using the Team Development Versions Dialog.
  • Performance improvements in 'update' operation

GeneXus for SAP Systems Improvements

Now you have a new property (Include In Master Page Menu) that allows you to customize your Fiori Master Page on an easier way.
Also, there is a new Fiori Mobile List Floorplan Template: the Timeline template, that allows you to list data on a template format.

Other

  • New properties related to UI and file uploading in File upload control 
  • New Compare function was released.
  • Improvements in reorganizations with SQL Server (avoiding the use of temporary tables in several cases)
  • Oracle Managed Driver as ADO.NET provider in .NET Applications
  • Improvements in several UX details, synchronization mechanism, user events and other for iOS and Android
  • IDE: Important Memory leak and fragmentation fixes

Compatibility

  • Interop.Scanbarcode method has been deprecated. Use Scanner.ScanBarcode method instead.
  • For KBs with offline smart devices applications, you need to manually import the GxBeforeEventReplicator procedure from <genexus_installation_directory>\Packages\Patterns\WorkWithDevices\Resources\SDApi.xml.
