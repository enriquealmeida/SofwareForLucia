Upgrade Licenses To A New GeneXus Version (GeneXus 16 or prior)

If you are comming from previous GeneXus version, the first step in order to use the new one is to migrate you licenses.

First of all, you need to uninstall all the licenses for those products that new version includes. This must be done using the previous version's License Manager by mean of the 'Uninstall' button. Copy and keep the uninstall code.

Important: Do not uninstall those products licenses that are not supported or released in the new version.

Then you need to ask for the last version licenses. In order to do that, run the GeneXus License Manager and follow the wizard after pressing the Authorize button. You need to attach the Uninstall code in this request.

Once you receive the answer with the site keys (a file named keys.gxa), run the Genexus License Manager again in order to authorize the products. 

Sharing Licenses With Previous Versions

When authorizing a GeneXus version, you are able to use previous versions (X Evolution 2, X Evolution 1, X, 9.0, 8.0, 7.5, 7.0, and 6.1) without the need to authorize or transfer the licenses between them. You just need to install some upgrades, as follows:

  • GeneXus 9.0: Install Development Environment 9.0 Upgrade 4 or higher.
  • GeneXus 8.0: Install Development Environment 8.0 Upgrade 7 or higher.
  • GeneXus 7.5: Install Development Environment 7.5 Upgrade 5 or higher.
  • GeneXus 7.0 and 6.1: Download this file and extract the protrun.dll file to GeneXus installation directory.

When authorizing make sure your are an administrator user; otherwise, you will get an error.
If your are using Windows Vista, Windows 7 or higher use the "Run as administrator" option to execute GeneXus License Manager.
 

If you use centralized licenses (installed in a server), you will have to install GeneXus Protection Server to be able to activate the keys.

To know the GeneXus Protection Server version installed, you can see how does the Control Panel description appears in Add or Remove Programs. If it does not detail GeneXus Protection Server 6.7, you must remove the version and install the new one.
