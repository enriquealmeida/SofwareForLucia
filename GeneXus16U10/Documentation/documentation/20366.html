Dynamic Services URL property

A common scenario in applications for Smart Devices is that the server where the app gets the data is given by the final user, so the Services URL can be changed at runtime.

Values

False
True

Description

Values

False: GeneXus generates the server URL in the code, and it doesn't allow the final user to change this value because it is embedded in the binary file. This is the default value.

True: GeneXus will allow the final user to change the server URL. The way to change this URL depends on the Operation System on which the device is running.

Note for iOS: Take into account that the URLs entered by the final user must comply with App Transport Security property group.

Note that if Dynamic Services URL property = False but the Services URL property is blank, the final user will be asked to enter its value anyway.
