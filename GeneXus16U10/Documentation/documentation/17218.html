Date Time storage timezone property

 

This is an Environment level property that controls whether DateTime data type attributes are all stored using the same TimeZone in the Database or not. It is intended to solve The TimeZone problem.

The property is also used to enable/disable TimeZone Support.

Values

Undefined

The TZ of DateTime attributes is Unknown. No conversion from or to the Current TimeZone is made.

This is the default value for Knowledge Bases converted from previous versions in order to preserve compatibility.
GMT/UTC

All DateTime data type atributes are stored in UTC. Convertion from and to the Current TimeZone is automatic.

This is the default value for new Knowledge Bases.
Application Server

All DateTime data type atributes are stored in the application server's TZ. Convertion from and to the Current TimeZone is automatic.

Use this value if you are converting your application from previous GeneXus versions and you positivelly know that the actual data in the Database is stored in the application server's timezone. You get TimeZone support without having to convert all DateTime values in your Database.

Description

Sets the TimeZone in which the DateTime fields are stored in the Data Base. Depending on the value set, it enables TimeZone Support.
