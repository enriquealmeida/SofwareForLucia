Deploy to cloud property

To facilitate cloud prototyping deploying the application to GeneXus cloud.

Values

No This is the default value.
Yes Set with default values to deploy the application to GeneXus Cloud

Description

Setting this property to "Yes", Database name property, Web Root property, Deploy Virtual Directory property and Deploy Server URL property, Services URL property are automatically set with default values to deploy the application to GeneXus cloud. Once the application is run it's deployed to GeneXus cloud.

Note

GAM repository and application is also deployed to the cloud. This means that the GAM database will be created automatically on the cloud.

Steps

  1. Set related properties

    1. Java Specific: The property "Use Annotation for servlet definition" must be set to "No" when deploying to apps2.genexusx.com. The reason is that apps2.genexus.com is hosted by Tomcat 6.0. 

    2. Review affected properties ( Database name property, Web Root property, Deploy Virtual Directory property and Deploy Server URL property, Services URL property). You may want to point to others Servers available for Cloud prototyping.

    3. iOS specific: You may need to configure App Transport Security property group if Services URL property uses SSL. 

  2. Create Database

  3. Rebuild all Objects

How to apply changes

To apply changes made by this property, do a Re-Build All.

Scope

Platforms: Web(.Net, Java)
