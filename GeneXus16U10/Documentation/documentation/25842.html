GeneXus Installation Steps

This chapter describes the steps to install GeneXus 16. Note that you must have Administrator rights to execute the installation and make sure to install as a user with full control of the target installation folder.

Below are detailed instructions for the installation process.

  1. Be sure to fulfill GeneXus 16 hardware and software requirements
  2. Run the Setup of GeneXus 16 installer
  3. A dialog like the following appears:

    By clicking on 'Install', you accept the License Terms, and the 'typical' installation process begins.
    • GeneXus 16 is installed in the displayed Product Install path
    • Default GAM Platforms are installed (i.e. SQL Server and MySQL)
      (follow up reading in step 5)
  4. By clicking on 'License Terms', these are displayed in 4 different languages:

    By clicking on 'Install', you accept the License Terms, and the 'typical' installation process begins.
    • GeneXus 16 is installed in the displayed Product Install path
    • Default GAM Platforms are installed (i.e. SQL Server and MySQL)
      (follow up reading in step 5)
  5. To change the GAM Platforms to be installed, you need to select 'Custom' in the dialog of step 3:

    By clicking on 'Next', you accept the License Terms, and you can select additional GAM Platforms to install.

  6. Click on 'Install' to install GeneXus 16 and the selected GAM Platforms.
  7. Now you are installing GeneXus 16.

    If you click on 'Cancel' the installation process is canceled.
  8. When installation finishes, the following dialog appears:

    Click on 'Close' to finish setup. Click on 'Run' to open GeneXus 16.

Note: At the end of the setup process a Genexus.exe /install operation is executed to register User Controls, Extensions, and Documentation. This process creates in the installation folder the files:

  • userControls.ari
  • *.supportfiles
  • *.html

These files are needed for the standard GeneXus usage.

FAQ

Q:Can I have more than one GeneXus 16 installation in the same PC?

A:You canâ€™t have more than one local installed version. When executing the setup for the second time, the previous installation is detected and a dialog box is displayed with the following options: Repair/Reinstall and Uninstall GeneXus 16. These options apply to the previous installation.

However, it is possible to have any number of local working versions, just duplicating GeneXus' root folder.

Also, you can have a local installation and use a network version by executing the Workstation Setup.

Q: Can I have more than one GeneXus version installed on the same PC?

A: Yes, you can. The only constraint is that they must be installed in different directories.

Q: Can I share a GeneXus installation?

Yes, a GeneXus version installed on a network disk can be shared with other users. They will need to run the GeneXus Workstation Setup

Q: I downloaded a GeneXus 16 Build and got the following error, Unable to load DLL 'ProtAuth.dll':The specified module could not be found (Exception from HRESULT: 0x8007007E). What should I do?

The error detail is the following:

Unable to load DLL 'ProtAuth.dll':The specified module could not be found (Exception from HRESULT: 0x8007007E) (Artech.Common.Helpers)
   at Artech.Common.Helpers.Protection.AuthorizationHelper.SetLanguage1_Com(UInt32 dwLCID)
   at Artech.Common.Helpers.Protection.AuthorizationHelper.SetLanguage1(Int32 lcid)
   at Genexus.Win.GeneXusMain.OxtrdDTQ5(String[]  , String&  , String&  )

If possible install directly from the GeneXus 16 setup so internal dependencies are installed. Another option is to manually download the Visual C++ Redistributable for Visual Studio 2017 (x86) and install it.
