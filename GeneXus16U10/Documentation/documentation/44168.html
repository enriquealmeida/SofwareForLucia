MSBuild options property

Defines switches to specify aspects of the build process such as project properties (i.e. Configuration type Debug or Release) or msbuild processes settings (i.e. maxcpucount to run builds in parallel).

Description

By default this property is hidden. If you create in the directory where GeneXus.exe is available, a file named "msbuildcompilation.flag" (empty, it's just a flag), the MSBuild options property appears at Generator level.

Run-time/Design-time

This property applies only at design-time.

Samples

Below is a configuration sample for this property:

/p:PlatformTarget=x86 /p:Configuration=Debug /p:GxExternalReference=Artech.Security.Common.dll

The above indicates three properties for the projects:

/p:PlatformTarget = x86 --- To be compiled to run in 32 bits.
/p:Configuration = Debug ---To be compiled in debug mode, so that it generates the .pdb of each dll.
/p:GxExternalReference=<assembly> --- References to external dll are added.

For the build process itself you could set:
/maxcpucount or / m --- If you compile several projects, you compile them in parallel, which is faster.

Availability

This property is available since GeneXus 16 upgrade 6.

Scope

Level: Generator
