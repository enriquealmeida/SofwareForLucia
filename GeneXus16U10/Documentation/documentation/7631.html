First year of 20th century property

When using two-digit numbers, it indicates to GeneXus if they belong to the 20th or to the 21st century.

Description

The programs generated by GeneXus require the user to enter only the last two digits of the year, but they internally store full four digit format. If the entered number is less than 40 (iSeries default), the year is assumed to be in the 21st century (not 20th). That is, it is not  possible for the user to type in a date prior to 1940.

This property will indicate to GeneXus that the two digit number still belongs to the 20th century. Digit numbers lower than the default value will be considered as belonging to the 21st century.

Values

Any value between 00 and 99.

Default Value: depends on the selected generator:

  •    Cobol, RPG, .NET and Java generators: 40
  •    Visual FoxPro generator: 0

Samples

10 is set as the start of the 20th century year. This tells GeneXus that if the user types in a number greater than or equal to 10 for a given year, it will be assumed that the date belongs to the 20th century (i.e. 10 means 1910, 70 means 1970, 07 means 2007).

Note:

This option does not apply to any internal date calculations, which are always performed on a full four digit year basis.

How to apply changes

To apply changes made by this property, do a Build with this Only of the object.

Scope

Platforms: Web(.Net, Java)
