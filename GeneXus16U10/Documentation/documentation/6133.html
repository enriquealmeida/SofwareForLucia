What Are Themes?

A Theme is a GeneXus object used to assign property values to the various web controls (buttons, grids, forms, etc.) in order to configure what they will look like when a web page is loaded. Themes can be applied to objects such as Web Panels and Transactions. The advantage of using Themes is basically that to change an application's look and feel you only have to change the Theme, nothing else.

Themes contain classes, and you can assign values to the predefined properties of the clasess. You cannot assign behaviors through these properties, only cosmetic features. For example, a button cannot be ReadOnly, but it can have rounded corners and a special color scheme chosen by the web designer. This is another interesting feature of Themes: they allow for high-level interaction between graphic designers and GeneXus analysts.

Once a Theme is associated to a Web Object through its Theme property, you can associate any of this object's controls (for example, controls inside a Web Form) to a Theme class. When you associate a control to a class, it inherits the configuration of properties given in the class.

In addition, Themes enable you to change these interface features at runtime.Â 

Technically speaking, this functionality is possible thanks to GeneXus technology developed on Cascade Style Sheets (CSS), that is to say, through the implementation of class hierarchy and inheritance in a CSS. This technology also provides a significant improvement in application performance, as all the possibilities of CSS can be applied.
