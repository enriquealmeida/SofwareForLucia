IndexOf Method

Allows searching for a string within another one. 

Syntax

str1.IndexOf(str2 [, start-char ]);

Type Returned:
Numeric

Where:
str1
    String on which the search will be done. It may be Char, Varchar or Long varchar type.

str2
    Search string. It may be Char, Varchar or Long varchar type.

start-char
    Optional. It indicates the search start position. This variable is a positive number type.

Description

IndexOf method returns the position (integer index starting in 1) of the str1 string where the str2 string was found. It searches from front to back. Optional is the starting position in the search; if it is omitted, the starting position is the position 1 of the str1 string.

Notes:

  • The start-char parameter is a positive number type and indicates the initial position of the search within the str1.  If it is 0, both functions return to 0.
  • The comparison between characters is case sensitive; i.e.:  'a' is different from 'A'.
  • If str1 is empty or the length is shorter than str2, both functions return 0, since the search cannot be carried out.
  • If str2 is empty, it returns start-char if it was initialized; otherwise it returns 1. 

Example

&str1 ="StrSearch function tests"
&Nbr = &str1.IndexOf("j")    // Returns 0 ("j" not found)
&Nbr = &str1.IndexOf("a")    // Returns 6
&Nbr = &str1.IndexOf("e", 7) // Returns 21

Scope

Objects: Procedures, Transactions, Web Panels, Work Panels
Languages: .NET, Java, Ruby (up to GeneXus X Evolution 3), Visual FoxPro (up to GeneXus X Evolution 3)
