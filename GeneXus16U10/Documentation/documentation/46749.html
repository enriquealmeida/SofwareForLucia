Manage Module References (GeneXus 16 Upgrade 11 or prior)

The Manage Module Reference option is another intended (along with Knowledge Manager Import) for knowledge-sharing among developers. You can install a module provided by other developers which have packaged their funcionalities for distribution. As a difference from importing an xpz, you are able only to see the object definition (i.e.. parm rule for Procedures, tree of properties/methods/events for External Objects, etc.).

The Manage Module Reference dialog allows you to:

  1. Look for external modules on servers (including your local machine).
  2. Install or upgrade a module from a specific server (including your own packaged modules).
  3. Get information about the module, such as its version, author, description, platforms available, etc.

The Install/Upgrade process updates the Output window with relevant process status, warnings and errors.
