Minimum Time Between Receives property

Applies for Offline Smart Devices applications with Automatic Offline Data SynchronizationÂ criteria. This property determinates the time between calls to the synchronization programs.

Description

The user can set the minimum time interval that needs to pass before the application calls the synchronization's receive services.

The application only performs the Receive operation when the application starts, and only if the time defined by this property has passed.

Values

Value (in seconds) Minimum time lapse the app waits between calls for the Synchronization.Receive programs.

How to apply changes

To apply changes made by this property, Run the main object.

Scope

Objects: Offline Database
Platforms: Web(.Net, Java), Smart Devices(Android, IOS)
