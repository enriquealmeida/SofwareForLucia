Save State property

Enables or disables state saving. When it is enabled, grid state information such as pagination position, filtering, and sorting will be saved in the Web Session. When the grid container page is reloaded, the grid state will be restored to the last known configuration.

Values

False
True

Scope

Objects: Web Panel
Platforms: Web (.Net, .Net Core, Java)
Controls: Grid, Free Style Grid

Description

The default value of this property is 'False,' except in grids created by the Work With for Web pattern that set it to 'True.'

When the property is set to 'True', the information about the state (such as pagination position, filtering, and sorting) is stored in the web session automatically and also retrieved from it. 

The program loads the stored information, if it exists, before the Start Event, and also stores it before the Refresh Event happens, when some related information changes.

Grid's State property

The Grid State property can be accessed and changed at runtime. Its type is SDT GridState, which is part of the GeneXus Core module. You may use it to persist the state somewhere else; for example, to store it in the database, associated with your application user's preferences.

Run-time/Design-time

This property applies only at design-time.

Samples

When Save State is 'True,' the grid screen state is implicitly loaded from the State property at the start of the Start Event. This means that to change the state to be loaded, you have to do so in the Start event as shown in the example below:

Event Start
  Grid1.State = GetGridStateFromBD(&pgmname, !”Grid1”)
EndEvent

In the same way, the grid screen state is implicitly stored in the State property at the start of the Refresh Event. So, to save the state somewhere else, you need to do so in the Refresh Event.

Event Grid1.Refresh
   SaveGridStateOnDB(&pgmname, !”Grid1”, Grid1.State)
EndEvent

Availability

This property is available since GeneXus 16 Upgrade 11.
